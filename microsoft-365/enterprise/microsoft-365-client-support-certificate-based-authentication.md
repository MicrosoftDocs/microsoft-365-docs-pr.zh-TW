<span data-ttu-id="f32c7-109">新式驗證是透過使用驗證程式庫（如 Active Directory 驗證程式庫 (ADAL) 或 Microsoft 驗證程式庫 (MSAL) ）來啟用。</span><span class="sxs-lookup"><span data-stu-id="f32c7-109">Modern authentication is enabled through the use of an authentication library, like Active Directory Authentication Library (ADAL) or Microsoft Authentication Library (MSAL).</span></span> <span data-ttu-id="f32c7-110">新式驗證是指用戶端用來驗證和授權 Microsoft 365 資源的存取。</span><span class="sxs-lookup"><span data-stu-id="f32c7-110">Modern authentication is what clients use to authenticate and authorize access to Microsoft 365 resources.</span></span> <span data-ttu-id="f32c7-111">新式驗證利用 OAuth，並為用戶端提供一種安全機制，以存取 Microsoft 365 服務，而不需要存取使用者認證。</span><span class="sxs-lookup"><span data-stu-id="f32c7-111">Modern authentication leverages OAuth and provides a secure mechanism for clients to access Microsoft 365 services, without requiring access to user credentials.</span></span> <span data-ttu-id="f32c7-112">在登入時，使用者會直接透過 Azure Active Directory 進行驗證，並接收 return 中的存取/重新整理權杖組。</span><span class="sxs-lookup"><span data-stu-id="f32c7-112">At sign-in, the user authenticates directly with Azure Active Directory and receives an access/refresh token pair in return.</span></span> <span data-ttu-id="f32c7-113">存取權杖會授與用戶端存取權給 Microsoft 365 租使用者中的適當資源。</span><span class="sxs-lookup"><span data-stu-id="f32c7-113">The access token grants the client access to the appropriate resources in the Microsoft 365 tenant.</span></span> <span data-ttu-id="f32c7-114">當目前的存取權杖到期時，會使用重新整理權杖來取得新的 access 或重新整理權杖。</span><span class="sxs-lookup"><span data-stu-id="f32c7-114">A refresh token is used to obtain a new access or refresh token pair when the current access token expires.</span></span>

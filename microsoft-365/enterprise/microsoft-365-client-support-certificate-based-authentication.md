<span data-ttu-id="c9d30-105">新式驗證是驗證和授權方法組合的傘條款。</span><span class="sxs-lookup"><span data-stu-id="c9d30-105">Modern authentication is an umbrella term for a combination of authentication and authorization methods.</span></span> <span data-ttu-id="c9d30-106">包括：</span><span class="sxs-lookup"><span data-stu-id="c9d30-106">These include:</span></span>

- <span data-ttu-id="c9d30-107">**驗證方法**：多重要素驗證;以用戶端憑證為基礎的驗證。</span><span class="sxs-lookup"><span data-stu-id="c9d30-107">**Authentication methods**: Multi-factor Authentication; Client Certificate-based authentication.</span></span>

<span data-ttu-id="c9d30-109">新式驗證是透過使用驗證程式庫（如 Active Directory 驗證程式庫 (ADAL) 或 Microsoft 驗證程式庫 (MSAL) ）來啟用。</span><span class="sxs-lookup"><span data-stu-id="c9d30-109">Modern authentication is enabled through the use of an authentication library, like Active Directory Authentication Library (ADAL) or Microsoft Authentication Library (MSAL).</span></span> <span data-ttu-id="c9d30-110">新式驗證是指用戶端用來驗證和授權 Microsoft 365 資源的存取。</span><span class="sxs-lookup"><span data-stu-id="c9d30-110">Modern authentication is what clients use to authenticate and authorize access to Microsoft 365 resources.</span></span> <span data-ttu-id="c9d30-111">新式驗證利用 OAuth，並為用戶端提供一種安全機制，以存取 Microsoft 365 服務，而不需要存取使用者認證。</span><span class="sxs-lookup"><span data-stu-id="c9d30-111">Modern authentication leverages OAuth and provides a secure mechanism for clients to access Microsoft 365 services, without requiring access to user credentials.</span></span> <span data-ttu-id="c9d30-112">在登入時，使用者會直接透過 Azure Active Directory 進行驗證，並接收 return 中的存取/重新整理權杖組。</span><span class="sxs-lookup"><span data-stu-id="c9d30-112">At sign-in, the user authenticates directly with Azure Active Directory and receives an access/refresh token pair in return.</span></span> <span data-ttu-id="c9d30-113">存取權杖會授與用戶端存取權給 Microsoft 365 租使用者中的適當資源。</span><span class="sxs-lookup"><span data-stu-id="c9d30-113">The access token grants the client access to the appropriate resources in the Microsoft 365 tenant.</span></span> <span data-ttu-id="c9d30-114">當目前的存取權杖到期時，會使用重新整理權杖來取得新的 access 或重新整理權杖。</span><span class="sxs-lookup"><span data-stu-id="c9d30-114">A refresh token is used to obtain a new access or refresh token pair when the current access token expires.</span></span>

<span data-ttu-id="c9d30-115">新式驗證支援不同的驗證機制，類似憑證型驗證。</span><span class="sxs-lookup"><span data-stu-id="c9d30-115">Modern authentication supports different authentication mechanisms, like certificate-based authentication.</span></span> <span data-ttu-id="c9d30-116">Windows、Android 或 iOS 裝置上的用戶端可以使用憑證型驗證 (CBA) ，以在裝置上使用用戶端憑證來驗證 Azure Active Directory。</span><span class="sxs-lookup"><span data-stu-id="c9d30-116">Clients on Windows, Android, or iOS devices can use certificate-based authentication (CBA) to authenticate to Azure Active Directory using a client certificate on the device.</span></span> <span data-ttu-id="c9d30-117">使用憑證來從 Azure Active Directory 取得存取/重新整理權杖，而不是一般的使用者名稱/密碼。</span><span class="sxs-lookup"><span data-stu-id="c9d30-117">Instead of a typical username/password, the certificate is used to obtain an access/refresh token pair from Azure Active Directory.</span></span>

<span data-ttu-id="c9d30-118">深入瞭解 [憑證型驗證](https://docs.microsoft.com/azure/active-directory/authentication/active-directory-certificate-based-authentication-get-started)。</span><span class="sxs-lookup"><span data-stu-id="c9d30-118">Learn more about [certificate-based authentication](https://docs.microsoft.com/azure/active-directory/authentication/active-directory-certificate-based-authentication-get-started).</span></span>

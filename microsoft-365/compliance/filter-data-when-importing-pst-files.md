# <a name="filter-data-when-importing-pst-files"></a>匯入 PST 檔案時篩選資料

使用 Microsoft 365 匯入服務中的新智慧匯入功能，篩選實際匯入至目標信箱的 PST 檔案中的專案。 以下為運作方式：
  
- 在您建立並送出 PST 匯入工作之後，PST 檔案會上傳至 Microsoft 雲端中的 Azure 儲存體區域。
  
- Microsoft 365 以安全且安全的方式來分析 pst 檔案中的資料，方式是識別信箱專案的保留天數，以及 PST 檔案中包含的不同郵件類型。
  
- 分析完成且資料準備好匯入時，您可以選擇是否要將 PST 檔案中的所有資料當做原樣匯入，或透過設定篩選器來剪裁所匯入的資料，以控制要匯入的資料。 例如，您可以選擇：
  
  - 只匯入某段時間內的專案。
  
  - 匯入選取的郵件類型。
  
  - 排除特定人員所傳送或接收的郵件。
  
- 設定篩選設定之後，Microsoft 365 只會將符合篩選準則的資料匯入到匯入工作中指定的目標信箱。
  
下圖顯示智慧匯入程式，並強調您執行的工作，以及 Office 365 所執行的任務。
  
![Office 365 中的智慧匯入程式](../media/f2ec309b-11f5-48f2-939c-a6ff72152d14.png)
  
## <a name="create-a-pst-import-job"></a>建立 PST 匯入工作

- 本主題中的步驟假設您已使用「網路上傳」或「磁片磁碟機運送」，在 Office 365 匯入服務中建立 PST 匯入工作。 如需逐步指示，請參閱下列其中一個主題：
    
  - [使用網路上傳將 PST 檔案匯入 Office 365](use-network-upload-to-import-pst-files.md)
    
  - [使用磁碟機寄送將 PST 檔案匯入 Office 365](use-drive-shipping-to-import-pst-files-to-office-365.md)
    
- 使用 [網路上傳] 建立匯入工作之後，安全性 & 合規性中心的 [匯入] 頁面上的匯入工作狀態會設定為 [**正在進行分析**]，這表示 Microsoft 365 正在分析您所上傳的 PST 檔案中的資料。 按一下[重新整理重新整理] ![ ](../media/165fb3ad-38a8-4dd9-9e76-296aefd96334.png) 以更新匯入工作的狀態。 
    
- 針對磁片磁碟機運送匯入工作，在 Microsoft 資料中心人員收到您的硬碟並上傳 PST 檔案至您組織的 Azure 儲存體區域之後，就 Microsoft 365 會分析資料。
  
## <a name="filter-data-that-gets-imported-to-mailboxes"></a>篩選匯入至信箱的資料

在您建立 PST 匯入工作之後，請遵循下列步驟，在將資料匯入至 Office 365 之前先加以篩選。
  
1. 移至 <https://compliance.microsoft.com>，然後使用您組織中的系統管理員帳戶認證來登入。
    
2. 在 [Microsoft 365 合規性中心] 的左窗格中，按一下 **[資訊控管]** \> **[匯入]**。
    
    您的組織的匯入工作會列在 [匯 **入**] 索引標籤上。[**狀態**] 欄中的 [**分析完成**] 值會指出已 Microsoft 365 進行分析的匯入工作，並且已準備好要匯入。
    
    ![分析完成狀態表示 Microsoft 365 已分析 PST 檔案中的資料](../media/de5294f4-f0ba-4b92-a48a-a4b32b6da490.png)
  
3. 選取您要完成的匯入工作，然後按一下 [匯 **入] 以 Office 365**。
  
    隨即會顯示彈出頁面，以及關於該匯入工作的 PST 檔案資訊和其他資訊。

4. 按一下 [匯 **入至 Office 365**]。
    
    隨即會顯示 [篩選您的資料]。 它包含有關匯入工作之 PST 檔案中資料的相關資訊，包括資料時期的相關資訊。 
    
    ![[篩選您的資料] 頁面會顯示匯入工作之 PST 檔案的資料洞察力](../media/3b537ec0-25a4-45a4-96d5-a429e2a33128.png)
  
5. 根據是否要裁切要匯入至 Microsoft 365 的資料，在 [**您想要篩選資料？**] 底下，執行下列其中一項操作：
  
    a. 按一下 **[是，我想要在匯入之前加以篩選]** ，以裁切您匯入的資料，然後按 **[下一步]**。
  
    [匯 **入資料至 Office 365** ] 頁面會顯示 Microsoft 365 所執行之分析的詳細資料洞察力。 
  
    ![Microsoft 365 顯示來自其 PST 檔案分析的詳細資料洞察力](../media/4881205f-0288-4c32-a440-37e2160295f2.png)
  
    此頁面上的圖形會顯示要匯入的資料量。 在 PST 檔案中的每一封郵件類型的相關資訊都會顯示在圖形中。 您可以將游標懸停于每個橫條圖上方，以顯示該郵件類型的特定資訊。 根據 PST 檔的分析，也有一個具有不同 age 值的下拉式清單。 當您在下拉式清單中選取年齡時，會更新圖形以顯示所選取的年齡要匯入的資料量。 
  
    b. 若要設定額外篩選以減少匯入的資料量，請按一下 [ **更多篩選選項**]。
  
    ![在 [其他選項] 頁面上設定篩選，以裁切匯入的資料](../media/3f8d68c3-3fe2-4b4e-9488-b368b98fa9fe.png)
  
    您可以設定這些篩選器：
  
      - **Age** -選取年齡，只會匯入比指定的年齡更新的專案。 請參閱 [詳細資訊](#more-information)一節，以取得 Microsoft 365 如何判斷 **年齡** 篩選的年齡桶的描述。 
  
      - **類型** -此區段會顯示在匯入工作的 PST 檔案中找到的所有郵件類型。 您可以在要排除的郵件類型旁取消勾選方塊。 您無法排除其他郵件類型。 請參閱 [More information](#more-information) 一節以取得其他類別中包含的信箱專案清單。
  
      - **使用者** -您可以排除特定人員所傳送或接收的郵件。 若要排除出現在 [寄件者：] 欄位、[收件者：] 欄位或郵件 [副本：] 欄位中的人員，請按一下該收件者類型旁的 [ **排除使用者** ]。 輸入人員的電子郵件地址 (的 SMTP 位址) **，按一下 [** ![ 新增圖示]， ](../media/457cd93f-22c2-4571-9f83-1b129bcfb58e.gif) 將其新增至該收件者類型的已排除使用者清單，然後按一下 [**儲存**] 以儲存排除的使用者清單。 
  
        > [!NOTE]
        > Microsoft 365 不會顯示設定 **人員** 篩選所產生的資料洞察力。 不過，如果您將此篩選器設定為排除特定人員所傳送或接收的郵件，則會在實際的匯入程式期間排除這些郵件。 
  
    c. 按一下 [**其他篩選選項**] [**飛入]** 頁面中的 [套用]，以儲存篩選設定。 
  
    [匯 **入資料至 Office 365** ] 頁面上的資料洞察力會根據您的篩選設定進行更新，包括將根據篩選設定匯入的總數據量。 篩選設定的摘要也會顯示出來。 您可以按一下篩選旁邊的 [ **編輯** ]，以視需要變更設定。 
  
    ![資料洞察力會根據您的篩選設定進行更新](../media/897e20fb-3b13-44c3-9d56-9f330750f2a3.png)
  
    d. 按 [下一步 **]**。
  
    隨即會顯示 [狀態] 頁面，顯示篩選設定。 同樣的，您也可以編輯任何篩選設定。
  
    e. 按一下 [匯 **入資料** ] 以開始匯入。 隨即會顯示要匯入的資料總量。 
  
    或者
  
    a. 按一下 [**否，我要** 匯入所有內容]，將 PST 檔案中的所有資料匯入 Office 365，然後按 **[下一步]**。
  
    b. 在 [匯 **入資料至 Office 365** ] 頁面上，按一下 [匯 **入資料**] 以開始匯入。 隨即會顯示要匯入的資料總量。 
  
6. **按一下 [** 匯 **入**] 索引標籤上的 [重新整理重新整理] ![ ](../media/165fb3ad-38a8-4dd9-9e76-296aefd96334.png) 。 [ **狀態** ] 欄中會顯示匯入工作的狀態。
  
7. 按一下 [匯入工作]，以顯示更詳細的資訊，例如每個 PST 檔案的狀態，以及您設定的篩選設定。

## <a name="more-information"></a>其他資訊

- Microsoft 365 如何決定年齡篩選的增加？ 當 Microsoft 365 分析 PST 檔案時，它會查看每個專案的傳送或接收時間戳記 (如果專案同時有傳送和接收的時間戳記，就會選取 [最舊的日期]) 。 然後 Microsoft 365 查看該時間戳記的 year 值，並將它與目前的日期進行比較，以判斷專案的年齡。 然後，這些年齡會做為 **時期** 篩選的下拉式清單中的值使用。 例如，如果 PST 檔案有來自2016、2015及2014的郵件，則 **Age** 篩選中的值會是 **1 年**、 **2 年** 和 **3 年**。

# <a name="working-with-microsoft-compliance-score-preview"></a>使用 Microsoft 合規性分數 （預覽）

## <a name="managing-your-workflow-with-improvement-actions"></a>管理您的工作流程與改進動作

使用合規性分數改進動作可以集中您合規性工作流程。 改進動作建議對齊資料保護規定與標準，建議的動作，並提供詳細的實作指南。 您可以將它們指派給使用者執行必要的實作和測試的工作。 您也可以儲存文件與備忘稿並記錄狀態更新本身的改進巨集指令中的權限。

## <a name="view-your-improvement-actions"></a>檢視您改進的動作

合規性分數儀表板會顯示您的**關鍵的改進動作**-閱讀有最高資料點的最重要的問題。

若要檢視所有的改進動作，請選取**改進動作**] 索引標籤上 [儀表板，或移至 [儀表板上的關鍵的改進動作清單中選取 [**檢視所有改進動作**。 這將帶您前往**改進動作**] 頁面，其中您可以在此處看到所有您組織的改進的動作。

如果您有動作的詳細清單，可能很有幫助篩選檢視。 若要這麼做，請選取 [右上角的 [動作] 清單的**篩選器**。 [**篩選**] 彈出式視窗窗格出現時，然後選取您想要的準則基於法規和標準、 解決方案和群組。 您可以也自訂您檢視在右上角中選取**群組**，從下拉式功能表中，選取 [檢視群組、 解決方案、 類別、 的動作類型或狀態。

此頁面上的預設檢視不會顯示**通過**測試狀態改進動作。 若要檢視已通過測試的動作，檢查 [**篩選**] 彈出式視窗窗格中的 [ **Passed** ] 方塊。 僅限動作與測試狀態的**Passed**會計入您的分數。

改進動作] 頁面會顯示下列資料點的每個改進動作：

- **分數影響**： 依據完成動作時，將會增加整體的分數的點
- **法規**： 規定或標準相關巨集指令
- **群組**： 群組對其指派巨集指令
- **解決方案**： 您可以移至執行動作的解決方案
- **「 評估 」**: 評估 （這會將組織以符合特定的合規性目標控制項） 中位於巨集指令
- **類別**： 相關的資料保護類別 （亦即，保護資訊、 管理裝置等）。
- **測試狀態**：
    - **無**-記錄沒有狀態更新
    - **不評估**-測試尚未啟動
    - **不在範圍**-合規性分數計算列印會排除，不會增加您的分數
    - **部分測試**-測試尚未完成
    - **失敗高風險**-實作的測試失敗，且具有適用的標準的非規範的風險較高
    - **Passed**實作順利測試
- **Pointed 達到**： 顯示點達到不足無法盈餘分析的最大值

若要開始實作工時改進巨集指令，您可以自行執行工作，或將其指派給另一位使用者。 指派的人員可能是：

- 企業原則擁有者

## <a name="change-improvement-action-status"></a>變更改進巨集指令狀態

您可以將記錄的實作狀態日期，並測試狀態與日期，每個改進巨集指令。 以下是可用的欄位及狀態選項：

- **實作狀態**： 選取 [從下列狀態的選項：
    - **尚未實作**-尚未實作巨集指令
    - **Implemented**實作巨集指令
    - **替代實作**-選取此選項如果您使用其他協力廠商工具，或是不採取其他動作中包含 Microsoft 建議
    - 實作計劃**計劃**巨集指令
    - **不在範圍**-貴組織; 不相關的動作的選項。選取此狀態會從計分; 排除巨集指令取消選取它將會在計分包含巨集指令
- **實作日期**： 可用的欄位，當實作狀態設為**Implemented**或**替代實作**;切換到快顯選取日期的行事曆
- **測試狀態**： 選取 [從下列選項：
    - **不評估**-測試尚未啟動
    - **Passed**實作順利測試
    - **失敗低風險**-測試失敗、 低風險
    - **失敗中度風險**-測試失敗中, 度風險
    - **失敗高風險**-測試失敗]、 [高風險
- **測試日期**： 切換到快顯選取日期的行事曆

# <a name="compliance-offering-health-insurance-portability-and-accountability-hipaa--hitech-acts"></a>合規性供應項目： 健康保險流通與責任 (HIPAA) & HITECH 動作

## <a name="hipaa-and-the-hitech-act-overview"></a>HIPAA 和 HITECH 動作概觀

健康保險流通與責任法案 (HIPAA) 已建立的使用中外, 洩，需求美國醫療法律與個別識別健康資訊保護。 它會套用至涵蓋實體 — 醫生的辦公室、 醫院、 健康情況負責確認保戶和其他醫療保健公司 — 存取病患的受保護的健康情況資訊 (PHI)，以及商務合作夥伴的資料，例如雲端服務和 IT 提供者，處理程序 PHI在其代理。 （例如宣告或其自己的資料處理函數不執行最涵蓋的實體; 他們所依賴業務夥伴，若要這麼做）。

法律規定了使用及散佈的 PHI 四個一般區域中：

- 隱私權、 涵蓋病患機密性。
- 安全性、 處理的資訊，包括實體、 技術層面，和系統管理的防護措施的保護。
- 識別項，都無法再釋出如果收集的研究用途的資訊類型。
- 電子醫療保健相關的交易，其中包括資格以及保險理賠和付款中的資料傳輸的代碼。

HIPAA 範圍已使用經濟和臨床健康情況 (HITECH) 動作的健康情況資訊技術條例擴充。HIPAA 和 HITECH 動作的規則在一起，包括：

- HIPAA 隱私權規則，以著重於控制的權限的個人使用其個人的資訊，並涵蓋 PHI，限制其使用和外洩的機密性。
- HIPAA 安全性規則，設定系統管理、 技術、 和實體的防護功能，可防止未經授權的存取、 使用及洩漏電子 PHI 的標準。 此外，它還會包含此類組織的需求與業務相關協議 (Baa)。

HITECH 外洩通知最後一個規則，它需要提供通知給個人及不安全 PHI 政府外洩時，就會發生。

## <a name="microsoft-and-hipaa-and-the-hitech-act"></a>Microsoft 的 HIPAA 和 HITECH 動作

HIPAA 法規要求涵蓋的實體與他們的公司建立關聯 — Microsoft 當它提供服務，包括雲端的服務在此例中，涵蓋實體 — 輸入以確保這些業務夥伴會適當地之間的合約保護 PHI。 這些合約或 Baa，釐清及限制如何處理 PHI，並設定每一方堅持 HIPAA 和 HITECH 動作中所設定的安全性和隱私權佈建商務關聯。一旦 BAA 已經準備就緒，Microsoft 客戶 — 涵蓋實體，可以使用其服務來處理及儲存 PHI。

目前 HIPAA 或 HITECH 動作規範沒有正式認證。 不過，涵蓋 BAA，這類 Microsoft 服務已經過的 Microsoft ISO/IEC 27001 憑證受到獨立稽核者所進行的稽核。

Microsoft 企業版雲端服務也會涵蓋 FedRAMP 評估。 Microsoft Azure 和 Microsoft Azure Government 操作接收暫時性的授權從 FedRAMP 聯合授權棋盤;Microsoft Dynamics 365 美國政府收到公家機構操作從美國部門的住宅 and 都市開發，像 Microsoft Office 365 US Government 從美國部門的健康狀況與 Human 服務。

了解 HIPAA 和 HITECH 在 Microsoft 雲端的優勢：[讀取 Zwanger — Pesiri Radiology 客戶本文](https://customers.microsoft.com/story/radiology-clinics-ease-compliance-drive-innovation-with-cloud-based-data-network)

## <a name="frequently-asked-questions"></a>常見問題集

**我的組織可以輸入 BAA，與 Microsoft 嗎？**

Microsoft 提供完整的公司或其供應商 BAA，涵蓋範圍內的 Microsoft 服務。

Microsoft 的雲端服務： [HIPAA 業務關聯協議](https://aka.ms/BAA)是透過線上服務條款預設情況下所涵蓋的實體或下 HIPAA 業務夥伴的所有客戶可使用。 此網頁此 BAA 線所涵蓋的雲端服務的清單，請參閱 「 Microsoft 範圍雲端服務 」。

Microsoft 專業服務服務： HIPAA 商務關聯修正是可用的範圍內的 Microsoft 專業服務要求您的 microsoft 服務的代表配合。

**並具有 BAA，與 Microsoft 確保我的組織的合規性 HIPAA 和 HITECH 動作？**

否。 提供 BAA，Microsoft 可以幫助的支援您的 HIPAA 合規性，但使用 Microsoft 服務不會不在其專屬達成目標。 您的組織是負責確保有足夠的合規性程式和內部處理程序的位置，以及您的 Microsoft 服務的特定使用配合 HIPAA 和 HITECH 動作。

**Microsoft 可以修改我的組織 BAA 嗎？**

Microsoft 不能修改 HIPAA BAA，因為 Microsoft 服務都是一致的所有客戶，因此必須遵循相同的程序的每個人。 不過，若要建立 Microsoft 的 HIPAA 管制客戶和其服務 BAA，Microsoft 彙集部分的前置字元的美國醫療學校和其 HIPAA 隱私權顧問，以及其他公用和私人-磁區 HIPAA 涵蓋實體。

**如何取得稽核報告的複本？**

[服務信任入口網站](https://www.microsoft.com/trustcenter/STP/default.aspx)提供獨立稽核規範符合性報告。 您可以使用入口網站來要求稽核報告，以便您稽核員 」 可以比較 Microsoft 的雲端服務結果與您自己的法律和法規需求。

**如何了解遵守 HIPAA 和 HITECH 動作？**

為了協助客戶完成這項作業，Microsoft 已發佈這些指南：

- *HIPAA/HITECH 動作實作指南* [azure](https://aka.ms/azurehipaaguidance)和[Dynamics 365](https://go.microsoft.com/fwlink/?LinkID=257510)和 Office 365。 它們的隱私權、 安全性和法務人員和其他人負責 HIPAA 和 HITECH 動作實作撰寫而成，說明您的組織可以採取維護規範的具體步驟。
- [若要設計安全狀況解決方案使用 Microsoft Azure 的實用指南](https://aka.ms/azureindustrysecurity)可協助您更加了解花費成功採用的雲端服務以安全方式。

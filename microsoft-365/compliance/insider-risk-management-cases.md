### <a name="case-notes"></a>案例的附註

若要新增至案例的附註：

### <a name="contributors"></a>參與者

在這種情況中的 [**參與者**] 索引標籤是其中風險分析師和現場可以新增其他檢閱者的大小寫。 預設所有使用者都指派**測試人員風險管理分析師**，**測試人員風險管理現場**角色列為為每個作用中及已關閉案例的參與者。

必須管理所有測試人員風險管理案例，以適當的存取控制，以維護機密性與調查的完整性。 為維持存取控制的情況下，使用者指派的存取權的兩種類型的其中一個案例：

- **永久存取**： 永久 access 會自動授與給與**測試人員風險管理分析師**和**測試人員風險管理現場**角色的使用者從警示建立這種情況時。 永久 access 會案例的完全控制權授與案例的存留時間，並授與新增其他案例參與者的功能。
- **暫時存取**： 暫時存取只授與給使用者可以永久存取案例的參與者。 一般而言，此存取層級是授與給使用者，必須將備忘稿新增至案例。 使用暫時存取參與者需要以外的所有案例的管理控制：
    - 若要確認或關閉提醒的權限
    - 若要編輯的情況下的參與者權限
    - 若要檢視檔案和郵件內容的檔案總管中的權限

若要將參與者新增至案例：

1. 在[Microsoft 365 合規性中心](https://compliance.microsoft.com)，移至**測試人員風險管理**並選取 [**案例**] 索引標籤。
2. 選取情況下，然後選取 [**參與者**] 索引標籤。
3. 選取 [**新增參與者**。
4. 在 [**新增參與者**] 對話方塊中，開始輸入您想要新增，然後從建議的使用者清單中選取 [使用者的使用者名稱。 此清單是從您的租用戶訂閱的 Azure Active Directory 產生。
5. 在 [**新增參與者**] 對話方塊中，選取參與者的存取層級。 您可以選取**永久**或**暫時**。
6. 選取 [**新增**]，以將使用者新增為參與者或選取 [**取消**關閉對話方塊不需將使用者新增為參與者。

## <a name="case-actions"></a>大小寫的動作

風險分析師和現場可以採取動作的案例，其中一種方法，根據的情況下，該員工的風險和貴組織的風險指導方針的歷程記錄的嚴重性。 在某些情況下，您可能需要擴大員工或資料進行調查，貴組織的其他區域與共同作業，並深入分析風險的活動的案例。 與其他 Microsoft 365 合規性功能，以協助您進行端對端解決方案管理緊密整合測試人員風險管理。

### <a name="send-a-notice"></a>傳送通知

在大多數情況下，建立原則相符項目提醒的員工動作是不慎或意外。 將提醒通知傳送給透過電子郵件的員工是記載案例檢閱和巨集指令，以及方法來提醒公司原則內的員工，或指向複習訓練有效方法。 從測試人員風險管理基礎結構的 [[請注意您建立的範本](insider-risk-management-notices.md)產生通知。

請務必記住，傳送通知給員工***並不會***解析為*已關閉*的案例。 在某些情況下，您可能要讓案例之後將通知傳送給要尋找的額外的風險的活動而不需要開啟新的案例員工保持開啟。 如果您想要解決案例之後傳送通知之外，您必須選取**解決案例**和後續步驟之後傳送通知。

若要將通知傳送給員工指派給案例：

1. 在[Microsoft 365 合規性中心](https://compliance.microsoft.com)，移至**測試人員風險管理**並選取 [**案例**] 索引標籤。
2. 選取 [情況下，然後選取案例動作] 工具列上的 [**傳送電子郵件通知**] 按鈕。
3. 在 [**傳送電子郵件通知**] 對話方塊中，選取 **[選擇通知範本**下拉式清單的控制項選取通知的通知範本。 此選取項目預先填入通知上的其他欄位。
4. 檢閱 [通知] 欄位，並更新為適當。 在這裡輸入的值會覆寫在範本的值。
5. 選取 [將通知傳送給該員工，或選取 [**取消**關閉對話方塊而不傳送通知給該員工的 [**傳送**]。 所有傳送通知會新增至案例的附註佇列**案例的附註**儀表板上。

### <a name="escalate-for-investigation"></a>針對調查向上呈報

向上呈報員工調查的其他合法檢閱其中所需的員工的風險活動的情況下的案例。 此呈報會在 Microsoft 365 組織中開啟新的進階電子文件探索案例。 進階電子文件提供的端對端工作流程，以保留、 收集、 檢閱、 分析和匯出回應您的組織內部和外部法律調查的內容。 它也可讓您管理整個合法持有通知工作流程與 custodians 參與案例進行通訊的法律小組。 指派檢閱者為進階 eDiscovery 中 custodian 從測試人員風險管理案例中建立的情況下可協助法律小組採取適當的動作和管理內容保留。 若要深入了解進階電子文件的情況下，請參閱[Microsoft 365 中的進階概觀 eDsicovery](overview-ediscovery-20.md)。

若要提升員工調查的案例：

1. 在[Microsoft 365 合規性中心](https://compliance.microsoft.com)，移至**測試人員風險管理**並選取 [**案例**] 索引標籤。
2. 選取情況下，然後選取 [**針對調查提報**工具列上的按鈕案例巨集指令。
3. 在 [**針對調查提報**] 對話方塊中，輸入新員工調查的名稱。 如有需要輸入附註這種情況，然後選取 [**提報**。
5. 選取 [ **Confirm**建立員工調查案例或選擇 [**取消**] 以關閉 [] 對話方塊中，不會建立新的員工調查案例]。

已經提報至新的員工調查案例的測試人員風險管理案例之後，您可以檢閱新的案例，在**eDiscovery** > Microsoft 365 合規性中心中的**進階**區域。

### <a name="resolve-the-case"></a>解決這種情況

風險分析師和現場其檢閱和調查完成之後，案例可以解析成目前包含在這種情況中的所有警示都採取動作。 解決案例會新增解決方案分類、 案例狀態變更為 [*已關閉*，並解析動作原因會自動新增至案例的附註佇列**案例的附註**儀表板上。 情況下會解析為一項：

- **Benign**： 的情況下原則符合警示的分類會評估為低風險，非嚴重或 false 正數。
- **已確認原則違規**： 分類為風險、 嚴重的原則相符項目會發出警示評估所在的情況下或惡意意圖的結果。

若要解決案例：

1. 在[Microsoft 365 合規性中心](https://compliance.microsoft.com)，移至**測試人員風險管理**並選取 [**案例**] 索引標籤。
2. 選取情況下，然後選取**解決 case**工具列上的按鈕案例巨集指令。
3. 在**解決案例**] 對話方塊中，選取**解析為**下拉式清單的控制項選取案例的解決方案分類。 選項為**Benign**或**已確認原則違規情形**。
4. 在**解決案例**] 對話方塊中，輸入 for 解析度分類的原因成**採取動作**的 [文字] 欄位。
5. 選取 [**解決**] 以關閉這種情況，或選取 [**取消**關閉對話方塊沒有解決這種情況。

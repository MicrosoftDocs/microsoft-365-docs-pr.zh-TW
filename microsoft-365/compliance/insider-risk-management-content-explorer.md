# <a name="insider-risk-management-content-explorer"></a>有問必答風險管理內容瀏覽器

「內部使用者風險管理」 **內容瀏覽器** 允許指派「 *內幕人員風險管理調查員* 」角色的使用者，檢查內容及與警示中的活動相關聯之內容的詳細資料。 內容瀏覽器中的案例資料會每天重新整理，以包含新的活動。 針對所有已確認為案例的提醒，資料和郵件檔案的副本會封存為專案的快照，同時保留儲存來源中的原始檔案和郵件。 如有需要，可以將資料檔案匯出為可擕式檔檔案 (PDF) 或原始檔案格式。

複製資料和郵件會對與警示相關聯的使用者或內容擁有人透明。 針對新案例，內容瀏覽器中填入內容通常需要一個小時的時間。 針對具有大量內容的情況，可能需要較長的時間來建立快照。 如果內容瀏覽器中仍載入內容，您將會看到顯示完成百分比的進度指標。

在某些情況下，與案例關聯的資料可能無法當作內容瀏覽器中的審閱快照使用。 當案例資料已刪除或移動，或是處理案例資料時出現暫時性錯誤時，就會發生此情況。 如果發生這種情況，請選取 [在警告列中 **查看** 檔案]，以查看檔案名、檔案路徑，以及每個檔案的失敗原因。 如有需要，可以將此資訊匯出至 .csv (逗號分隔值) 檔。

如果內容包含資訊版權管理許可權，則系統會為複製的內容維護這些許可權，而且如果使用者需要開啟及查看檔案，則指派「 *內幕人員風險管理調查* 人員」角色的使用者將需要這些許可權和許可權。 針對管理目的，每個檔案和郵件會在內幕風險管理案例中自動指派唯一的檔案識別碼。 與裝置指示器活動相關聯的檔不會包含在內容瀏覽器中。

>[!Note]
>內容瀏覽器包含與 Microsoft Office 檔案相關的活動。 網站層級的活動，例如刪除 SharePoint 網站時或網站許可權變更時，不會包含在內容瀏覽器中。

## <a name="column-options"></a>欄選項

若要讓風險分析人員和調查人員更容易查看已捕獲的資料和郵件，並在案例中查看內容，則內容瀏覽器會包含數個篩選和排序工具。 針對基本排序， **日期** 和 **檔類別** 欄支援使用內容佇列窗格中的欄標題進行排序。 其他的佇列資料列可以新增至視圖，以在檔案和郵件上提供不同的轉動。

若要新增或移除內容佇列的欄標題，請使用 **Edit columns** 控制項，然後從下列資料行選項中進行選取。 這些欄會對應至內容瀏覽器所支援的常見、電子郵件及檔案屬性條件，本文稍後會列出。

| **欄選項** | **描述** |
|:------------------|:----------------|
| **Author** | Office 檔中的 author 欄位，會在檔案複製時保留。 例如，如果使用者建立檔，並將它的電子郵件給其他人，然後再將其上傳至 SharePoint，該檔仍會保留原來的作者。 |
| **Bcc** | 可用於電子郵件訊息 [Bcc 郵件] 欄位中的使用者。 |
| **Cc** | 可用於電子郵件訊息 [Cc 郵件] 欄位中的使用者。 |
| **複合路徑** | 描述專案來源的人可讀取路徑。 |
| **交談識別碼** | 郵件中的交談識別碼。 |
| **交談索引** | 郵件中的交談索引。 |
| **建立時間** | 檔或電子郵件的建立時間。 |
| **日期 (UTC)** | 電子郵件，是由收件者接收或傳送給寄件者的郵件。 對於檔，是上次修改檔的日期。 日期 (UTC) 的協調通用時間。|
| **主要主題** | 針對分析進行計算的主要主題。 |
| **電子郵件集識別碼** | 相同電子郵件集中所有郵件的群組識別碼。 |
| **系列識別碼** | 系列 Id 群組所有專案;若為電子郵件，此欄會包含郵件和所有附件;對於檔，此欄會包含檔及任何內嵌專案。 |
| **檔類別** | SharePoint 與 OneDrive 的內容：**檔**;Exchange 的內容：**電子郵件** 或 **附件**。 |
| **檔識別碼** | 在案例內唯一的檔識別碼。 |
| **檔案類型圖示** | 檔的副檔名;例如，.docx、one、.pptx 或 .xlsx。 此欄位的屬性與 FileExtension 網站屬性相同。 |
| **ID** | 檔案的 GUID 識別碼。 |
| **不彈性識別碼** | 在 Office 365 中儲存的永恆識別碼。 |
| **包含類型** | 針對分析計算的非獨佔類型： **0** -無包含; **1** -包含; **2** 個包含減去; **3** 個包含的副本。 |
| **上次修改日期** | 上次變更檔的日期。 |
| **標示為代表** | 每一組確切的重複專案的一個檔會標示為代表。 |
| **郵件類型** | 要搜尋的電子郵件類型。 可能的值：連絡人、檔、電子郵件、外部資料、傳真、im、日記、會議、microsoft 小組 (會從 Microsoft Teams 中的交談、會議及通話傳回專案) 、附注、帖子、RSS 摘要、工作、語音信箱 |
| **參與者** | 郵件的所有參與者清單;例如，寄件者、收件者、抄送、Bcc。 |
| **樞紐分析表識別碼** | Pivot 的識別碼。 |
| **收到日期** | 收件者接收到電子郵件的日期。 此欄位的屬性與接收的電子郵件屬性相同。 |
| **Recipients** | 電子郵件訊息中的所有收件者欄位。 這些欄位是 To、Cc 和 Bcc。 |
| **代表識別碼** | 每一組確切重複專案的數值識別碼。 |
| **Sender** | 電子郵件的寄件者。 |
| **寄件者/作者** | 電子郵件，是指傳送訊息的人員。 對於檔，[author] 欄位中從 Office 檔中提及的人員。 您可以輸入一個以上的名稱，以逗號分隔。 OR 運算子會以邏輯方式連接兩個或多個值。 |
| **敏感性資訊類型** | 內容中識別的敏感資訊類型。 |
| **敏感度標籤** | 套用至內容的靈敏度標籤。 |
| **Sent** | 寄件者傳送電子郵件的日期。 此欄位的屬性與已傳送的電子郵件屬性相同。 |
| **大小** | 電子郵件和檔的專案大小 (以位元組) 。 |
| **Subject** | 電子郵件主題行中的文字。 |
| **主旨/職稱** | 電子郵件，郵件的主旨行中的文字。 檔的標題。 如先前所述，Title 屬性是 Microsoft Office 檔中指定的中繼資料。 您可以輸入多個主體/標題的名稱，以逗號分隔。 OR 運算子會以邏輯方式連接兩個或多個值。 |
| **主題清單** | 針對分析所計算的主題清單。 |
| **Title** | 檔的標題。 Title 屬性是 Office 檔中指定的中繼資料。 它與檔的檔案名不同。 |
| **To** | [收件者] 欄位中的電子郵件收件者。 |

## <a name="filtering"></a>篩選

您可以使用一或多個篩選器縮小搜尋範圍，並傳回一組更為細化的結果。 若要設定篩選器，請在內容佇列的頂端選取 [ **篩選器** ]。 許多篩選包括其他篩選選項，以協助縮小篩選所傳回的結果。 例如，[*日期*] 篩選包括設定 **日期** 篩選的 *開始日期* 和 *結束日期* 的控制項。 從下列類別選取一個或多個篩選項目：

### <a name="common-filters"></a>通用篩選

| **Filter** | **描述** |
|:---------------------|:----------------|
| **日期 (UTC)** | 電子郵件，是由收件者接收或傳送給寄件者的郵件。 對於檔，是上次修改檔的日期。 |
| **寄件者/作者** | 電子郵件，是指傳送訊息的人員。 對於檔，[ *Author* ] 欄位中從 Office 檔中提及的人員。 您可以輸入一個以上的名稱，以逗號分隔。 |
| **Source** | 組織中檔的位置。 例如，特定的 SharePoint 網站位置。 |
| **主旨/職稱** | 電子郵件，郵件的主旨行中的文字。 檔的標題。 檔中的 Title 屬性是 Microsoft Office 檔中指定的中繼資料。 您可以輸入多個主體/標題的名稱，以逗號分隔。 OR 運算子會以邏輯方式連接兩個或多個值。 |

### <a name="email-filters"></a>電子郵件篩選器

| **Filter** | **描述** |
|:---------------------|:----------------|
| **Bcc** | 電子郵件訊息的 [密件副本] 欄位。 |
| **Cc** | 電子郵件訊息的 [副本] 欄位。 |
| **具有附件** | 會指出郵件是否有附件。 值會列為 **true** 或 **false**。 |
| **電子郵件附件** | 如果檔是附件，此值會列為 **[是]**。 |
| **內嵌檔** | 如果檔內嵌在電子郵件中，則此值會列為 **[是]**。 |
| **內嵌附件** | 如果檔是電子郵件中的內嵌附件，則此值會列為 **[是]**。 |
| **參與者** | 電子郵件訊息中的所有 [人員] 欄位。 這些欄位是從、To、Cc 和 Bcc。 |
| **收到日期** | 收件者接收到電子郵件的日期。 |
| **收件者網域** | 郵件收件者的所有網域清單。 |
| **Recipients** | 電子郵件訊息收件者。 |
| **Sender** | 寄件者 (來自郵件類型) 欄位。  Format 為 **DisplayName \<SmtpAddress>**。 |
| **寄件者網域** | 寄件者的網域。 |
| **To** | 電子郵件訊息的 [至] 欄位。 |
| **電子郵件集中的唯一** | False 表示電子郵件組中的附件重複。 |

## <a name="document-filters"></a>檔篩選

| **篩選** | **描述** |
|:---------------------|:----------------|
| **符合性標籤** | 套用於 Office 365 中的符合性標籤。 |
| **(UTC) 建立時間** | 建立檔或電子郵件訊息的日期和時間。 日期和時間為協調世界時 (UTC) 。 |
| **上次修改日期 (UTC)** | 上次變更檔的日期。 日期和時間為協調世界時 (UTC) 。 |
| **檔案副檔名** | 檔案的分機類型。 |
| **使用者活動事件** | 案例中與特定使用者活動相關之專案的活動。  例如，當您在案例的 **使用者活動** 頁面中，當您選取「探索內容」的連結時，會使用此篩選顯示與該活動相關的專案。 |
| **工作產品** | 檔的工作產品類型。 例如，檔中的批註或標記。 |

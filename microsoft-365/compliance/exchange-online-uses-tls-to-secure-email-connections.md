<span data-ttu-id="7ef84-108">TLS 之前的傳輸層安全性 (TLS) 和 SSL 是密碼編譯通訊協定，該協定使用安全性憑證來加密電腦之間的連線以保護透過網路進行的通訊。</span><span class="sxs-lookup"><span data-stu-id="7ef84-108">Transport Layer Security (TLS), and SSL that came before TLS, are cryptographic protocols that secure communication over a network by using security certificates to encrypt a connection between computers.</span></span> <span data-ttu-id="7ef84-109">TLS 取代了安全通訊端層 (SSL)，且往往被稱為 SSL 3.1。</span><span class="sxs-lookup"><span data-stu-id="7ef84-109">TLS supersedes Secure Sockets Layer (SSL) and is often referred to as SSL 3.1.</span></span> <span data-ttu-id="7ef84-110">針對 Exchange Online，我們使用 TLS 來加密 Exchange 伺服器與 Exchange 伺服器之間的連線，以及與您的內部部署 Exchange 伺服器或收件者的郵件伺服器等其他伺服器之間的連接。</span><span class="sxs-lookup"><span data-stu-id="7ef84-110">For Exchange Online, we use TLS to encrypt the connections between our Exchange servers and the connections between our Exchange servers and other servers such as your on-premises Exchange servers or your recipients' mail servers.</span></span> <span data-ttu-id="7ef84-111">一旦連線經過加密，所有透過該連線傳送的資料將會透過加密通道傳送。</span><span class="sxs-lookup"><span data-stu-id="7ef84-111">Once the connection is encrypted, all data sent through that connection is sent through the encrypted channel.</span></span> <span data-ttu-id="7ef84-112">不過，若轉寄透過 TLS 加密連線所傳送的郵件，該郵件則不一定會加密。</span><span class="sxs-lookup"><span data-stu-id="7ef84-112">However, if you forward a message that was sent through a TLS-encrypted connection, that message isn't necessarily encrypted.</span></span> <span data-ttu-id="7ef84-113">這是因為在簡單的字詞中，TLS 不會加密郵件，只是連接。</span><span class="sxs-lookup"><span data-stu-id="7ef84-113">This is because, in simple terms, TLS doesn't encrypt the message, just the connection.</span></span>

<span data-ttu-id="7ef84-119">若要為我們的客戶提供一流的加密，Microsoft 已棄用 (TLS) 版本1.0 和1.1 的 [office 365](tls-1.0-and-1.1-deprecation-for-office-365.md) 和 [office 365 GCC](tls-1-2-in-office-365-gcc.md)中的傳輸層安全性。</span><span class="sxs-lookup"><span data-stu-id="7ef84-119">To provide the best-in-class encryption to our customers, Microsoft has deprecated Transport Layer Security (TLS) versions 1.0 and 1.1 in [Office 365](tls-1.0-and-1.1-deprecation-for-office-365.md) and [Office 365 GCC](tls-1-2-in-office-365-gcc.md).</span></span> <span data-ttu-id="7ef84-120">不過，您可以繼續使用沒有任何 TLS 的未加密 SMPT 連接。</span><span class="sxs-lookup"><span data-stu-id="7ef84-120">However, you can continue to use an unencrypted SMPT connection without any TLS.</span></span> <span data-ttu-id="7ef84-121">建議您不要在未加密的情況下進行電子郵件傳輸。</span><span class="sxs-lookup"><span data-stu-id="7ef84-121">We don't recommend email transmission without any encryption.</span></span>

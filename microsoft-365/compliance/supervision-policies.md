# <a name="supervision-policies-in-office-365"></a>Office 365 中的监督策略

Office 365 中的监督策略允许您捕获员工通信，供指定的审阅者检查。 您可以定义特定策略，以捕获组织中的内部和外部电子邮件、Microsoft 团队或第三方通信。 然后，审阅者可以检查邮件，以确保它们符合组织的邮件标准，并采用分类类型解决它们。 

这些政策还可以帮助您克服许多现代合规性挑战，包括：

- 监控不断增加的通信通道类型
- 消息数据量不断增加
- 监管执法&罚款风险

在某些组织中，IT 支持和合规性管理组之间可能存在职责分离。 Office 365 支持将监督策略功能配置与捕获的通信策略配置分开。 例如，组织的 IT 组可能负责设置角色权限和组，以支持由组织的合规性团队配置和管理的监督策略。

有关监督策略的快速概述，请参阅[Microsoft 机械频道](https://www.youtube.com/user/OfficeGarageSeries)[上的"监督策略"视频。](https://youtu.be/C3Y8WZ7o_dI)

要了解有关即将推出的"监督"功能改进和可用性的更多信息，请参阅[Microsoft 365 路线图](https://www.microsoft.com/microsoft-365/roadmap)。

## <a name="scenarios-for-supervision-policies"></a>监管策略的方案

监督策略可帮助在以下几个领域监控组织中的通信：

- **公司政策**

    员工在其所有业务相关通信中必须遵守可接受的使用、道德标准和其他公司政策。 监督策略可以检测策略违规，并帮助您采取纠正措施，以帮助缓解这些类型的事件。 例如，您可以监视组织的潜在人力资源违规行为，例如骚扰或在员工通信中使用不当或冒犯性语言。

- **风险管理**

    组织负责分布在其基础设施和公司网络系统中的所有通信。 使用监管策略来帮助识别和管理潜在的法律风险和风险，有助于在风险损害公司运营之前将风险降至最低。 例如，您可以监视组织是否针对机密项目（如即将进行的收购、合并、收益披露、重组或领导团队变革）进行未经授权的通信。

- **法规遵从性**

    作为正常操作程序的一部分，大多数组织必须遵守某些类型的法规遵从性标准。 这些法规通常要求组织对其行业实施某种适合其行业的消息传送的监督或监督流程。 金融业监管局（FINRA）第3110条是一个很好的例子，它要求各组织制定监督程序，以监测其雇员的活动及其参与的业务类型。 另一个示例可能是需要监控组织中的经纪人交易商，以防止潜在的洗钱、内幕交易、串通或贿赂活动。 监督策略通过提供监控和报告公司通信的流程来帮助您的组织满足这些要求。

## <a name="components"></a>零件

### <a name="supervision-policy"></a>监督政策

您可以在合规中心创建监督策略。 这些策略定义组织中哪些通信和用户需要审核，定义通信必须满足的自定义条件，并指定谁应执行审核。 "监督审核"角色组中的用户可以设置策略，分配了此角色的任何人都可以访问合规中心中的"监督"页面。

### <a name="supervised-users"></a>受监督的用户

在开始使用监督之前，您必须确定谁需要审查他们的通信。 在策略中，用户电子邮件地址标识要监督的个人或组。 这些组的一些示例包括 Office 365 组、基于 Exchange 的通讯组列表和 Microsoft Teams 渠道。 您还可以将特定用户或组排除在受监督组或组列表的监督之外。

> [!IMPORTANT]
> 受监督策略监视的用户必须具有 Microsoft 365 E5 合规性许可证、具有高级合规性加载项的 Office 365 企业版 E3 许可证，或者包含在 Office 365 企业版 E5 订阅中。如果您没有现有的企业 E5 计划，并且想要尝试监督，则可以注册 Office [365 企业版 E5 的试用版。](https://go.microsoft.com/fwlink/p/?LinkID=698279)

### <a name="reviewers"></a>檢閱者

创建监督策略时，必须确定谁将执行对受监督用户的消息的审查。 在策略中，用户电子邮件地址标识要检查受监督通信的个人或组。 所有审阅者都必须在 Exchange 联机上托管邮箱。

### <a name="groups-for-supervised-users-and-reviewers"></a>监督用户和审阅者的组

要简化设置，请为需要审核其通信的人员创建组，并为审阅这些通信的人员创建组。 如果使用组，则可能需要多个组。 例如，如果要监视两个不同的人员组之间的通信，或者要指定不受监督的组。

当您为受监督的用户选择 Office 365 组时，策略将监视共享 Office 365 邮箱和与该组关联的 Microsoft Teams 通道的内容。 选择通讯组列表时，策略将监视各个用户邮箱。

### <a name="supported-communication-types"></a>支持的通信类型

使用监督策略，您可以选择监视以下一个或多个通信平台中的消息：

- **交换电子邮件：** 作为 Office 365 订阅的一部分在 Exchange Online 上托管的邮箱都有资格接受邮件监督。 与监管政策条件相匹配的电子邮件和附件可立即用于监控和监督报告。 支持的用于监督的附件类型与 Exchange[邮件流规则内容检查支持的文件类型](https://docs.microsoft.com/exchange/security-and-compliance/mail-flow-rules/inspect-message-attachments#supported-file-types-for-mail-flow-rule-content-inspection)相同。

- **微软团队：** 可以监督公共和私有 Microsoft Teams 渠道中的聊天通信和关联附件以及个人聊天。 团队聊天匹配的监督政策条件每 24 小时处理一次，然后可用于监控和监督报告。 使用以下组管理配置来监督团队中的单个用户聊天和渠道通信：

    - **对于团队聊天监督：** 为单个用户分配单个用户或将[通讯组](https://support.office.com/article/Distribution-groups-E8BA58A8-FAB2-4AAF-8AA1-2A304052D2DE)分配给监督策略。 这适用于一对一或一对多用户/聊天关系。
    - **对于团队渠道通信：** 将要监视的每个 Microsoft 团队通道或 Office 365 组分配给监督策略，其中包含特定用户。 如果将同一用户添加到其他 Microsoft Teams 频道或 Office 365 组，请确保将这些新通道和组添加到监督策略中。

- **业务在线 Skype：** 可以监督 Skype 中在线 Skype 中的聊天通信和相关附件。 Skype 业务在线聊天匹配监督政策条件每 24 小时处理一次，然后可用于监控和监督报告。 受监督的聊天对话来源于[以前保存在Skype业务在线的对话。](https://support.office.com/article/Find-a-previous-Skype-for-Business-conversation-18892eba-5f18-4281-8c87-fd48bd72e6a2)  使用以下组管理配置来监督 Skype 中在线业务中的用户聊天通信：

    - **对于 Skype 业务在线聊天监督：** 为单个用户分配单个用户或将[通讯组](https://support.office.com/article/Distribution-groups-E8BA58A8-FAB2-4AAF-8AA1-2A304052D2DE)分配给监督策略。 这适用于一对一或一对多用户/聊天关系。

- **第三方来源：** 您可以监督来自第三方来源（如来自 Facebook 或 DropBox）的通信，以用于导入到您组织的 Office 365 邮箱中的数据。 [了解如何将第三方数据导入 Office 365](archiving-third-party-data.md)。

默认情况下，即使用户离开您的组织并删除其邮箱，跨这些平台捕获的通信也会为每个策略保留七年。

### <a name="policy-settings"></a>策略设置

#### <a name="direction"></a>方向

默认情况下，将显示"**方向是"** 条件，无法删除。 策略中的通信方向设置是单独或一起选择的：

- **入站：****您可以选择"入站"** 来查看发送给您**** 选择**从**政策中未包括的人发出的通信。
- **出站：** 如果要查看**从**选择监管的人**发送给**未包含在策略中的人员的通信，则可以**选择"出站"。**
- **内部：****您可以选择"内部"** 来查看您在策略中标识的人**之间**发送的通信。

#### <a name="sensitive-information-types"></a>敏感資訊類型

您可以选择将敏感信息类型作为监督策略的一部分。 敏感信息类型是预定义的或自定义的数据类型，可帮助识别和保护信用卡号、银行帐号、护照号码等。 作为 Office 365[数据丢失防护 （DLP）](data-loss-prevention-policies.md)的一部分，敏感信息配置可以使用模式、字符邻近度、置信度甚至自定义数据类型来帮助识别和标记可能敏感的内容。 默认敏感信息类型为：

- 財務
- 医疗与健康
- 隱私權
- 自定义信息类型

要了解有关敏感信息详细信息和默认类型中包含的模式的详细信息，请参阅[查找的敏感信息类型。](what-the-sensitive-information-types-look-for.md)

#### <a name="custom-keyword-dictionaries"></a>自定义关键字词典

配置自定义关键字字典（或词典），以提供特定于组织或行业的关键字的简单管理。 关键字字典支持每个字典最多 100，000 个术语。 如果需要，可以将多个自定义关键字字典应用于单个策略，或者每个策略具有单个关键字字典。 这些字典在监督策略中分配，可以从文件（如 .csv 或 .txt 列表）或从可以在[合规中心导入](create-a-keyword-dictionary.md)的列表中获取。

#### <a name="offensive-language"></a>攻击性语言

监视组织中发送或接收的电子邮件，以传播攻击性语言。 该模型使用机器学习、人工智能和关键字的组合来识别电子邮件中可能违反反骚扰和欺凌策略的语言。 攻击性语言模型当前支持英语关键字并监视电子邮件正文。

> [!NOTE]
> 如果需要以下条件，使用阻止术语的[自定义关键字字典](create-a-keyword-dictionary.md)创建[数据丢失防护策略：](create-test-tune-dlp-policy.md)
>
> - 监视组织中 Microsoft 团队的通信，以使用攻击性语言
> - 防止或阻止组织中通信中的冒犯性语言

请注意，该模型没有提供详尽的冒犯性语言列表。 此外，语言和文化标准不断变化，并鉴于这些现实，Microsoft 保留自行决定更新模型的权利。 虽然该模型可能帮助您的组织监视攻击性语言，但该模型并不旨在提供组织监视或解决此类语言的唯一方法。 您的组织（而不是 Microsoft）仍负责与监视和阻止攻击性语言相关的所有决策。

攻击性语言模型监控电子邮件中与以下语言类型相关的情绪：

|**類型**|**描述**|
|:-----|:-----|
| **亵渎** | 使大多数人难堪的表达方式。 |
| **辱骂** | 表达对特定群体（如种族、族裔、性取向、残疾）的偏见的表达。 |
| **嘲弄** | 嘲弄、谴责、嘲笑或可能导致愤怒或暴力的表达。 |
| **伪装表达式** | 其含义或发音与另一个更冒犯性的术语相同的表达式。 |

#### <a name="conditional-settings"></a>条件设置
<a name="ConditionalSettings"> </a>

您为该政策选择的条件适用于来自您组织中的电子邮件和第三方来源（如来自 Facebook 或 DropBox）的通信。

下表详细介绍了每个条件。
  
|**條件**|**如何使用此条件**|
|:-----|:-----|
| **从这些域中的任何一个接收消息**  <br><br> **不会从这些域中接收消息** | 应用策略以包括或排除已接收邮件中的特定域或电子邮件地址。 输入每个域或电子邮件地址，用逗号分隔多个域或电子邮件地址。 输入的每个域或电子邮件地址都单独应用，只有一个域或电子邮件地址必须应用于该策略才能应用于邮件。 <br><br> 如果要监视来自特定域的所有电子邮件，但想要排除不需要审阅的邮件（新闻稿、公告等），则必须配置**消息未从这些域中接收的条件，** 该条件不包括电子邮件地址（例如"newsletter@contoso.com"）。 |
| **消息发送到这些域中的任何一个**  <br><br> **消息未发送到这些域中的任何一个** | 应用策略以在已发送邮件中包括或排除特定域或电子邮件地址。 输入每个域或电子邮件地址，用逗号分隔多个域或电子邮件地址。 每个域或电子邮件地址单独应用，只有一个域或电子邮件地址必须应用于该策略才能应用于邮件。 <br><br> 如果要监视发送到特定域的所有电子邮件，但想要排除不需要审阅的已发送邮件，则必须配置两个条件： <br> -**消息发送到**定义域（"contoso.com"）的这些域条件中的任何一个，以及 <br> -**消息不会发送到排除**电子邮件地址（"subscriptions@contoso.com"）的任何这些域条件。 |
| **消息与这些标签中的任何一个分类**  <br><br> **消息未与这些标签分类** | 在邮件中包含或排除某些保留标签时应用策略。 必须单独配置保留标签，并选择配置的标签作为此条件的一部分。 您选择的每个标签都单独应用（只有一个标签必须应用于要应用于邮件的策略）。 有关配置保留标签的详细信息，请参阅[保留标签概述](labels.md)。|
| **消息包含任何这些单词**  <br><br> **消息不包含这些单词** | 要在邮件中包含或排除某些单词或短语时应用策略，请在单独的行上输入每个单词或短语。 您输入的每行单词都单独应用（只有其中一行必须应用于要应用于邮件的策略）。 有关输入单词或短语的详细信息，请参阅下一节[将单词和短语与电子邮件或附件匹配。](supervision-policies.md#Matchwords)|

| **附件是任何这些文件类型**  <br><br> **附件不是这些文件类型** | 要监督包含或排除特定类型的附件的通信，请输入文件扩展名（如 .exe 或 .pdf）。 如果要包含或排除多个文件扩展名，请在单独的行中输入这些扩展名。 只有一个附件扩展程序必须匹配才能应用策略。|
| **消息大小大于**  <br><br> **消息大小不大于** | 要根据特定大小查看邮件，请使用这些条件指定邮件在审阅之前的最大大小或最小大小。 例如，如果**指定"邮件"大小大于**\> **1.0 MB，** 则所有 1.01 MB 及较大的邮件都将受到审阅。 对于这种情况，您可以选择字节、千字节、兆字节或千兆字节。|
| **附件大于**  <br><br> **附件不大于** | 要根据附件的大小查看邮件，请指定附件在邮件及其附件接受审阅之前的最大大小或最小大小。 例如，如果指定**附件大于**\> **2.0 MB，** 则所有附件为 2.01 MB 及以上的邮件都将受到审核。 对于这种情况，您可以选择字节、千字节、兆字节或千兆字节。|
   
##### <a name="matching-words-and-phrases-to-emails-or-attachments"></a>将单词和短语与电子邮件或附件匹配
<a name="Matchwords"></a>您输入的每行单词都单独应用（只有一行必须应用于策略条件才能应用于电子邮件或附件）。 例如，让我们使用**条件，Message 包含这些单词中的任何一**个，关键字"Banker"和"内幕交易"分别行。 本政策适用于包含"银行家"或"内幕交易"一词的任何消息。 要应用此策略条件，只能出现这些单词或短语中的一个。 邮件或附件中的单词必须与您输入的单词完全匹配。

要扫描电子邮件和附件中相同的关键字，请为要监视的术语创建具有[自定义关键字字典](create-a-keyword-dictionary.md)的[数据丢失防护策略。](create-test-tune-dlp-policy.md) 此策略配置标识电子邮件**或**电子邮件附件中显示的已定义的关键字。 使用标准条件策略设置（*邮件包含任何这些单词，**附件包含这些单词中的任何*一个）来标识邮件和附件中的术语，这需要**消息和**附件。
  
##### <a name="enter-multiple-conditions"></a>输入多个条件

如果输入多个条件，Office 365 将使用所有条件一起确定何时将策略应用于通信项目。 设置多个条件时，必须满足所有条件才能应用策略，除非您输入异常。 例如，如果邮件包含单词"trade"且大于 2 MB，则需要应用的策略。 但是，如果邮件中还包含"Contoso 财务批准"字样，则该政策不应适用。 因此，在这种情况下，三个条件如下：
  
- **消息包含任何这些词，** 关键字"交易"

- **消息大小大于**，值为 2 MB

- **消息不包含这些单词，** 关键字为"Contoso 财务团队批准"

#### <a name="review-percentage"></a>审核百分比

如果要减少要审阅的内容量，可以指定受监督策略管辖的所有通信的百分比。 从与所选策略条件匹配的内容总百分比中选择实时随机内容样本。 如果希望审阅者审阅所有项目，则可以在监督策略中输入**100%。**

## <a name="monitor--manage"></a>监视&管理

监视监督策略的结果并应用解决方案标记很容易。 您可以快速查看：

- 已审核项目的状态
- 受监督的用户和组
- 指定为审阅者的用户和组

### <a name="supervision-policy-dashboard"></a>监督策略仪表板

使用监督策略仪表板来管理监督策略结果并解决未完成的项目。 此仪表板允许审阅者查看需要审阅的项目，对项目执行操作，并查看以前审阅和解析每个监督策略的项目的结果。 **您可以在"监督** > *自定义策略* > **打开"** 中的合规中心访问监督策略仪表板。

#### <a name="dashboard-home"></a>仪表板主页

**仪表板主页**有几个部分可帮助您快速执行监督策略。 在这里，您可以：

- 快速查看本周的待处理和已解决的亮点
- 查看所选策略的监督用户和受监督组的列表
- 查看所选策略的审阅者和审阅团队的列表
- 查看哪些通信平台包含政策监管的内容

#### <a name="review-tab"></a>审阅选项卡

"**审阅"** 选项卡是审阅者对所选策略标识的项目进行分类和解析的位置。 在这里，您可以：

- 按挂起、合规、不符合和有问题的项目进行筛选。
- 将单个项目标记为合规、不合规或可疑。 您还可以使用项目录制注释，以帮助澄清所执行的标记操作。
- 批量将多个项目标记为合规、不合规或可疑。 您还可以记录具有多个项的注释，以帮助澄清所执行的标记操作。
- 查看单个项的标记历史记录。 包括谁解决了项目、操作的日期和时间、解析标记以及任何包含的注释。
- 将以前审核过的项目重新分类为合规、不合规或可疑。 您还可以记录具有单个或多个项的注释，以帮助澄清所采取的重新分类操作。

#### <a name="resolved-items-tab"></a>已解决的项目选项卡

在"**已解决项目"** 选项卡中，审阅者可以查看所选策略的所有以前解析的项目。 在这里，您可以：

- 快速查看和排序已解决项目的主题、发件人和日期
- 查看任何选定项目的分类和注释历史记录

## <a name="reports"></a>報表

使用监督报告查看策略和审阅者级别的审核活动。 对于每个策略，您还可以查看有关审核活动的当前状态的实时统计信息。 您可以使用监督报告：
  
- 验证您的策略是否按预期工作。
- 了解需要审核的通信量。
- 了解有多少通信不合规，哪些通信正在通过审核。 此信息可以帮助您决定是微调策略还是更改审阅者的数量。

### <a name="view-the-supervision-report"></a>查看监督报告

1. 使用具有查看监督报告权限的管理员帐户的凭据登录到[合规性中心。](https://compliance.microsoft.com)
2. **转到"报告**\>**仪表板"或"监督"，** 查看监督报告小部件，了解当前监督政策活动的摘要。 ****
3. 选择"**监督"** 微件以打开详细报告页面。

> [!NOTE]
> 如果您**无法访问"报告"** 页，请检查您是否是监督审核角色组的成员，[如"在组织中提供监督"](configure-supervision-policies.md)中所述。 包含在此角色组中，您可以创建和管理监督政策并运行报表。
  
### <a name="how-to-use-the-report"></a>如何使用报表

本报告列出了审查过程中每个阶段的每项政策和通信数量。 使用报表：
  
- 查看所有或特定策略的数据。
- 查看按标记类型、审阅者或消息类型分组的数据。
- 根据活动日期、策略和审阅者活动将数据导出到 CSV 文件。
- 根据活动日期、标记类型、审阅者和消息类型筛选数据。

下面是**标记类型**列中显示的值的细目。
  
|**標記類型**|**它的含义**|
|:-----|:-----|
| **未审核** | 尚未审阅的电子邮件数。 这些电子邮件正在等待 Office 365 监督仪表板中的审核。
| **Compliant** | 审核并标记为合规的电子邮件数。 这些消息仍然需要解决。 |
| **有疑問的** | 已审阅并标记为可疑的电子邮件数量。 充当其他审阅者的标志，帮助检查电子邮件是否需要调查是否符合要求。 这些消息仍然需要解决。 |
| **不符合（活动）** | 审阅者当前正在调查的不符合要求的电子邮件数。 |
| **不符合（已解决）** | 审阅者调查和解决的不符合要求的电子邮件数。 |
| **命中策略** | 来自 Exchange、团队和第三方数据源的消息总数（每日）与监管策略中定义的一个或多个条件匹配 |
| **在普维** | 由监管策略扫描的来自 Exchange、团队和第三方数据源的邮件总数（每日） |
| **Resolved** | 来自 Exchange、团队和第三方数据源的邮件总数分类**为"已解决"**|

> [!NOTE]
> 监督策略在出现在报告中之前必须预配。 如果删除了策略，则仍显示历史数据。 但是，它们被指示为"不存在的策略"，并且**导出**函数不可用。

## <a name="audit"></a>审计

在某些情况下，您必须向监管或合规审核员提供信息，以证明对员工活动和沟通的监督。 这可能是与既定政策相关的所有监督活动的摘要，或者任何监督政策更改时。 监督策略具有内置的审核跟踪，可完全准备好进行内部或外部审计。 监督政策监控的每个行动的详细审计记录都提供了监督程序的证明。

以下监督策略活动经过审核，可在统一的 Office 365 审核日志中提供：

|**活動**|**关联命令**|
|:-----|:-----|
| **创建策略** | [新监督审查政策V2](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance/new-supervisoryreviewpolicyv2) <br> [新监督审查规则](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance/new-supervisoryreviewrule) |
| **编辑策略** | [设置-监督审查策略V2](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance/set-supervisoryreviewpolicyv2) <br> [设置-监督审查规则](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance/set-supervisoryreviewrule) |
| **刪除原則** | [删除-监督审查策略V2](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance/remove-supervisoryreviewpolicyv2) |

在统一审核日志中或使用[搜索统一审核日志](https://docs.microsoft.com/powershell/module/exchange/policy-and-compliance-audit/search-unifiedauditlog)PowerShell cmdlet 查看审核活动。

例如，以下示例返回所有监督审查活动（政策和规则）的活动，并列出每个活动的详细信息：

```
Search-UnifiedAuditLog -StartDate 3/1/2019 -EndDate ([System.DateTime]::Now) -RecordType DataGovernance -ResultSize 5000 | Where-Object {$_.Operations -like "*SupervisoryReview*"}  | fl CreationDate,Operations,UserIds,AuditData
```

除了在监督报告和日志中提供的信息外，您还可以使用[Get-监督审查活动](https://docs.microsoft.com/powershell/module/exchange/reporting/get-supervisoryreviewactivity?view=exchange-ps)PowerShell cmdlet 返回所有监管政策活动的完整详细列表。

# <a name="export-results-in-advanced-ediscovery-classic"></a>在高級 eDiscovery （古典）中匯出結果

> [!NOTE]
> 進階電子文件探索需要具有進階合規性附加元件的 Office 365 E3，或適用於您組織的 E5 訂閱。如果您沒有該方案，且想要嘗試進階電子文件探索，您可以[註冊 Office 365 企業版 E5 試用版](https://go.microsoft.com/fwlink/p/?LinkID=698279)。 
  
本主題說明高級 eDiscovery 匯出設定選項。
  
 **本主題內容：**
  
- [定義匯出批次和會話](export-results-in-advanced-ediscovery.md#BK_Define)
    
- [增量和其他匯出](export-results-in-advanced-ediscovery.md#BK_IncrementalReports)
    
- [設定批次匯出參數](export-results-in-advanced-ediscovery.md#BK_SetUpExport)
    
- [匯出報表輸出檔案](export-results-in-advanced-ediscovery.md#BK_ExportOutputFIles)
    
## <a name="defining-export-batches-and-sessions"></a>定義匯出批次和會話
<a name="BK_Define"> </a>

匯出批次允許使用一組定義的參數進行匯出處理。 「高級 eDiscovery」可讓您定義批次，以自訂每個匯出。
  
每個匯出批次定義參數。 預設會為案例的第一個批次建立名為 "Export batch 1" 的批次。 您也可以編輯批次名稱和描述。
  
匯出會話是在匯出批次中執行高級 eDiscovery 匯出。
  
## <a name="incremental-and-additional-exports"></a>增量和其他匯出
<a name="BK_IncrementalReports"> </a>

您可以在匯出批次中執行多個匯出會話，以根據相同的匯出範本和參數，確保結果一致。 針對批次中的每個會話，您可以針對每個「增量」，匯出新處理案例資料的分析和處理。
  
若要使用不同的參數組匯出，您必須先建立新的批次。 新批次中的第一個會話會產生目前為止處理的檔案的結果，不論這些檔案是透過一或多個匯入，還是進行處理。 每個批次重新計算透視、相似性、inclusives 等。會話使用為批次定義的參數，而不重新計算透視、相似性、inclusives 等每個會話執行的參數。
  
例如，假設已匯入案例，並分析其資料。 若要針對累加的資料取得近乎重複的電子郵件串接結果，請按一下 [在先前用於匯出資料的相同批次中**建立匯出會話**]。 
  
## <a name="set-up-batch-export-parameters"></a>設定批次匯出參數
<a name="BK_SetUpExport"> </a>

EDiscovery 匯出工具是用來將搜尋結果從 Advanced eDiscovery 匯出至您的本機電腦。 若要增加資料傳輸輸送量並加速匯出程式，您可以在用來匯出搜尋結果的電腦上設定 Windows 登錄設定。 如果您想要增加下載速度，請先設定登錄設定，再設定匯出參數。 如需詳細資訊，請參閱[在從 Office 365 匯出 eDiscovery 搜尋結果時，提高下載速度](increase-download-speeds-when-exporting-ediscovery-results.md)。
  
1. 在 [Advanced eDiscovery] 中，選取案例，然後按一下 [**匯出** \> **設定**]。
    
    - 從 [**匯出批次**] 清單中，選取批次名稱或匯出結果以匯出批次01（預設批次）。 
    
    - 若要匯出您新增至現有案例的新檔案的結果，請繼續使用目前的批次。 若要在批次中建立會話，請選取相同的批次號碼，然後按一下 [**建立匯出會話**] 您可以使用此選項，以增量方式匯出與上一個批次相同的參數。 
    
    - 若要匯出至新的批次**Add** ![，請按一下](../media/c2dd8b3a-5a22-412c-a7fa-143f5b2b5612.png)[add add icon]，並以**批次名稱**輸入新名稱（或接受預設值），並在 [**批次描述**] 中輸入描述。 按一下 [確定]****。
    
    - 若要編輯批次名稱或描述，請選取 [**匯出批次**] 中的名稱](../media/3d613660-7602-4df2-bdb9-14e9ca2f9cf2.png)，然後按一下 [**編輯** ![編輯圖示]，然後修改欄位。
    
      > [!NOTE]
      > 在您執行匯出批次的會話後，便無法將其刪除。 此外，在執行第一個會話後，只可編輯部分參數。 
  
    - 若要建立重複的匯出批次，請選擇 [**重複匯出批次** ![建立](../media/3f6d5f59-e842-4946-a493-473528af0119.jpg)重複的匯出批次圖示]，然後在 [面板] 中輸入重複批次的名稱和描述。 
    
    - 若要刪除匯出批次，請選擇 [**刪除** ![] [](../media/92a9f8e0-d469-48da-addb-69365e7ffb6f.jpg)刪除匯出批次圖示]。
    
    - 若要查看批次的記錄，請選擇 [**批次記錄** ![] view 病史 history 圖示](../media/a80cc320-d96c-4d91-8884-75fe2cb147e2.jpg)。
    
2. 在 [**人口**] 底下，選取 [**僅包含相關性切出分數**和/或**調整輸出批次**以上的檔案] （如果您要微調匯出批次的設定）。 
    
3. 如果您選取 [**只包含相關性的僅限**]，則會啟用**問題**。 如果檔案的相關性分數高於所選取問題的切削分數，將會匯出檔案，除非它是由「做為審閱」篩選排除。 
  
    如果您選取 [**調整匯出批次**]，就會啟用 [**取消重復資料**刪除] 和 [依審閱篩選] 欄位選項按鈕。 如果您選擇 [**重復資料**刪除]，則會根據定義的原則來篩選掉重複的檔案（預設值）：從整個案例中的每一組重複檔案，除了一個檔案之外，還會取消 duped 所有檔。 保管人層級：從同一保管人的每一組重複的檔案，除了一個檔案之外，還會解除 duped。匯出輸出包含所有重複檔案的記錄。 如果您選擇 [**依審閱篩選**] 欄位，請選取 [**中繼資料**] 底下的 [修改]，以輸入您的「做**為審閱**」欄位設定。 選取 [**包含輸入檔**]，以在套件內容中包含來源檔案。 您可以清除此設定，以加速匯出程式。 請注意，在任何情況下都會匯出原生檔案。 
    
4. 在 [**中繼資料**] 底下，選取 [**匯出範本**] 清單中的下列選項（每次會話一次）。 
    
    - **Standard**：一組基本的資料項目目、中繼資料和屬性。 當匯入資料已在高級 eDiscovery 中處理，並將匯出資料上傳至已包含檔案的系統時，請使用此選項。 [匯出範本] 欄預設會建立並填滿。
    
    - **All**：完整的標準元資料集（包括所有的處理資料），以及分析與相關性分數。 當 Advanced eDiscovery 執行第一次上傳至外部系統的處理及檔資料時，此範本是必要的。
    
    - **問題**：請選取**所有問題**，或選取您建立的特定問題。 
    
5. 在 [**目的地**：] 下
    
    - **下載到本機電腦**
    
    - **匯出至使用者定義的 Azure blob**：若已勾選此專案，您可以指定容器 URL 和 SAS 權杖。
    
      > [!NOTE]
      > 匯出套件儲存至使用者定義的 Azure blob 之後，就不會再透過「高級 eDiscovery」管理該資料。它是由 Azure blob 所管理。 這表示如果您刪除此案例，匯出的檔案仍會保留在 Azure blob 上。 
  
    - **儲存 sas token 以供未來匯出會話**：如果檢查，則會在高級 eDiscovery 的內部資料庫中加密 sas 權杖，以供日後使用。
    
      > [!NOTE]
      > 目前的 SAS 權杖會在每月後到期。 如果您嘗試在數個月後下載，您必須撤銷上一個會話，然後再匯出一次。 
  
6. 按一下 [**修改**] 以設定「審閱」欄位設定。 
    
    ![設定審閱匯出批次的欄位設定](../media/39451aba-f6fe-4a01-8ed0-0be6a6ce889a.png)
  
   - 在 [**審閱欄位設定**] 底下的 [**選取案例**] 下拉式清單中，選取案例及檢查範圍。 設定會根據您的選取範圍來顯示。
    
      - **全部檢查**（預設值）：預設會選取所有電子郵件、附件及檔。 
    
      - **檢查集合中的所有唯一內容**： Inclusives 及唯一包含副本，電子郵件集合層級中的唯一附件，包含每一組完全相同的專案。
    
      - **檢查集合中所有的唯一內容**： Inclusives，電子郵件集合階層中的唯一附件，包含每一組完全相同的副本。
    
      - **檢查所有的唯一內容和相關的系列**檔案： Inclusives、電子郵件集合層級中的唯一附件、每一組完全重複的代表，展開以包含 [家人] 檔案。
    
      - **自訂**（可讓您定義對話方塊中的選項）：預設值是保留目前的選取範圍並啟用所有的對話方塊選項，以允許其選取範圍。 如果您選取此選項，則可以自訂電子郵件、檔、附件及其他的設定。
    
    - 在 [**電子郵件**] 底下，選取您要匯出的電子郵件。
    
      - **所有電子郵件**：（預設值）會選取所有電子郵件。
    
      - **Inclusives**：包含電子郵件是指執行緒的最後一封電子郵件，其中包含來自該執行緒的所有其他電子郵件。
    
      - **Inclusives 和唯一包含副本**：包含相同主體、內文和附件的副本和 Inclusives;唯一包含的副本是這些電子郵件的獨特副本。
    
    - 在 [**檔**] 底下，選取您要匯出的檔。 
    
      - **所有檔**：（預設值）已選取所有檔。
    
      - 「**旋轉**：」會選擇為代表臨近重複設定的檔案，通常會在審閱集時用作基準。
    
      - **每一組完全相同的代表**：唯一接近重複的檔案（包括 pivot）。
    
    - 在 [**附件**] 底下，選取您要匯出的附件。 
    
      - **所有附件**：（預設值）會選取所有附件。
    
      - **案例層級的唯一附件**：指定之案例內的唯一附件檔案。
    
      - **電子郵件集層級中的唯一附件**：指定的電子郵件案例內的唯一附件檔案。
    
   - 在 [**Micellaneous**] 底下，您可以選擇將**附件當做檔對待**、將**電子郵件視為檔**，或**展開以包含系列**檔案。 當您選擇 [**展開] 以包含系列**檔案時，會針對每個已標記為要檢查的檔案，也會標記同一系列的所有檔案。
    
7. 選擇 [**儲存**] 以儲存設定。 
    
8. 指定匯出參數之後，請按一下 [建立匯出的**會話**] 以開始匯出批次。
    
    在匯出期間，狀態會顯示在 [**任務狀態**] 中。 結果會顯示在 [**匯出摘要**] 中。
    
9. 在 [**下載**檔案] 視窗中，按一下 [**複製到剪貼簿**] 以複製匯出機碼。 
    
    ![下載檔案](../media/99cf2c13-4954-479f-9741-80d7458c1a15.png)
  
10. 按一下 [關閉]****。 
    
    電子檔探索匯出工具已啟動。
    
    ![eDiscovery 匯出工具](../media/705756ca-ee97-4d24-b70f-8b23513f6d11.gif)
  
11. 在 [ **EDiscovery 匯出工具**] 中：
    
    -  在 [**貼上用來連線至來源的共用存取簽名]** 中，將 youcopied 至剪貼簿的匯出金鑰貼到步驟7中。
    
    - 按一下 **[流覽]** ，以選取在本機電腦上儲存已下載之匯出檔案的目標位置。 
    
    - 按一下 [**開始**]。匯出檔案會下載到本機電腦。 如果您在步驟4中選擇 [**匯出至使用者定義的 Azure blob** ]，則會話會匯出至您所選擇的 BLOB 儲存 URL 目的地。
    
如需匯出報告中欄位的完整描述，請參閱[匯出報告欄位](export-report-fields-in-advanced-ediscovery.md)。
  
## <a name="export-report-output-files"></a>匯出報表輸出檔案
<a name="BK_ExportOutputFIles"> </a>

下表列出當您執行匯出批次時所產生的輸出檔案。
  
|**檔案名稱**|**檔案類型**|**描述**|
|:-----|:-----|:-----|
|匯出摘要  <br/> |Csv  <br/> |電子檔探索匯出工具所產生的記錄檔。  <br/> |
|跟蹤  <br/> |Txt  <br/> |電子檔探索匯出工具所產生的記錄檔。  <br/> |
|解壓縮的文字檔  <br/> |檔資料夾  <br/> |包含匯出檔案的解壓縮文字檔的資料夾。  <br/> |
|輸入或本機檔  <br/> |檔資料夾  <br/> |包含匯出檔案的原生和輸入檔案的資料夾。  <br/> |
|匯出清單  <br/> |.xlsx  <br/> |以 .xlsx 格式匯出的檔中繼資料。 檔案中的欄位是根據範本使用者選取匯出。 如有需要，會建立數個檔案，每個檔案會包含 100-150K 列。 如果特定值包含的字元數超過 Excel 儲存格可以包含的字元數（目前的限制為32767個字元），則會將此值修整為允許的最大長度。 如果值已被裁去，該儲存格的背景色彩是紅色，表示這是使用者。電子郵件參與者 "是一種可能超出長度限制的欄位範例（如果電子郵件傳送到大型分配）。 如需輸出欄位的詳細資訊，請參閱[匯出報告欄位](export-report-fields-in-advanced-ediscovery.md)。  <br/> |
|載入檔  <br/> |Csv  <br/> |匯出的檔中繼資料（csv 格式），以用於載入至不同的應用程式。 檔案中的欄位是根據範本使用者選取匯出。  <br/> |
|成功指示器  <br/> |Txt  <br/> |僅在匯出至協力廠商 Azure blob 時建立。 如果匯出完全成功，將會建立檔案。 當失敗或部分成功時，將不會建立檔案。 將會在根資料夾中建立檔案，允許在不同的匯出批次/會話狀態上進行自動追蹤。 這是空白的檔案。 其名稱為： TenantId_CaseId_ExternalCaseId_CaseName_ExportBatchId_SessionId_DateTime .txt。  <br/> |

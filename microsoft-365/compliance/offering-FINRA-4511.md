## <a name="about-finra-rule-4511"></a>關於 FINRA 規則 4511

[財務產業法規機構 (FINRA)](https://www.finra.org/#/)是最大獨立主體規範與在美國境內的多個 4500 brokerage 公司負責監督的證券公司。 由美國國會授權 」，以保護美國的投資者確定代理人經銷商產業運作相當且誠實。 」

2011 年中的美國安全性和 Exchange 委員會 （秒） 核准的書籍和電子儲存媒體上的記錄保留的秒規則 FINRA 採用。 [FINRA 規則 4511(c)](https://www.finra.org/sites/default/files/NoticeDocument/p123548.pdf)指定的 「 所有書籍和要依照 FINRA 規則進行所需記錄應被都保留在格式和遵守 SEA （證券 Exchange 法案） 規則 17a-4 的媒體。 」

此外，FINRA 規則 4511(c) 需要保留一段至少六個年適用 FINRA 或 SEA 規則] 下沒有指定的保留期限是這些書籍和記錄的公司。 有效率地，如果書籍和記錄適用於帳戶，在保留期間強制規定是設為下列帳戶關閉的六個年。 否則，保留期間是之後這類書籍的六個年，而建立記錄]。

## <a name="microsoft-and-finra-rule-4511c"></a>Microsoft 和 FINRA 規則 4511(c)

財務服務的客戶，表示下列其中一個領域中受到最嚴厲管制產業受限於複雜的佈建像保留金融交易和相關的通訊中的非可清除與非可修改的狀態。 它們之間是規則 4511 的財務產業法規機構 (FINRA) 的規定管制接管工作，保留書籍和記錄電子儲存媒體上的實體的嚴格要求。 儲存的記錄必須竄改與沒有變更或刪除它們之後指定的保留期間內的能力。

Microsoft Azure 不變的 Blob 儲存與原則鎖定和 Microsoft Office 365 with 保留鎖定有助於金融機構符合 FINRA 規則 4511(c) 不變的儲存需求。

## <a name="microsoft-azure"></a>Microsoft Azure

若要評估與 FINRA 規則 4511(c) Azure 合規性，Microsoft 保留獨立評估公司專門進行記錄管理及資訊治理規範，Cohasset 會產生關聯。 產生的報告，[秒 17a-4(f) & CFTC 1.31 (c-d) 合規性評估： Microsoft Azure 儲存體](https://servicetrust.microsoft.com/ViewPage/MSComplianceGuide?command=Download&downloadType=Document&downloadId=19b08fd4-d276-43e8-9461-715981d0ea20&docTab=4ce99610-c9c0-11e7-8c2c-f908a777fa4d_GRC_Assessment_Reports)，圍繞 Azure 的合規性 FINRA 規則 4511(c)，延後 SEC Rule 17a-4(f) 格式和媒體需求。

Cohasset 驗證[Azure 不變的 Blob 儲存體](https://docs.microsoft.com/azure/storage/blobs/storage-blob-immutable-storage)使用原則鎖定選項時，用來保留時間為基礎的 Blob，以非可清除與非可讀寫的 （蠕蟲） 格式時, 符合相關 FINRA 儲存需求。 每個 Blob （記錄） 受到防止修改、 覆寫，或刪除之前所需的保留期間已經過期和已釋放任何關聯的合法持有。

軟體提供者和機密的工作負載的協力廠商可以立即自信地仰賴 Azure 不變的 Blob 儲存體作為記錄保留與固定儲存一次商店雲端解決方案。 金融機構現在可以建置自己運用這些功能時剩餘相容的應用程式。

## <a name="microsoft-office-365"></a>Microsoft Office 365

若要評估 Office 365 的合規性 FINRA 規則 4511(c)，Microsoft 保留專門法規問題，Covington & 前置獨立的法律事務所可能 Burling、 LLP。 在所產生的報告，請在 Microsoft Office 365 中，資料保留封存和規則 17a-4 規範、 Covington 驗證[Office 365 with 保留鎖定](https://docs.microsoft.com/office365/securitycompliance/retention-policies#locking-a-retention-policy)包含封存功能，可讓受管制的客戶，包括代理人協會，來儲存資料的方式，可協助他們符合記錄保留 FINRA 需求。

在 Office 365 中封存可協助保留廣泛的資料，包括電子郵件、 語音信箱、 共用文件、 立即訊息和協力廠商資料。 特別是，在 Office 365 中封存可讓客戶若要設定全域或細微通訊保留原則，可將資料儲存在已定義的期間內和 beyond 非可重複燒錄、 非可清除的格式。

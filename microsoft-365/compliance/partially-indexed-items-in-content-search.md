# <a name="partially-indexed-items-in-content-search"></a>內容搜尋中已局部編製索引的項目

您從安全性 & 合規性中心執行的內容搜尋，會在您執行搜尋時，自動在預估搜尋結果中包含一部分索引項目目。 已部分編制索引的專案是 Exchange 信箱專案和檔，其上的 SharePoint 和 OneDrive 出於某些原因尚未完全編制搜尋索引的商務網站。 在 Exchange 中，部分編制索引的專案通常會包含檔案類型的檔案，但該檔案是無法建立索引的，其附加至電子郵件訊息。 以下是一些其他一些原因：專案無法編制索引以進行搜尋，而且在您執行搜尋時會傳回成部分索引項目目： 
  
- 檔案類型無法辨識或不支援索引編製功能。
    
-  郵件具有附加的檔案，但沒有有效的處理常式，例如映射檔;這是部分索引電子郵件專案最常見的原因。 
    
- 檔案類型支援索引編製，但在編製特定檔案的索引時發生錯誤。
    
- 附加至電子郵件的檔案太多。
    
- 附加到電子郵件的檔案太大。
    
- 檔案已使用非 Microsoft 技術加密。
    
- 檔案受密碼保護。
    
> [!NOTE]
> 大多陣列織的內容少於1% 的內容，而且不會以部分索引的大小小於12%。 磁片區和大小之間差異的原因是，較大的檔案包含無法完全編制索引的內容的可能性較高。 
  
為了進行法律調查，您的組織可能需要查看部分編制索引的專案。 您也可以在將搜尋結果匯出至本機電腦時，或在使用 Advanced eDiscovery 準備分析結果時，指定是否要包含部分索引的專案。 如需詳細資訊，請參閱[在 eDiscovery 中調查已部分索引的專案](investigating-partially-indexed-items-in-ediscovery.md)。
  
## <a name="file-types-not-indexed-for-search"></a>未建立搜尋索引的檔案類型

某些類型的檔案 (例如點陣圖或 MP3 檔案) 並不包含可編製索引的內容。 因此，Exchange 和 SharePoint 中的搜尋索引伺服器並不會對這些檔案類型執行全文檢索。 這些類型的檔案被視為不支援的檔案類型。 也有些檔案類型的全文檢索編製已依預設停用，或是被系統管理員停用。 不支援和已停用的檔案類型在內容搜尋中標示為未編制索引的專案。 如先前所述，當您執行搜尋、將搜尋結果匯出至本機電腦，或準備高級 eDiscovery 的搜尋結果時，可將部分索引項目目包含在搜尋結果集中。 
  
如需支援和已停用的檔案格式清單，請參閱下列主題：
  
- **Exchange**  - [Exchange 搜尋編制索引的檔案格式](https://go.microsoft.com/fwlink/p/?LinkID=386618)
    
- **Exchange**  - [Get-SearchDocumentFormat](https://go.microsoft.com/fwlink/p/?LinkID=724037)
    
- **SharePoint**  - [SharePoint 中的預設編目副檔名及分析檔案類型](https://go.microsoft.com/fwlink/p/?LinkID=404033)
    

  
## <a name="messages-and-documents-with-partially-indexed-file-types-can-be-returned-in-search-results"></a>可在搜尋結果中傳回具有部分索引檔案類型的郵件和檔

並非每封電子郵件都具有部分索引的檔案附件或每個已部分索引的 SharePoint 檔，會自動以部分索引項目目的形式傳回。 這是因為其他郵件或檔案屬性（例如電子郵件中的**Subject**屬性和檔的**Title**或**Author**屬性）已編制索引，可供搜尋。 例如，關鍵字搜尋「財務」會傳回具有部分索引檔案附件的專案，如果該關鍵字出現在電子郵件的主旨或檔案名或檔標題中。 不過，如果關鍵字只會出現在檔案的本文中，則郵件或檔會傳回成部分索引項目目。 
  
同樣地，當其他的郵件或檔案屬性（已編制索引及可搜尋）符合搜尋準則時，搜尋結果中也會包含已部分編制索引之檔案類型的郵件。 會編入搜尋索引中的郵件內容，包括傳送和接收日期、寄件者和收件者、附件的檔案名稱，以及郵件內文中的文字。 為搜尋編制索引的檔案屬性包括已建立及已修改的日期。 因此，即使郵件附件可能是部分編制索引的專案，如果其他郵件或檔案屬性的值符合搜尋準則，郵件就會包含在一般搜尋結果中。
  
如需在安全性 & 規範中心內使用搜尋功能來搜尋的電子郵件和檔案屬性清單，請參閱[內容搜尋的關鍵字查詢和搜尋條件](keyword-queries-and-search-conditions.md)。
  
## <a name="partially-indexed-items-included-in-the-search-results"></a>包含在搜尋結果中的部分索引項目目

您的組織可能需要識別並對部分索引的專案進行其他分析，以判斷其所在的專案、包含的內容，以及是否與特定調查相關。 如先前所述，搜尋的內容位置中的部分索引項目目會自動包含在預估的搜尋結果中。 您可以選擇在匯出搜尋結果時包含這些已部分索引的專案，也可以選擇準備高級 eDiscovery 的搜尋結果。
  
請記住下列部分編制索引的專案：
  
- 當您執行內容搜尋時，會在詳細資料窗格中的 [搜尋統計資料] 中顯示「部分索引 Exchange 專案（由搜尋查詢傳回）的總數和大小，並標示為**索引項目目**。 請注意，在詳細資料窗格中顯示之部分索引項目目的統計資料，在 SharePoint 或 OneDrive 中不會包含部分索引的專案。
    
- 如果您要從中匯出結果的搜尋是針對您組織中的特定內容位置或所有內容位置進行搜尋，則只會匯出包含符合搜尋準則之專案的內容位置中未編制索引的專案。 In other words, if no search results are found in a mailbox or site, then any unindexed items in that mailbox or site won't be exported. 其原因是匯出組織中大量位置的部分索引項目目時，可能會增加匯出錯誤的可能性，並增加匯出及下載搜尋結果所需的時間。
    
    若要從搜尋的所有內容位置匯出已部分索引的專案，請將搜尋設定為傳回所有專案（從搜尋查詢移除任何關鍵字），然後在匯出搜尋結果時（只要按一下 [**輸出選項**] 下的 [**僅限未辨識的專案，則會加密，或是未針對其他原因編制索引**]）匯出部分索引的專案。
    
- 如果您選擇包含搜尋結果中的所有信箱專案，或者搜尋查詢未指定任何關鍵字或只指定日期範圍，則部分索引的專案可能不會複製到包含部分索引項目目的 PST 檔案中。 這是因為所有專案（包括所有已編制索引的專案）都會自動包含在一般搜尋結果中。
    
- 不能預覽已部分索引的專案。 您必須匯出搜尋結果，以查看搜尋傳回的部分索引項目目。

此外，當您匯出搜尋結果並在匯出中包含部分索引項目目時，SharePoint 專案中的部分索引項目目會匯出至名為**Uncrawlable**的資料夾中。 當您匯出部分索引的 Exchange 專案時，這些專案會根據是否符合搜尋查詢的部分索引項目目和匯出設定的設定，以不同方式匯出。 

下表顯示已編制索引及部分索引項目目的匯出行為，以及是否每個專案都包含在不同的匯出設定設定中。

|**匯出設定**|**符合搜尋查詢的索引項目目**|**符合搜尋查詢的部分索引項目目**|**不符合搜尋查詢的部分索引項目目**|
|:-----|:-----|:-----|:-----|
|只匯出已編制索引的專案  <br/> |出口<br/> |匯出（包含匯出的索引項目目）<br/>  |未匯出 <br/>|
|只匯出部分索引項目目  <br/> |未匯出  <br/> |匯出（做為部分索引的專案）<br/> |匯出（做為部分索引的專案）|
|匯出已編制索引及部分索引的專案  <br/> |出口<br/> |匯出（包含匯出的索引項目目）<br/>  |匯出（做為部分索引的專案）<br/>|

## <a name="partially-indexed-items-excluded-from-the-search-results"></a>從搜尋結果中排除的部分索引項目目

如果專案已部分編制索引，但不符合搜尋查詢準則，則搜尋結果中不會包含在內容中的部分索引項目目。 換句話說，該專案會從搜尋結果中排除。 例如，假設您執行搜尋，但不包含任何關鍵字或屬性，因為您想要包含所有內容。 不過，您必須包含查詢的日期範圍條件。 若部分編制索引的專案在該日期範圍之外，則不會將其包含為部分編制索引的專案。 日期範圍是從搜尋結果中排除部分索引項目目的有效方式。
  
同樣地，如果您選擇在匯出搜尋結果時包含部分索引的專案，則不會匯出從搜尋結果中排除的部分已編制索引的專案。
  
這項規則的一個例外是當您建立與 eDiscovery 案例相關聯的查詢型保留。 如果您建立以查詢為基礎的 eDiscovery 保留，則會保留所有部分索引項目目。 這包括部分編制索引的專案，此專案不符合搜尋查詢準則及可能超出日期範圍條件的部分索引項目目。 如需建立以查詢為基礎的 eDiscovery 保留的詳細資訊，請參閱[建立 eDiscovery 保留](create-ediscovery-holds.md)。
  
## <a name="indexing-limits-for-messages-in-content-search"></a>內容搜尋中郵件的索引限制

下表說明可導致電子郵件在 Office 365 的內容搜尋中傳回成部分索引項目目的索引限制。
  
如需 SharePoint 檔的索引限制清單，請參閱[線上 SharePoint 的搜尋限制](https://docs.microsoft.com/sharepoint/search-limits)。
  
|**索引限制**|**最大值**|**描述**|
|:-----|:-----|:-----|
|附件大小上限（不含 Excel 檔案）  <br/> |150 MB  <br/> |會分析索引的電子郵件附件大小上限。 不會分析大於此限制的任何附件，以進行索引編制，而且會將具有附件的郵件標示為已部分編制索引。  <br/><br/> **附注：** 剖析是一種處理常式，其中索引服務會從附件中解壓縮文字、移除不必要的字元（如標點符號和空格），然後再將文字分割成文字（在稱為「標記程式」的程式中），然後儲存在索引中。           |
|Excel 檔案大小上限  <br/> |4 MB  <br/> |位於網站上的 Excel 檔案大小上限，或附加至將會針對索引進行分析的電子郵件訊息。 不會分析大於此限制的任何 Excel 檔案，而且檔案或電子郵件將會標示為未索引的附件。  <br/> |
|附件數目上限  <br/> |250  <br/> |要針對索引進行分析的電子郵件所附加的檔案數目上限。 如果郵件包含超過250的附件，就會分析和編制第一個250附件，並將該郵件標示為部分索引，因為它有其他尚未剖析的附件。  <br/> |
|附件深度上限  <br/> |大約  <br/> |所解析之嵌套附件的數目上限。 例如，如果電子郵件訊息有另一封郵件附加到它，而且附加的郵件有附加的 Word 檔，就會將該 Word 檔及附加的郵件編制索引。 這種行為會繼續最多30個嵌套的附件。  <br/> |
|附加的影像數目上限  <br/> |0  <br/> |分析器會略過附加到電子郵件訊息的影像，而且不會編制索引。  <br/> |
|分析專案所花費的時間上限  <br/> |30 秒  <br/> |分析專案以進行索引時，最多可花30秒的時間。 如果剖析時間超過30秒，該專案就會標示為已部分編制索引。  <br/> |
|分析器輸出上限  <br/> |200 萬個字元  <br/> |已編制索引之分析程式的文字輸出數目上限。 例如，如果剖析器解壓縮了檔中的8000000個字元，只會為前2百萬個字元編制索引。  <br/> |
|註解標記上限  <br/> |2000000  <br/> |當電子郵件訊息編制索引時，每個字都會以不同的處理指示加以批註，以指定如何索引該 word。 每一組處理指示稱為「批註」標記。 若要在 Office 365 中維護服務品質，電子郵件的限制是2000000的註解標記。  <br/> |
|索引中的最大主體大小  <br/> |67000000個字元  <br/> |電子郵件內文中的字元總數及其所有附件。 當電子郵件訊息編制索引時，郵件和所有附件中的所有文字都會串連成單一字串。 已編制索引的此字串大小上限為67000000個字元。  <br/> |
|Body 中的唯一標記上限  <br/> |1 百萬  <br/> |如先前所述，token 是從內容提取文字、移除標點符號和空間，然後分割成索引中儲存的字詞（稱為標記）的結果。 例如，片語 `"cat, mouse, bird, dog, dog"` 包含5個權杖。 但只有4個是唯一的標記。 每封電子郵件都有1000000個唯一的標記限制，這有助於防止索引因過大而無法使用隨機標記。  <br/> |

## <a name="more-information-about-partially-indexed-items"></a>有關部分索引項目目的詳細資訊

- 如先前所述，由於郵件和檔案屬性及其中繼資料已編制索引，如果該關鍵字出現在索引中繼資料中，關鍵字搜尋可能會傳回結果。 不過，如果關鍵字只會出現在不支援的檔案類型之專案的內容中，則相同的關鍵字搜尋可能不會傳回相同的專案。 在此情況下，會將專案當做部分編制索引的專案傳回。

- 如果搜尋結果中包含已部分索引的專案，而該專案符合搜尋查詢準則（但未排除），則在預估的搜尋統計資料中，將不會包含在部分編制索引的專案。 此外，當您匯出搜尋結果時，它不會包含部分索引的專案。

- 雖然檔案類型支援索引，而且已編制索引，但可能會有索引或搜尋錯誤，導致檔案以部分索引項目目的形式傳回。 例如，搜尋非常大的 Excel 檔案可能會部分成功（因為會有索引前 4 MB），但由於超過檔案大小限制而失敗。 在此情況下，您可能會傳回相同的檔案與搜尋結果，並以部分索引的專案。

- 使用 Microsoft 技術加密的附加檔案會編制索引，而且可以進行搜尋。 使用非 Microsoft 技術加密的檔案會進行部分編制索引。

- 以 S/MIME 加密的電子郵件會部分編制索引。 其中包括含有或不含檔案附件的加密郵件。

- 使用資訊版權管理（IRM）保護的郵件會進行索引，如果符合搜尋查詢，將會包含在搜尋結果中。

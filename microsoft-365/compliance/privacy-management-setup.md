# <a name="get-started-with-privacy-management-preview"></a>開始使用隱私權管理 (預覽) 

在本文中：瞭解如何設定組織的 **隱私權管理的存取** ，如何 **開始** 評估您的資料，以及如何處理重要 **設定**。

## <a name="sign-up"></a>註冊

您可以在 Microsoft 365 合規性中心內使用隱私權管理。 使用 E1、E3 及 E5 Office 365 和 Microsoft 365 企業版授權的組織可取得隱私權管理的公開預覽。 在正式使用隱私權管理時，組織必須取得新的授權。

請注意，美國政府 Community (GCC) 適中、GCC 高或防衛 (DoD 客戶都無法使用隱私權管理的公開預覽。

若要開始使用公開預覽，請從系統管理中心取得預覽訂閱。 如果您第一次在「規範中心」選取隱私權管理，還沒有授權，您會被導向系統管理中心以開始使用。 建議全域管理員登入，並在第一次造訪隱私權管理時，依照下文所述的方式來設定使用者許可權。 如果您未保留必要的角色以取得訂閱或同意使用隱私權管理的條款，系統會提示您聯繫您的全域系統管理員以取得協助。

確認您是否想要開始使用隱私權管理信號，以同意條款和個人資料評估程式。 您可以在繼續之前複查所提供的連結。

## <a name="set-user-permissions-and-assign-roles"></a>設定使用者權限並指派角色

隱私權管理使用以角色為基礎的存取控制 (RBAC) 許可權模型。 只有獲指派角色的使用者才可存取隱私權管理，且每個使用者所允許的動作受到角色類型的限制。

您可以在 Microsoft 365 合規性中心中處理隱私權管理的許可權和角色指派，如下所示。 請注意，在 Azure Active Directory 中不會出現隱私權管理特有的角色。

### <a name="in-the-microsoft-365-compliance-center"></a>在 Microsoft 365 合規性中心

- 在左側導覽中選取 [許可權]。
- 展開 [規範中心]，然後選取 [角色]。 將會顯示完整的角色群組清單。 
- 向下滾動以尋找隱私權管理群組，或依關鍵字搜尋，例如 "隱私權"。
- 選取相關的角色群組，以查看描述、指派的角色及成員清單。
- 使用這些區段旁邊的 [編輯] 連結，可新增或變更使用者或編輯設定。

### <a name="learn-about-role-groups-and-roles"></a>深入瞭解角色群組和角色

本節概述與隱私權管理相關的角色群組和角色。 成員應依頂級管理員指派給角色群組，視其需要完成的工作及適當的檔案存取層級而定。 每個角色群組都包含一或多個角色。 這些角色可能與特定隱私權管理工作有關，或會對應至已啟用或限制該群組成員的重要功能。

如有需要，可以自訂角色群組。 為了避免意外遺失存取，我們建議您建立您想要自訂之現有角色群組的複本，讓複製可辨識的名稱，並對新的群組進行變更並驗證您的變更，並視需要指派使用者。

**隱私權管理**：此群組包含單一群組中的所有隱私權管理許可權角色。 這是快速開始使用隱私權管理的最簡單方式，以及管理將使用隱私權管理之其他群組的存取控制。 這也適用于不需要個別使用者群組定義個別許可權的組織。

**隱私權管理管理員**：此角色群組的成員會重點關注設定和管理工作，並具有隱私權管理功能的廣泛存取權，包括建立、讀取、更新及刪除隱私權管理原則、主體權利要求、隱私權管理許可權，以及隱私權管理設定。

**隱私權管理分析員**：此角色群組的成員擔當隱私權管理案例分析者。 他們可以調查原則相符、查看檔案中繼資料，以及採取修正動作。 這個群組無法透過內容瀏覽器存取完整檔案。

**隱私權管理調查** 人員：此群組的成員擔當隱私權管理資料調查人員。 他們可以調查原則相符、查看關聯的檔案內容，以及採取修正動作。 這個群組可以透過內容瀏覽器存取檔。

**隱私權管理檢視器**：此群組的成員可以查看隱私權管理中的分析資訊，例如《總覽」、「資料設定檔」及「主題要求報告」。

**主體權利要求管理員**：此群組的成員具有管理的完整存取權，並建立主體許可權要求。

**隱私權管理參與者**：此群組的成員具有對主體權力要求的存取權。

## <a name="configure-settings"></a>進行設定

您可以透過隱私權管理的主要頁面右上角的齒輪輪存取設定頁面。 它可讓隱私權管理管理員設定隱私權管理之間的基本屬性。 選項包括下列。

### <a name="anonymization"></a>匿名

這項功能可讓您向特定角色中的使用者顯示隱私權管理功能中的匿名版本。 這會將可辨識的顯示名稱取代為 "AnonyIS8-988" 等一般標籤，以協助掩蓋使用者在查看機密資料時的身分識別。 此選項不適用於主體權利要求模組。

### <a name="user-notification-emails"></a>使用者通知電子郵件

當我們偵測到資料處理原則的相符時，隱私權管理可以將電子郵件傳送給受影響的使用者，並採取糾正動作和隱私權訓練的連結。 在設定中，您可以啟用或停用隱私權管理整體的電子郵件通知功能。 您可以在建立或編輯原則時，啟用個別通知、設定電子郵件頻率，以及指定訓練 URL。 如果在設定中關閉通知功能，將會停用特定通知訊息的任何原則層級設定。 若要深入瞭解原則，請參閱 [建立及管理原則](privacy-management-policies.md)。

### <a name="teams-collaboration"></a>Teams 共同作業

整合 Microsoft Teams 功能與隱私權管理，以加強與專案關係人的共同作業。 每次建立主體權力要求時，都會建立相關聯的小組。 使用者可以從 [要求的合作者] 索引標籤中新增至小組。若要深入瞭解主題權力要求，請參閱 [管理主體權力要求](privacy-management-subject-rights-requests.md)。

### <a name="power-automate-flows"></a>Power Automate 流量

使用 Power Automate 流量，以自動管理隱私權相關的程式和工作。 您可以使用內建的隱私權管理範本，在設定區段中建立流程，或使用 Power Automate 主控台建立自訂流程。 若要深入瞭解 Power Automate，請參閱[Power Automate](/power-automate/)檔。

### <a name="data-matching"></a>資料符合

使用此區段可上傳描述資料主體之屬性的資料架構，可協助您在 Microsoft 365 環境中搜尋個人資料時，識別正確的資料主體。 架構和規則套件是以 XML 格式建立及上傳。 在 [個人資料上傳] 底下，您也可以提交與所提供架構相符的個人資料。 您可以建立和上傳您自己的檔案，或選擇從 Azure 上傳個人資料。 若要深入瞭解主題權力要求，請參閱 [管理主體權力要求](privacy-management-subject-rights-requests.md)。

### <a name="data-retention-periods"></a>資料保留期間

在 [主體許可權要求] 中，選擇您要在要求關閉後，保留最後收集和報告的資料多久。 您可以選擇30或90天之間的日期。 若要深入瞭解主題權力要求，請參閱 [管理主體權力要求](privacy-management-subject-rights-requests.md)。

### <a name="data-review-tags"></a>資料審閱標記

管理您將用來標示主體權利要求中所檢索的檔案的標記。 在本節中，您可以編輯自訂標記的名稱和描述。 您也可以編輯系統所提供之內建標記的標記描述。 無法變更系統標記的名稱。 若要深入瞭解主題權力要求，請參閱 [管理主體權力要求](privacy-management-subject-rights-requests.md)。

## <a name="get-initial-data-insights"></a>取得初始資料洞察力

登入隱私權管理後，您將會看到 [ **概述** ] 頁面。 此頁面提供有關儲存在 Microsoft 365 環境中個人資料的動態見解，以協助您快速找出問題、識別風險指標，並採取行動以修正問題。 您的概要應該在註冊前24小時內填入初步深入資訊。 當您繼續使用隱私權管理時，[一覽] 頁面會重新整理，以繼續提供目前的資訊。

若要進一步深入瞭解您在一段時間內的資料，您的 **資料設定檔** 頁面將提供更多的視覺化和分析功能，並為您提供組織資料的高層次視圖，依地理位置和 Microsoft 365 服務。

若要深入瞭解這些頁面，請參閱 [尋找及形象化您的資料](privacy-management-data-profile.md)。

# <a name="national-health-service-nhs-information-governance-ig-toolkit"></a>國民健保 (NHS) 資訊控管 (IG) 工具組

## <a name="nhs-ig-toolkit-overview"></a>NHS IG 工具組概觀

國民健保 (NHS) 是英國的國家醫療體系，為英國公民提供大部分醫療保健服務，涵蓋所有照護作業領域。 NHS 創立於 1948 年，是世界上第一個單一支付者的醫療體系。 這也是世界上最大的醫療體系，雇用了超過 150 萬人，2016 年的預算為 1164 億英鎊。

NHS 管理超過 6400 萬 NHS 病患的健康資料。 NHS 病患資料的收集、儲存和處理受多項法令規定的約束，包括 1998 年資料保護法 (Data Protection Act) 和機密性 NHS 實務規範。

NHS 委託健康與社會照護資訊中心 (HSCIC) 制定並維護管理病患資料收集、儲存和處理的單一標準，即[資訊控管 (IG) 工具組](https://www.igt.hscic.gov.uk/resources/About%20the%20IG%20Toolkit.pdf)。 IG 工具組旨在鼓勵和指導有意透過遵循指導方針的流程來託管個人健康資料的組織。

所有有權存取 NHS 病患資料的組織都需要使用 NHS IG 工具組提供證據，證明他們正在採取適當措施保護病患資料。

此外，像 Microsoft 這樣的為醫療保健提供者提供平台的組織也使用該工具組針對 NHS 資訊控管、安全性和隱私權要求，對他們的安全性和隱私權控制進行自我評估。

遵守 NHS IG 工具組有助於保護病患資料的完整性和機密性，避免未經授權的存取、遺失、損壞和毀壞。 必須採取適當的緩解步驟，以補救在評估過程中發現的任何不符合合規性問題。

NHS IG 工具組旨在：

- 提供標準，解決客戶對 NHS 病患資料安全性和機密性，以及對業務之影響的共同顧慮
- 展示可衡量的合規性，並深入了解病患資料的潛在風險
- 提升 NHS 和合作夥伴組織的信任和公眾信任

## <a name="frequently-asked-questions"></a>常見問題集

**誰可以利用 Azure NHS IG 工具組評估？**

希望使用 Microsoft Azure 作為託管 NHS 病患資料之平台的組織可以在 Microsoft 的評定報告的基礎上啟動自己的 NHS IG 工具組評估，同時牢記他們還必須處理任何其他控制措施，特別是那些不由雲端服務提供者負責的控制措施。

**我要從何處著手組織自身的合規性工作？**

鼓勵有意託管 NHS 病患資料的組織透過[檢閱 NHS IG 工具組指導方針](https://www.igt.hscic.gov.uk/requirementsorganisation.aspx)來確定其必須具備的範圍和控制措施，以熟悉管理要求。

**IG 工具組的等級為何？**

IG 工具組的等級為 0 到 3：

- 沒有足夠的證據達到等級 1
- 組織已開始規劃符合規範所需的原則、程式和/或流程
- 已核准並實作的 IG 原則和程序已提供給所有相關員工
- 監控並確保員工合規性，以及原則和程序的有效性

# <a name="create-and-manage-policies-in-privacy-management-preview"></a>在隱私權管理 (預覽中建立及管理原則) 

在本文中：瞭解如何建立及自訂處理個人資料的 **原則** 、取得原則相符的 **警示** ，以及修正 **問題**。

## <a name="purpose-of-policies"></a>原則用途

原則可讓您定義要在公司的 Microsoft 365 資料中查看的隱私權風險類型，並在找到相符專案的情況下建立慣用的結果。 您的組織可以從產生的提醒工作，以複查任何相符的資料，並修正問題，全部都來自隱私權管理解決方案內。

隱私權管理提供範本，可讓您輕鬆開始建立原則，並可讓您透過廣泛的自訂選項微調您的方法。 隱私權管理範本所涵蓋的主要案例包括下列專案：

- **資料 overexposure**：偵測 overexposed 個人資料，並提示使用者保護其安全。
- **資料傳輸**：斑點及協助限制跨部門或地區界限傳輸個人資料。
- **資料** 最小化：協助使用者識別並減少未使用的個人資料量。

若要深入瞭解每個範本的功能，請參閱下一節。

## <a name="policy-types"></a>原則類型

### <a name="data-overexposure"></a>資料 overexposure

隱私權管理可協助您偵測並處理已儲存的資料 insufficiently 安全的情況。 例如，如果對內部網站的存取開啟太寬的群組，或許可權設定尚未更新，則儲存在該網站上的個人資料可能會遭到破壞。 您可以使用隱私權管理的資料原則範本來評估您的資料，並提醒您可能發生的問題。

### <a name="data-transfer"></a>資料傳輸

跨部門或地區界限傳輸資料可能會增加資料洩密的風險，例如，如果是透過未加密的電子郵件傳送，或是未獲授權的收件者。 這類動作可能會有法規影響，或可能會違反隱私權的作法。 使用資料傳輸範本建立隱私權管理原則，可找出並協助限制這類轉移。

> [!NOTE]
> 在公開預覽期間，執行資料傳送原則的某些租使用者可以偵測跨區域的傳輸，可能會發生同步處理問題，使 Exchange 和 Teams 資料中的原則相符。 建議您在預覽此原則類型時，專注于 SharePoint 和 OneDrive 資料。 此問題的更新預期為秋季2021。

### <a name="data-minimization"></a>資料最小化

經過一段時間後，公司可以從客戶或員工收集大量的個人資料。 這種情況下，這包括因需要而收集的資料，或是未使用的資料，因此應該減少，以限制該資訊的隱私權風險。 資料最小化範本可用於處理這種類型的風險。

## <a name="get-started-with-default-templates"></a>開始使用預設範本

隱私權管理會透過使用資料最小化、資料 overexposure 及資料傳輸的範本，以預設設定來建立三個原則，以協助 kickstart 您的資料評估程式。 預設會啟用這些原則，但不會自動觸發通知訊息或修復提示。 初次安裝之後，您可以繼續建立及自訂您自己的原則。

## <a name="create-a-privacy-management-policy"></a>建立隱私權管理原則

有兩個路徑可以建立隱私權管理原則。 第一個選項是從預先定義的樣板集合中選擇。 您也可以自訂您自己的原則，使用任何範本做為開始點。

### <a name="create-a-policy-from-a-template"></a>從範本建立原則

若要立即開始使用原則，請選取三個預先設定的原則類型之一。 若要查看有關任何專案的詳細資料，您可以選取 [View settings] （查看）以查看構成原則的特定屬性，包括資料類型、資料位置，以及觸發原則相符的條件。

當您直接從範本建立原則時，會自動為您選擇許多設定。 這包括依預設開啟原則。 如果您想要在動作中預覽原則，然後才完全啟動原則，請在建立後，在清單中尋找該原則，然後編輯原則，並將其切換至測試模式。 如需詳細資訊，請參閱 [測試您的原則](#test-your-policy)。

### <a name="create-custom-policy"></a>建立自訂原則

若要對原則的設定進行精細的控制，您可以使用其中一個現有範本做為基準來建立自訂原則。 隱私權管理可提供嚮導逐步指導您完成這些步驟。

可自訂屬性包括：

- **名稱和類型**：選擇要在其上建立原則的範本，然後命名並描述您的版本。
- **要監視的資料**：選取您的原則將監視的個人資料類型。 您可以選擇可用的分類群組，或從個別的敏感資訊類型清單中選擇。 若要深入瞭解，請參閱選擇下列資料監控選項。
- **使用者**：選擇是否要將此原則套用至所有使用者或選取的使用者。 如果您選擇第二個選項，您最多可以從提供的清單中選擇300個使用者進行選擇。
- **位置**：選擇您的原則應該涵蓋 Microsoft 365 內的位置，例如組織的 SharePoint 或 Exchange 資料。
- **條件**：選取原則類型的相關條件。 例如，您可以指定資料傳送原則應針對哪些類型的傳輸，或最近使用資料最小化原則的資料。
- **結果**：在偵測到原則相符時定義結果。 您的選項取決於您所開始的原則類型。 可能的結果包括：
  - **電子郵件通知**：此設定可讓您觸發摘要電子郵件通知，包括相關訓練的連結。 如需詳細資訊，請參閱設定使用者的電子郵件通知。
  - **Teams**：提供使用者 Teams 原則提示和建議，以及相關訓練的連結。 此選項適用于資料傳送原則。
- **警示**：當偵測到原則相符時，決定系統管理員的提醒頻率。 在達到特定臨界值時，選項會包含沒有警示、每個原則相符的警示或警示。 如果您選擇 [臨界值] 選項，請設定您想要的參數。
- **模式**：決定是否要先執行測試模式中的原則或立即開啟原則。 如需詳細資訊，請參閱測試您的原則。
當您在嚮導中逐步執行所有設定時，請複查您的設定，並在必要時進行任何最後的編輯，並儲存原則。

#### <a name="choose-data-monitoring-options"></a>選擇資料監控選項

設定自訂原則時，系統會要求您選取原則應監視的資料類型。 選項如下：

- **分類群組**：以主要隱私權法規（如 GDPR 或 HIPAA）為基礎的可搜尋資料集清單。 查看任何群組的詳細資料，以查看其所涵蓋的敏感資訊類型。 選取一或多個這些集合以使用。 目前可用的群組如下：
  - 澳大利亞狀況記錄法案 (HRIP 法案) 增強
  - 澳大利亞隱私權法案增強功能
  -  (歐盟) 一般資料保護法規 (GDPR) 增強
  - 日本個人身分識別資訊 (PII) 資料已增強
  - 加強個人資訊的日本保護
  - 美國 Gramm-Leach-Bliley 法案 (GLBA) 增強
  - 美國衛生保險業法案 (HIPAA) 增強
  - 美國愛國法案增強型
  - U.S. 個人身分識別資訊 (PII) 資料增強
  - 增強的美國狀態違規通知法規
- **個別的敏感資訊類型**：您可以自行選擇特定的敏感資訊類型，如社會保險號碼或駕駛執照資訊，也可以自訂您自己的群組或要尋找的資料群組。 這個嚮導可讓您從隱私權管理中完整的敏感資訊類型清單中進行選取。 每種資訊類型都有自己的屬性。 使用 [資訊] 按鈕旁邊的 [資訊] 按鈕，以取得有關建議設定的詳細資料和備註。 如果您建立一個以上的群組，嚮導會讓您套用 Boolean 運算子以建立關聯，並定義其運作順序。

如果您使用預先設定的分類群組，您也無法選取個別類型或建立您自己的群組。 為了獲得最大的彈性，請選擇個別的敏感資訊類型。 若要使用最常見的標準，請從分類群組中選擇。

#### <a name="test-your-policy"></a>測試您的原則

如果您想要在完全啟動新原則之前加以評估，請將原則設定為測試模式。 測試模式可讓您尋找過去30天的相符專案、估量原則行為，以及檢查所產生的警示類型。 建議至少執行五天的測試原則，以取得代表的結果。 您可以在測試階段期間使用此選項，來編輯和調整原則的設定。 當您取得執行測試的洞察力後，您就可以繼續開啟原則。 當原則在測試模式中執行時，不會傳遞任何使用者通知訊息。

#### <a name="set-user-email-notifications"></a>設定使用者電子郵件通知

透過電子郵件通知，使用者會收到有關原則相符和重要工作完成的直接通知。 收件者會收到電子郵件摘要，以摘要要檢查的資料和可能的動作，例如，將檔設為私密、保留檔、報告任何誤報，以及新增記事以供日後參考。 這些電子郵件也包含訓練收件者如何處理這些案例的連結。 在初次設定通知時，提供這些連結是必要的，並且應該指向您自己的內部檔，以處理常式和最佳作法。

您可以在建立自訂原則或編輯任何原則時，為個別原則啟用通知。 使用 [結果] 區段定義偵測到原則時所發生的動作，包括啟用這些通知的選項，以及設定您要傳送這些摘要的頻率。

電子郵件通知功能是在設定內的全域層級加以控制。 預設為啟用。 關閉此設定會停止所有電子郵件，即使特定的通知已設定于個別原則層級也是一樣。 如需詳細資訊，請參閱 Configure 「 [開始使用隱私權管理](privacy-management-setup.md#configure-settings)」下的設定設定。

## <a name="view-policy-details"></a>查看原則詳細資料

在建立原則之後，請在 [主要 **原則** ] 頁面上選取原則，以查看其完整的概要。 [原則詳細資料] 頁面會深入瞭解您的資料，讓您能夠查看特定原則相符的內容，並建議您在後續步驟。 如果您的原則是在測試模式中執行，則此頁面也可讓您在測試完成時開啟原則。

在作用中使用原則之後，您可以繼續檢查其 [原則詳細資料] 頁面，以查看有關問題範圍、警示嚴重性和趨勢，以及採取的糾正動作的日常洞察力。

## <a name="resolve-policy-alerts-and-issues"></a>解決原則警示及問題

一旦您的原則已啟用，隱私權管理就會通知您 apprised 重要的發現，其方式是通知您原則相符、評分嚴重性，並讓您能夠建立及解決問題以採取行動。

隱私權管理的 [一覽] 頁面可讓您透過有關關注重要方面的動態更新（例如，最相符的原則和您目前使用的原則警示），提供這些調查結果的視圖。 您也可以透過 [主要原則] 頁面，來存取警示和問題的詳細資料。

### <a name="alerts"></a>警示

若要評估您的使用中警示，並指定需要跟進的問題，請存取您的 **提醒** 頁面。 它會提供您的原則所產生之警示的可篩選清單，您可以個別檢查您的原則，以判斷觸發的情況。

選取任何警示會開啟 flyover 窗格，其中會顯示其他詳細資料，例如原則類型、相符專案的數目，以及原則設定所判斷的嚴重性。 您可以在 [ **內容** ] 索引標籤下複查此警示中的檔案。 此資訊可提供其他深入瞭解觸發警示的特定事件（檔案所在的位置），以及涉及哪些類型的個人資料。 警示的觸發器是由每個原則的特定條件所決定。 例如，如果隱私權管理偵測到原則所指定的部門或地區之間的轉移，可能會在資料傳送原則上觸發警示。

在評估清單中的任何警示後，您可以使用 [ **建立問題** ] 動作來提示進一步調查和動作。 系統會要求您命名問題，並為內容新增相關的批註。 您也可以在不需要跟進的情況取消通知。

### <a name="issues"></a>問題

如 [警示] 區段中所述，在評估原則相符的警示時，會建立問題。 若要追蹤並解決指定的關注事項，請流覽 [問題] 頁面。 您可以從這裡查看個別問題、調查 instigating 條件、查看資料，並採取必要的步驟來關閉案例。

此頁面提供所有已開啟問題的清單。 會依名稱列出問題，並依嚴重性排序，以協助您排定案例優先順序，包括「高」、「中」和「低類別」及「未指派」。 在清單中選取任何問題，以複查其內容並採取動作加以解決。 您可以在審核期間對未指派的問題提供嚴重性評級。

#### <a name="issue-overview"></a>問題一覽

「問題詳細資料」頁面可協助您逐步瞭解如何處理已識別的隱私權風險，以及正確地處理指定的檔案。 在 [ **概覽** ] 索引標籤上，您可以看到目前的步驟，以指出問題的狀態及下一個建議的動作。 您也可以查看相關內容的基本資訊、相關聯的原則、警示的詳細資料，以及時程表。

後續的索引標籤會提供相關警示和內容的進一步詳細資訊，以及小組中處理問題之其他人的任何記事。 您可以透過 [ **合作者** ] 索引標籤來管理作用中的參與者清單。

#### <a name="share-the-issue"></a>分享問題

將人員新增為合作者可讓您透過安全的 Microsoft Teams 通道、公司電子郵件，或直接將連結共用至隱私權管理中問題的頁面，與其他企業成員共用問題。 這些選項可在 [ **共用** ] 按鈕底下使用。 透過 Teams 進行共用時，系統會要求您從組織中的可用團隊中進行選取，並選取特定通道，然後留下有關該問題的訊息，該訊息會與指定的通道共用。

#### <a name="review-content-and-remediate"></a>審閱內容和修正

若要查看與問題相關的內容，請選擇 [ **檢查內容** 動作時出現提示] 或 [開啟內容] 索引標籤。選取清單中的任何檔案，以完整查看檔案。 若要管理此檔案，您可以在這裡看到檔案的詳細資料、記錄的任何活動，以及其修復歷程記錄。

使用 [ **修正** ] 按鈕，為此內容進行您自己的資料處理決策。 選取此按鈕可讓您選擇一或多項修復動作。 選項包括：

**所有原則**

- **Notify**：通知內容擁有者所偵測到的問題。
- 套用 **保留標籤**：新增此專案之資料保留標籤的標籤。 
- 套用 **敏感度標籤**：新增此專案資料敏感度的標籤。
- **標示為不符合** 專案：將搜尋結果識別為誤報，以移除內容專案的考慮。

**資料最小化**

- **回收/刪除**：使用此選項以虛刪除資料。 內容會移至 [刪除的郵件] 資料夾或 [回收] bin 中 (Exchange、SharePoint、OneDrive) 或已刪除，且具有復原 (Teams 郵件) 的選項。 刪除可在設定的一段時間內逆轉，視服務的設定而定。

**資料 overexposure 及資料傳輸**

- 取消 **共用：停止** 共用此內容專案的連結。

每個選項都會提示您在確認您的選擇之前，留下內容擁有人的批註及任何其他必要的支援資訊。

採取所有修正步驟並準備關閉問題後，請使用 [解決] 按鈕並新增您的最終評論，再提交。

## <a name="delete-a-policy"></a>刪除原則

如果您需要移除現有的隱私權管理原則，請在 [原則] 頁面上的清單中找到它，然後選取 [動作] 功能表 (垂直橢圓形) ]，然後選擇 [刪除原則] 動作。 在刪除之前，系統會要求您確認所做的選擇，而且會永久移除該原則。 刪除原則不會影響任何先前由原則評估的檔案，而且原則所產生的問題和警示仍會保留。

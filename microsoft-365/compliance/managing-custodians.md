# <a name="work-with-custodians-and-non-custodial-data-sources-in-advanced-ediscovery"></a>在高級 eDiscovery 中搭配保管人和非 custodial 資料來源

當組織對法律調查作出回應時，識別、保留及收集可能相關之內容的工作流程是以組織中的人員（相關資料的保管人）為基礎。 在 eDiscovery 中，這些個人稱為「*資料保管人*」（或「僅限*保管人*」），而且會定義為「具有檔或電子檔的系統管理控制權」的人員。 例如，電子郵件的系統管理員可以是包含相關郵件之信箱的擁有者。

此外，信箱和網站中的內容也可能不會與保管人相關聯，但與案例有關。 若要區分內容位置（即「保管人」不具備系統管理控制，但可能包含相關資料），這稱為*非 custodial 資料來源*。

在高級 eDiscovery 案例中，法律小組可以將組織中的個人新增為保管人，並自動識別及保留 custodial 資料來源，例如 Exchange 信箱、OneDrive 帳戶，以及 SharePoint 和小組網站。 他們也可以識別及保留非 custodial 的資料來源。 透過使用內建的保管人和資料來源管理工具，可將電子化儲存的資訊從無意（或有意的）刪除中保護。 這可讓您避免手動執行合法保留程式的耗時且容易出錯的程式。

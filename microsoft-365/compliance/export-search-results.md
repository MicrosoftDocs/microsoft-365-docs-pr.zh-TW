# <a name="export-content-search-results"></a>匯出內容搜尋結果

成功运行内容搜索后，可以将搜索结果导出到本地计算机。 當您匯出電子郵件結果時，其會以 PST 檔案下載到您的電腦。 当您从 SharePoint 和 OneDrive 导出业务网站的内容时，将导出本机 Office 文档的副本。 导出的搜索结果中还包含其他文档和报告。
  
内容搜索结果中包含的任何 RMS 加密电子邮件将在导出它们（作为单个邮件）时解密。 默认情况下，电子数据展示管理器角色组的成员启用此解密功能。 这是因为 RMS 解密管理角色分配给此角色组。 有关导出搜索结果时 RMS 解密的详细信息，请参阅[详细信息](#more-information)部分。 
  
导出内容搜索的结果涉及准备结果，然后将它们下载到本地计算机。
  
## <a name="before-you-begin"></a>開始之前

- 要导出搜索结果，您必须在安全&合规性中心中分配导出管理角色。 此角色分配给内置电子数据展示管理器角色组。 默认情况下，它不分配给组织管理角色组。 有关详细信息，请参阅[分配电子数据展示权限](assign-ediscovery-permissions.md)。
    
- 用于导出搜索结果的计算机必须满足以下系统要求：
    
  - 32 位或 64 位版本的 Windows 7 和更高版本
    
  - 微软 .NET 框架 4.7
    
  - 支援的瀏覽器：
    
     - Microsoft Edge
    
        或
    
     - 微软互联网浏览器10及更高版本
    
    **注意：** Microsoft 不会为 ClickOnce 应用程序制造第三方扩展程序或加载项。 不支持使用不支持的浏览器导出搜索结果以及第三方扩展或加载项。 
    
- 下载搜索结果（步骤 2 中介绍）时，可以通过在用于导出搜索结果的计算机上配置 Windows 注册表设置来提高下载速度。 有关详细信息，请参阅从[Office 365 导出电子数据展示搜索结果时提高下载速度。](increase-download-speeds-when-exporting-ediscovery-results.md)
    
- 导出搜索结果时，数据在下载到本地计算机之前，会暂时存储在 Microsoft 云中 Microsoft 提供的 Azure 存储位置。 确保您的组织可以连接到 Azure 中的终结点，该终结点是**\*.blob.core.windows.net（** 通配符表示导出的唯一标识符）。 搜索结果数据在创建两周后从 Azure 存储位置删除。 
    
- 如果您的组织使用代理服务器与 Internet 通信，则需要定义用于导出搜索结果的计算机上的代理服务器设置（以便导出工具可以由代理服务器进行身份验证）。 为此，请打开与 Windows 版本匹配的位置中的*machine.config*文件。 
    
  - **32 位：**`%windir%\Microsoft.NET\Framework\[version]\Config\machine.config`
    
  - **64 位：**`%windir%\Microsoft.NET\Framework64\[version]\Config\machine.config`
    
    将以下行添加到 和`<configuration>``</configuration>`标记之间的*计算机.* 确保为您的组织替换`ProxyServer``Port`和 正确值;例如， `proxy01.contoso.com:80` . 
    
    ```
    <system.net>
       <defaultProxy enabled="true" useDefaultCredentials="true">
         <proxy proxyaddress="http://ProxyServer :Port " 
                usesystemdefault="False" 
                bypassonlocal="True" 
                autoDetect="False" />
       </defaultProxy>
    </system.net>
    ```
    
## <a name="step-1-prepare-search-results-for-export"></a>步骤 1：准备导出的搜索结果

第一步是准备要导出的搜索结果。 准备结果时，它们将上载到 Microsoft 云中 Microsoft 提供的 Azure 存储位置。 来自邮箱和网站的内容以每小时 2 GB 的最大速率上载。
  
1. 移至 [https://protection.office.com](https://protection.office.com)。
    
2. 使用公司或學校帳戶登入 Office 365。
    
3. 在"安全&合规性中心"的左侧窗格中，****\>单击"**搜索内容搜索"。**
    
4. 在"**内容"搜索**页上，选择搜索。 
    
5. 在详细信息窗格中，在"**将结果导出到计算机"** 下，**单击"开始导出"。**
    
    > [!NOTE]
    > 如果搜索结果超过 7 天，系统将提示您更新搜索结果。 如果发生这种情况，请取消导出，单击所选搜索的详细信息窗格**中的"更新搜索结果"，** 然后在更新结果后再次启动导出。 
  
6. 在"**导出"搜索结果**页上**的"输出选项"** 下，选择以下选项之一：
    
    - 所有项目（不包括具有无法识别格式的项目）均已加密或由于其他原因未编制索引
    
    - 所有项目（包括具有无法识别格式的项目）均已加密或由于其他原因未编制索引
    
    - 只有具有无法识别的格式、加密或由于其他原因未编制索引的项目
    
    有关如何导出部分索引项的说明，[请参阅"详细信息"](#more-information)部分部分部分部分部分部分部分部分。 有关部分索引项的详细信息，请参阅[内容搜索 中部分索引的项目。](partially-indexed-items-in-content-search.md)
    
7. 在"**导出交换"内容下，** 选择以下选项之一：
    
    - **每个邮箱一个 PST 文件：** 为每个包含搜索结果的用户邮箱导出一个 PST 文件。 来自用户存档邮箱的任何结果都包含在同一 PST 文件中。 此选项从源邮箱重现邮箱文件夹结构。 
    
    - **一个包含所有消息的 PST 文件：** 导出单个 PST 文件（名为*Exchange.pst），* 该文件包含搜索中包含的所有源邮箱的搜索结果。 此选项将为每个邮件重现邮箱文件夹结构。 
    
    - **一个 PST 文件，其中包含单个文件夹中的所有邮件：** 将搜索结果导出到单个 PST 文件，其中所有邮件都位于单个顶级文件夹中。 此选项允许审阅者按时间顺序审阅项目（项目按发送日期排序），而无需为每个项目导航原始邮箱文件夹结构。 
    
    - **个人消息：** 使用 .msg 格式将搜索结果导出为单个电子邮件。 如果选择此选项，电子邮件搜索结果将导出到文件系统中的文件夹。 将结果导出到 PST 文件时，单个邮件的文件夹路径与使用的文件夹路径相同。 
    
      > [!IMPORTANT]
      > 要在导出 RMS 加密邮件时解密邮件，必须将电子邮件搜索结果导出为单个邮件。 如果将搜索结果导出为 PST 文件，加密邮件将保持加密状态。 
  
8. 单击"**启用重复数据消除"** 复选框以排除重复邮件。 仅当搜索的内容源包括 Exchange 邮箱或公用文件夹时，才会显示此选项。 
    
    如果选择此选项，即使搜索的邮箱中找到同一邮件的多个副本，也只会导出邮件的一个副本。 导出结果报告 （Results.csv） 将包含重复邮件的每个副本的行，以便您可以标识包含重复邮件副本的邮箱（或公用文件夹）。 有关重复数据消除以及如何识别重复数据项目的详细信息，请参阅[电子数据展示搜索结果中的重复数据消除。](de-duplication-in-ediscovery-search-results.md)
    
9. 单击"**包含 SharePoint 文档的版本"** 复选框可导出所有版本的 SharePoint 文档。 仅当搜索的内容源包括"共享点"或"企业网站的 OneDrive"时，才会显示此选项。 
    
10. 单击**压缩（压缩）文件夹中的"导出文件"** 复选框，将搜索结果导出到压缩文件夹。 仅当选择将 Exchange 项目导出为单个邮件以及搜索结果包含 SharePoint 或 OneDrive 文档时，此选项才可用。 此选项主要用于在导出项目时绕用 Windows 文件路径名称中的 260 个字符限制。 [请参阅"详细信息"](#more-information)部分中的"导出项目的文件名称"。 
    
11. **单击"开始导出"。**
    
    搜索结果已准备好用于下载，这意味着它们已上载到 Microsoft 云中的 Azure 存储位置。 当搜索结果准备好下载时，详细信息窗格中的"**将结果导出到计算机"** 下将显示"**下载导出的结果"** 链接。 
  
## <a name="step-2-download-the-search-results"></a>第 2 步：下载搜索结果

下一步是将搜索结果从 Azure 存储位置下载到本地计算机。
  
如前所述，您可以通过在用于导出搜索结果的计算机上配置 Windows 注册表设置来提高下载速度。 有关详细信息，请参阅从[Office 365 导出电子数据展示搜索结果时提高下载速度。](increase-download-speeds-when-exporting-ediscovery-results.md)
  
1. 在开始导出的搜索的详细信息窗格中，在将**结果导出到计算机**下，**单击"下载导出的结果"。**
    
    **将显示"下载导出的结果"** 窗口，其中包含有关将下载到您的计算机的搜索结果的以下信息。 
    
    - 要下载的项目数。
    
    - 将下载的项目的估计总大小。
    
    - 将导出索引还是未索引。 未编制索引的项目是具有可识别格式、已加密或由于其他原因未编制索引的项目。 有关详细信息，请参阅[内容搜索 中的未编制索引的项目。](partially-indexed-items-in-content-search.md)
    
    - 是否下载 SharePoint 文档的版本。
    
    - 导出准备过程的状态。 即使数据准备未完成，您也可以开始下载搜索结果。
    
2. **在"导出键"** 下，**单击"复制到剪贴板"。** 在步骤 5 中使用此密钥下载搜索结果。
    
    > [!NOTE]
    > 由于任何人都可以安装和启动电子数据展示导出工具，然后使用此密钥下载搜索结果，因此请务必采取预防措施来保护此密钥，就像保护密码或其他安全相关信息一样。 
  
3. **单击"下载结果"。**
    
4. 如果系统提示您安装**MicrosoftOffice 365 电子数据展示导出工具，** 请单击"**安装"。**
    
5. 在**电子数据展示导出工具**中，将步骤 2 中复制的导出密钥粘贴到相应的框中。
    
6. 按一下 [瀏覽]**** 以指定搜尋結果檔案要下載到的位置。 
    
    > [!NOTE]
    > 由于磁盘活动量大（读取和写入），您应该将搜索结果下载到本地磁盘驱动器;因此，应将搜索结果下载到本地磁盘驱动器。不要将它们下载到映射的网络驱动器或其他网络位置。 
  
1. 按一下 [開始]**** 將搜尋結果下載至您的電腦。 
    
    **电子数据展示导出工具**显示有关导出过程的状态信息，包括要下载的剩余项目的数量（和大小）的估计值。 导出过程完成后，可以访问这些文件下载位置中的文件。

 ### <a name="export-limits"></a>匯出限制
  
- 从安全&合规性中心导出搜索结果具有以下限制：
    
  - 从单个内容搜索最多可导出 2 TB 的数据。 如果搜索结果大于 2 TB，请考虑使用日期范围或其他类型的筛选器来减小搜索结果的总大小。
    
  - 您的组织可以在一天内导出最多 2 TB 的数据。
    
  - 组织内最多可以同时运行 10 个导出。
    
  - 单个用户最多可以同时运行三个导出。

  > [!NOTE]
  > 仅从内容搜索导出报表也会计算同时运行的导出数和单个用户可以运行的导出数。
    
- 如前所述，来自邮箱和站点的搜索结果将上载到 Microsoft 提供的 Azure 存储位置（如[步骤 1：准备导出的搜索结果），](#step-1-prepare-search-results-for-export)最大速率为每小时 2 GB。
    
- 默认情况下，可导出的 PST 文件的最大大小为 10 GB。 这意味着，如果用户邮箱的搜索结果大于 10 GB，则邮箱的搜索结果将在两个（或更多）单独的 PST 文件中导出。 如果选择导出单个 PST 文件中的所有搜索结果，则如果搜索结果的总大小大于 10 GB，则 PST 文件将溢出到其他 PST 文件中。 如果要更改此默认大小，可以编辑用于导出搜索结果的计算机上的 Windows 注册表。 请参阅[导出电子数据展示搜索结果时更改 PST 文件的大小。](change-the-size-of-pst-files-when-exporting-results.md)
    
    此外，除非单个邮箱中的内容超过 10 GB，否则特定邮箱的搜索结果不会在多个 PST 文件之间分配。 如果选择在一个 PST 文件中导出搜索结果，该文件包含单个文件夹中的所有邮件，并且搜索结果大于 10 GB，则这些项目仍按时间顺序组织，因此将根据发送的 d 将这些项目分出其他 PST 文件吃。
     
 ### <a name="export-reports"></a>导出报告
  
- 导出搜索结果时，除了搜索结果之外，还包括以下报告。
    
  - **导出摘要**包含导出摘要的 Excel 文档。 这包括搜索的内容源数、搜索结果的估计大小和下载大小以及导出的项目的估计和下载数量等信息。 
    
  - **清单**包含搜索结果中包含的每个项的信息的清单文件（以 XML 格式）。 
    
  - **结果**包含作为搜索结果下载的每个项目的信息的 Excel 文档。 对于电子邮件，结果日志包含有关每条消息的信息，包括： 
    
      - 邮件在源邮箱中的位置（包括邮件是在主邮箱还是存档邮箱中）。
        
      - 邮件的发送或接收日期。
        
      - 邮件中的"主题"行。
        
      - 邮件的发件人和收件人。
        
      - 如果在导出搜索结果时启用了重复数据消除选项，则该消息是否是重复消息。 重复消息在"**重复到项目"** 列中具有一个值，该值将消息标识为重复消息。 "**复制到项目"** 列中的值包含已导出的消息的项标识。 有关详细信息，请参阅[电子数据展示搜索结果中的重复数据消除。](de-duplication-in-ediscovery-search-results.md)
        
      对于来自 SharePoint 和 OneDrive 的业务站点的文档，结果日志包含有关每个文档的信息，包括：
        
      - 文档的 URL。
        
      - 文档所在的网站集的 URL。
        
      - 文档上次修改的日期。
        
      - 文档的名称（位于结果日志的"主题"列中）。
    
  - **未编制索引的项目**包含搜索结果中包含的任何部分索引项的信息的 Excel 文档。 如果在生成搜索结果报告时未包含部分索引项目，则此报告仍将下载，但将为空。 
    
  - **错误和警告**包含导出过程中遇到的文件的错误和警告。 有关特定于每个错误或警告的信息，请参阅"错误详细信息"列。 
    
  - **跳过的项目**当您从 SharePoint 和 OneDrive 导出业务网站的搜索结果时，导出通常包括跳过的项目报告（已跳过的 Items.csv）。 此报表中引用的项目通常是不会下载的项目，例如文件夹或文档集。 不导出这些类型的项目是设计使然。 对于跳过的其他项目，跳过的项目报告中的"错误类型"和"错误详细信息"字段显示项目跳过的原因，并且未与其他搜索结果一起下载。 
    
  - **跟踪日志**包含有关导出过程的详细日志记录信息，可帮助在导出期间发现问题。 
    
    > [!NOTE]
    > 您无需导出实际搜索结果，即可导出这些文档。 请参阅[导出内容搜索报告](export-a-content-search-report.md)。 
  
 ### <a name="exporting-partially-indexed-items"></a>导出部分索引项
  
- 如果要从返回搜索结果中的所有邮箱项目的内容搜索中导出邮箱项目（因为搜索查询中不包含任何关键字），则部分索引的项目不会复制到包含未编制索引项目的 PST 文件中。 这是因为所有项目（包括任何部分索引项）都将自动包含在常规搜索结果中。 这意味着部分索引项将包含在包含其他索引项的 PST 文件（或单个消息中）。
    
    如果同时导出索引项和部分索引项，或者仅从返回所有项的内容搜索中导出索引项，则将下载相同数量的项。 即使内容搜索的估计搜索结果（显示在安全&合规性中心的搜索统计信息中）仍将包含部分索引项数的单独估计值，也会发生这种情况。 例如，假设包含所有项（搜索查询中没有关键字）的搜索估计值显示已找到 1，000 个项目，并且还发现了 200 个部分索引的项目。 在这种情况下，1，000 项包括部分索引项，因为搜索返回所有项。 换句话说，搜索总共返回了 1，000 个项目，而不是 1，200 个项目（如您所料）。 如果导出此搜索的结果并选择导出索引项和部分索引项（或仅导出部分索引项），则将下载 1，000 个项目。 同样，这是因为当您使用空白搜索查询返回所有项目时，部分索引项包含在常规（索引）结果中。 在同一示例中，如果选择仅导出部分索引项，则仅下载 200 个未编制索引的项目。
    
    另请注意，在前面的示例中（当您导出索引和部分索引项或仅导出索引项时），导出搜索结果中包含的**导出摘要**报告将列出 1，000 个估计项目和 1，000 个下载的项目项，原因与前面所述相同。 
    
- 如果导出结果的搜索是搜索组织中的特定内容位置或所有内容位置，则仅导出来自包含符合搜索条件的项目的内容位置的部分项目。 换句话说，如果在邮箱或站点中找不到搜索结果，则该邮箱或站点中任何部分索引的项目将不会导出。 这样做的原因是，从组织中的许多位置导出部分索引的项目可能会增加导出错误的可能性，并增加导出和下载搜索结果所需的时间。
    
    要从搜索的所有内容位置导出部分索引项，请将搜索配置为返回所有项目（通过从搜索查询中删除任何关键字），然后在导出搜索结果时仅导出部分索引项。
    
    ![使用第三个导出选项仅导出未编制索引的项目](media/5d7be338-a0e5-425f-8ba5-92769c24bf75.png)
  
- 从 SharePoint 或 OneDrive 导出业务网站的搜索结果时，导出未编制索引的项目的能力还取决于您选择的导出选项以及搜索的网站是否包含符合搜索条件的索引项。 例如，如果您搜索特定的 SharePoint 或 OneDrive 的业务网站，但未找到搜索结果，则如果选择第二个导出选项来导出索引项和非索引项，则不会导出这些网站的未编制索引项目。 如果站点中的索引项符合搜索条件，则在导出索引项和非索引项时，将导出该站点中的所有未编制索引的项目。 下图描述了基于网站是否包含与搜索条件匹配的索引项的导出选项。
    
    ![根据网站是否包含与搜索条件匹配的索引项选择导出选项](media/94f78786-c6bb-42fb-96b3-7ea3998bcd39.png)

    
    a. 仅导出与搜索条件匹配的索引项。 不导出部分索引项。
    
    b. 如果站点中的索引项不符合搜索条件，则不会导出来自同一站点的部分索引项。 如果在搜索结果中返回来自站点的索引项，则导出该站点的部分索引项。 换句话说，仅导出来自包含与搜索条件匹配的项目的网站的部分索引项。
    
    c. 无论网站是否包含符合搜索条件的项，搜索中所有网站中的所有部分索引项都将导出。
    
    如果选择导出部分索引的项目，则部分索引邮箱项目将导出到单独的 PST 文件中，而不考虑**在"导出交换"内容下选择的选项。**

- 如果在搜索结果中返回部分索引的项目（因为部分索引项的其他属性与搜索条件匹配），则这些部分索引的项目将导出与常规搜索结果。 因此，如果选择同时导出索引项和部分索引项（通过**选择"所有项目"（包括具有无法识别格式、已加密的项或由于其他原因未编制索引**的项目）导出的部分索引项），则导出的部分索引项常规结果将列在结果.csv 报告中。 它们将不会在未编制索引的项目.csv 报表中列出。
    
 ### <a name="exporting-individual-messages-or-pst-files"></a>导出单个消息或 PST 文件
  
- 如果消息的文件路径名称超过 Windows 的最大字符限制，则文件路径名称将被截断。 但原始文件路径名称将列在清单和结果日志中。
    
- 如前所述，电子邮件搜索结果将导出到文件系统中的文件夹。 单个邮件的文件夹路径将复制用户邮箱中的文件夹路径。 例如，对于名为"ContosoCase101"的搜索，用户收件箱中的邮件将位于文件夹路径`~ContosoCase101\\<date of export\Exchange\user@contoso.com (Primary)\Top of Information Store\Inbox`中。 
    
- 如果选择在一个 PST 文件中导出包含单个文件夹中所有邮件的电子邮件，则 PST 文件夹的顶层中**包含"已删除邮件"** 文件夹**和"搜索文件夹"** 文件夹。 这些文件夹为空。 
  
 ### <a name="decrypting-rms-encrypted-messages"></a>解密 RMS 加密的邮件
  
- 如前所述，要在导出 RMS 加密邮件时解密它们，必须将搜索结果导出为单个消息。 如果将搜索结果导出到 PST 文件，则 RMS 加密的邮件将保持加密状态。
    
- 内容搜索中的 RMS 解密功能不会在导出搜索结果时解密使用 Office 365 邮件加密 （OME） 加密的邮件。 但是，如果使用 OME 加密的邮件由组织中的用户发送，则用户"已发送"文件夹中的邮件副本不会加密，导出后将可查看。 但是，如果组织中的用户收到使用 OME 加密的邮件，则在导出邮件后不会对其进行解密。 有关 OME 的详细信息，请参阅[Office 365 邮件加密](https://go.microsoft.com/fwlink/p/?linkid=844966)。
    
- 解密的邮件在**结果日志**报告中标识。 此报表包含一**个名为"解码状态"** 的列，此列**中的"解码"** 值标识已解密的消息。 
    
- 目前，此解密功能不包括来自 SharePoint 和 OneDrive 的加密内容。 导出邮件时，只会解密 RMS 加密的电子邮件。
    
- 如果 RMS 加密的电子邮件具有附件（如文档或其他电子邮件）也已加密，则仅解密顶级电子邮件。
    
- 您无法预览 RMS 加密的电子邮件。 要查看加密邮件，必须导出它。
    
- 如果需要阻止某人解密 RMS 加密的邮件，则必须创建自定义角色组（通过复制内置电子数据展示管理器角色组），然后从自定义角色组中删除 RMS 解密管理角色。 然后，将不想解密邮件的人员添加为自定义角色组的成员。
  
 ### <a name="filenames-of-exported-items"></a>导出项目的文件名
  
- 对于导出到本地计算机的电子邮件和站点文档的完整路径名称，有 260 个字符的限制（由操作系统强制实施）。 导出项目的完整路径名称包括项目的原始位置和下载搜索结果的本地计算机上的文件夹位置。 例如，如果指定在电子数据展示导出工具中将`C:\Users\Admin\Desktop\SearchResults`搜索结果下载到 ，则下载的电子邮件项的完整路径名称将是`C:\Users\Admin\Desktop\SearchResults\ContentSearch1\03.15.2017-1242PM\Exchange\sarad@contoso.com (Primary)\Top of Information Store\Inbox\Insider trading investigation.msg`。
    
    如果超过 260 个字符的限制，则项的完整路径名称将被截断。
    
  - 如果完整路径名称超过 260 个字符，则文件名将缩短以低于限制;请注意，截断的文件名（不包括文件扩展名）不会少于 8 个字符。
    
  - 如果完整路径名称在缩短文件名后仍然太长，则项目将从其当前位置移动到父文件夹。 如果路径名仍然太长，则重复该过程：缩短文件名，如有必要，再次移动到父文件夹。 此过程将重复，直到完整路径名低于 260 个字符的限制。
    
  - 如果已截断的完整路径名称已存在，则版本号将添加到文件名的末尾;如果已截断的完整路径名称已存在，则版本号将添加到文件名的末尾。例如， `statusmessage(2).msg`.
    
    为了帮助缓解此问题，请考虑将搜索结果下载到路径名称较短的位置;如果将搜索结果下载到具有短路径名称的位置，请考虑将搜索结果下载到该位置。例如，将搜索结果下载到名为`C:\Results`的文件夹，与将搜索结果下载到名为`C:\Users\Admin\Desktop\Results`的文件夹相比，导出项目的路径名称中添加的字符更少。
    
- 导出网站文档时，还可能修改文档的原始文件名。 对于已置于保留状态的 SharePoint 或 OneDrive 业务网站中删除的文档，尤其会发生这种情况。 删除保留站点上的文档后，已删除的文档将自动移动到网站的保留保留库（在网站置于保留状态时创建）。 将已删除的文档移动到保留保留库时，将随机生成的唯一 ID 追加到文档的原始文件名中。 例如，如果文档的文件名是`FY2017Budget.xlsx`，并且该文档稍后被删除并移动到保留保留库，则移动到保留保留库的文档的文件名将修改为 类似`FY2017Budget_DEAF727D-0478-4A7F-87DE-5487F033C81A2000-07-05T10-37-55.xlsx`。 如果保留保留库中的文档与内容搜索的查询匹配，并且导出该搜索的结果，则导出的文件具有已修改的文件名;如果导出的文件具有已修改的文件名，则导出的文件将具有"保留保留"库中的文档。在此示例中，导出文档的文件名将为`FY2017Budget_DEAF727D-0478-4A7F-87DE-5487F033C81A2000-07-05T10-37-55.xlsx`。
    
    修改保留站点上的文档（并且已启用站点中的文档库的版本控制）时，将自动在保留保留库中创建文件的副本。 在这种情况下，随机生成的唯一 ID 也会追加到复制到保留保留库的文档的文件名中。
    
    将文档或文档的文件名移动到保留保留库的原因就是防止文件名冲突。 有关在站点和保留保留库上放置保留的详细信息，请参阅[SharePoint Server 2016 中的就地保留概述。](https://support.office.com/article/5e400d68-cd51-444a-8fe6-e4df1d20aa95)
    
 ### <a name="miscellaneous"></a>雜項
  
- 所有搜索结果和导出报告都包含在与内容搜索同名的文件夹中。 导出的电子邮件位于名为**Exchange**的文件夹中。 文档位于名为**SharePoint**的文件夹中。 
    
- 将文档导出到本地计算机时，维护 SharePoint 和 OneDrive 业务站点上文档的文件系统元数据。 这意味着在导出文档时不会更改文档属性（如已创建日期和上次修改日期）。

- 如果搜索结果包含与搜索查询匹配的 SharePoint 中的列表项，则列表中的所有行都将导出与搜索查询匹配的项目。 这包括列表中的任何附件。 这样做是为了为在搜索结果中返回的列表项提供上下文。 另请注意，其他列表项和附件可能会导致导出项的计数与搜索结果的原始估计值不同。

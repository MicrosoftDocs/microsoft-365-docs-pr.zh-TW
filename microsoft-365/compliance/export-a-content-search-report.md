# <a name="export-a-content-search-report"></a>匯出內容搜尋報告

您可以只导出导出搜索结果时生成的相同报告，而不是从安全&合规性中心（以及与电子数据展示案例关联的内容搜索）中导出来自内容搜索的完整搜索结果集.
  
导出报表时，该报表将下载到与内容搜索同名的文件夹中，但该文件夹已附加*在 _Reports Only*中。 例如，如果内容搜索名为*ContosoCase0815，* 则报告将下载到名为*ContosoCase0815_报告仅*的文件夹。 有关报表中包含的文档列表，请参阅[报表中包含的内容。](#whats-included-in-the-report)

## <a name="before-you-begin"></a>開始之前

- 要导出内容搜索报告，您必须在安全&合规性中心中分配合规性搜索管理角色。 此角色分配给内置电子数据展示管理器和组织管理角色组。 默认情况下，它不分配给组织管理角色组。 有关详细信息，请参阅[分配电子数据展示权限](assign-ediscovery-permissions.md)。
    
- 导出报表时，数据在下载到本地计算机之前，会暂时存储在 Microsoft 云中的唯一 Windows Azure 存储区域中。 确保您的组织可以连接到 Azure 中的终结点，该终结点是**\*.blob.core.windows.net（** 通配符表示导出的唯一标识符）。 搜索结果数据在创建两周后从 Azure 存储区域中删除。 
    
- 用于导出搜索结果的计算机必须满足以下系统要求：
    
  - 32 位或 64 位版本的 Windows 7 和更高版本
    
  - 微软 .NET 框架 4.7
    
  - 支援的瀏覽器：
    
    - Microsoft Edge
    
      或
    
    - 微软互联网浏览器10及更高版本
    
    **注意：** Microsoft 不会为 ClickOnce 应用程序制造第三方扩展程序或加载项。 不支持使用不支持的浏览器导出搜索结果以及第三方扩展或加载项。 

- 如果内容搜索返回的结果的估计总大小超过 20&nbsp;TB，则导出报表将失败。 要成功导出报表，请尝试缩小范围并重新运行搜索，以便结果的估计大小小于 20&nbsp;TB。

- 导出内容搜索报告将计算同时运行的最大导出数和单个用户可以运行的最大导出数。 有关导出限制的详细信息，请参阅[导出内容搜索结果](export-search-results.md#export-limits)。

## <a name="generate-and-download-a-content-search-report"></a>生成和下载内容搜索报告

生成和下载内容搜索报告的步骤与实际导出搜索结果非常相似。
  
## <a name="step-1-generate-the-report-for-export"></a>步骤 1：生成导出报告

第一步是准备报告以下载到您的计算机导出。 报表时，报表文档将上载到 Microsoft 云中的 Azure 存储区域。
  
1. 移至 [https://protection.office.com](https://protection.office.com)。
    
2. 使用公司或學校帳戶登入 Office 365。
    
3. 在"安全&合规性中心"的左侧窗格中，****\>单击"**搜索内容搜索"。**
    
4. 在"**内容"搜索**页上，选择搜索。 
    
5. 在详细信息窗格中，**在"将报表导出到计算机"** 下，单击"**生成报表"。**
    
    > [!NOTE]
    > 如果搜索结果超过 7 天，系统将提示您更新搜索结果。 如果发生这种情况，请取消导出，单击所选搜索的详细信息窗格**中的"更新搜索结果"，** 然后在更新结果后再次启动报表导出。 
  
6. 在"**导出报表"** 页上，在"**从搜索中包括这些项目"** 下，选择以下选项之一：
    
    - 仅导出索引项
    
    - 导出索引和未编制索引的项目
    
    - 仅导出未编制索引的项目
    
    有关未编制索引的项目的详细信息，请参阅[内容搜索 中部分索引的项目。](partially-indexed-items-in-content-search.md)
    
7. 选择包括所有版本的 SharePoint 文档的搜索统计信息。 仅当搜索的内容源包括"共享点"或"企业网站的 OneDrive"时，才会显示此选项。
    
8. **单击"生成报告"。**
    
    搜索结果报告已准备下载，这意味着报告文档将上载到 Microsoft 云中的 Azure 存储区域。 当报表准备好可供下载时，在详细信息窗格中的"**将报表导出到计算机"** 下**将显示"下载报表"** 链接。 
    
> [!NOTE]
> 您还可以导出与电子数据展示案例关联的内容搜索的报告。 为此，请转到![](media/ebd260e4-3556-4fb0-b0bb-cc489773042c.gif)**电子数据展示，**\>**** 选择一个案例，**然后单击"编辑**编辑"图标 。 在"**搜索"** 页上，选择搜索，**然后单击"导出**![导出"搜索结果](media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png)\>**图标"导出报告"。** 
  
## <a name="step-2-download-the-report"></a>第 2 步：下载报告

下一步是将报告从 Azure 存储区域下载到本地计算机。
  
1. 在生成报表的搜索详细信息窗格中，**在"将报表导出到计算机"** 下，**单击"下载报表"。**
    
    **将显示"下载报告"** 页，其中包含有关报表的以下信息，直到下载到您的计算机。 
    
    - 要下载的项目数。
    
    - 将下载的项目的估计总大小。
    
    - 将导出索引还是未索引。 未编制索引的项目是具有可识别格式、已加密或由于其他原因未编制索引的项目。
    
    - 是否下载 SharePoint 文档的版本。
    
    - 报表导出过程的状态。 即使报表的编写未完成，您也可以开始下载报表。
    
2. **在"导出键"** 下，**单击"复制到剪贴板"。** 您将在步骤 5 中使用此密钥下载报告。
    
    > [!IMPORTANT]
    > 由于任何人都可以安装和启动电子数据展示导出工具，然后使用此密钥下载搜索报告，因此请务必采取预防措施来保护此密钥，就像保护密码或其他安全相关信息一样。 
  
3. **单击"下载报告"。**
    
4. 如果系统提示您安装**MicrosoftOffice 365 电子数据展示导出工具，** 请单击"**安装"。**
    
5. 在**电子数据展示导出工具**中，将步骤 2 中复制的导出密钥粘贴到相应的框中。
    
6. **单击"浏览"** 以指定要下载报表的位置。 
    
7. 按一下 [開始]**** 將搜尋結果下載至您的電腦。 
    
    **电子数据展示导出工具**显示有关导出过程的状态信息，包括要下载的剩余项目的数量（和大小）的估计值。 导出过程完成后，可以访问这些文件下载位置中的文件。 
    
> [!NOTE]
> 您可以下载与电子数据展示案例关联的内容搜索的报告。 为此，请转到![](media/ebd260e4-3556-4fb0-b0bb-cc489773042c.gif)**电子数据展示，**\>**** 选择一个案例，**然后单击"编辑**编辑"图标 。 在"**导出"** 页上，选择报表导出，然后在详细信息窗格中单击"**下载报表"。** 
  
## <a name="whats-included-in-the-report"></a>报告中包含的内容

生成和导出有关内容搜索结果的报告时，将下载以下文档：
  
- **导出摘要**- 包含导出摘要的 Excel 文档。 这包括搜索的内容源数、每个内容位置的搜索结果数、估计的项目数、要导出的项目的实际数量以及项目的估计和实际大小等信息将导出。 
    
    > [!NOTE]
    > 如果在导出报表时包含未编制索引的项目，则未编制索引的项目数将包含在估计搜索结果的总数中，以及"如果要导出搜索结果"中列出的已下载搜索结果的总数中。导出摘要报告。 换句话说，要下载的项目总数等于估计结果的总数和未编制索引的项目的总数。 
  
- **清单**- 一个清单文件（以 XML 格式），其中包含有关搜索结果中包含的每个项的信息。 
    
- **结果**- Excel 文档，其中包含包含有关将随搜索结果一起导出的每个索引项的信息的行。 对于电子邮件，结果日志包含有关每条消息的信息，包括： 
    
  - 邮件在源邮箱中的位置（包括邮件是在主邮箱还是存档邮箱中）。
    
  - 邮件的发送或接收日期。
    
  - 邮件中的"主题"行。
    
  - 邮件的发件人和收件人。
    
    对于来自 SharePoint 和 OneDrive 业务网站的文档，结果日志包含有关每个文档的信息，包括：
    
  - 文档的 URL。
    
  - 文档所在的网站集的 URL。
    
  - 文档上次修改的日期。
    
  - 文档的名称（位于结果日志的"主题"列中）。
    
    > [!NOTE]
    > **"结果"** 报表中的行数应等于将下载的搜索结果总数减去**未编制索引的项目**报表中列出的项目总数。 
  
- **未编制索引的项目**- 包含搜索结果中将包括的任何未编制索引项的信息的 Excel 文档。 如果在生成搜索结果报告时未包含未编制索引的项目，则此报告仍将下载，但将为空。

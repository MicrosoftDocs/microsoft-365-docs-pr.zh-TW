# <a name="create-and-apply-information-management-policies"></a>创建和应用信息管理策略

信息管理策略使您的组织能够控制保留内容的时间、审核人员如何处理内容以及向文档添加条形码或标签。 策略可帮助强制遵守法律和政府法规或内部业务流程。 作为管理员，您可以设置策略来控制如何跟踪文档以及保留文档的时间。
  
您可以创建信息管理策略可以在站点层次结构中的三个不同位置（从最宽到最窄）：
  
- 创建用于网站集中的多种内容类型的策略。
    
- 为网站内容类型创建策略。
    
- 为列表或库创建策略。
    
有关详细信息，请参阅[信息管理策略简介](intro-to-info-mgmt-policies.md)。
  
## <a name="create-a-policy-for-multiple-content-types-within-a-site-collection"></a>为网站集中的多种内容类型创建策略
<a name="__toc261001590"> </a>

2. 在"网站设置"页上，**在"网站集管理**\>**内容类型策略模板"** 下。 
  
![网站设置页面上的内容类型策略模板链接](media/26d3466a-23ec-443f-88f0-2aaff38e992b.png)

## <a name="create-a-policy-for-a-list-library-or-folder-location-based-retention-policy"></a>建立清單、文件庫或資料夾的原則 (位置保留原則)
<a name="__create_a_policy"> </a>

您可以定义仅适用于特定列表、库或文件夹的保留策略。 但是，如果以这种方式创建保留策略，则无法在其他列表、库、文件夹或网站上重用此策略，也不能将网站集策略应用于基于位置的策略。
  
如果要将单个保留策略应用于单个位置中的所有类型的内容，则很可能希望使用基于位置的保留。 在大多数其他情况下，您需要验证是否为所有内容类型指定了保留策略。
  
 每个子文件夹继承其父文件夹的保留策略，除非您选择中断继承并在子级别定义新的保留策略。 
  
如果要定义信息管理策略，而不是保留到列表或库，则需要为与该列表或库关联的每个列表内容类型定义信息管理策略。
  
 如果您在任何时候决定为列表或库从内容类型切换到基于位置的策略，则仅保留策略将用作基于位置的策略。 所有其他管理策略（审核、条形码和条形码）将从关联的内容类型继承。 
  
 通过停用"库"和"基于文件夹的保留"功能，可以禁用网站集的基于位置的策略。 这使网站集管理员能够确保其内容类型策略不会被列表管理员基于位置的策略覆盖。 
  
至少需要"管理列表"权限才能更改列表或库的信息管理策略设置。
  
1. 导航到要为其指定信息管理策略的列表或库。 
    
2. 在功能区上，**选择"库"** 或"**列表"**\>**选项卡""库设置"** 或"**列表设置"。**
    
    在"共享点**联机"中，单击"设置"，****然后单击"列表设置"****或"库设置"。** 
    
3. **在"权限和管理**\>**信息管理策略"下设置**。
  
![文档库设置页面上的信息管理策略链接](media/9fa6d366-6aab-49e1-a05c-898ac6f536e6.png)
  
4. 在"信息管理策略设置"页上，确保列表或库的保留源设置为"库"和"文件夹"。 
  
如果**内容类型**显示为源，请单击"**更改源"，****然后单击"库"和"文件夹"。** 您会收到内容类型保留策略将被忽略的提醒。 選擇 [確定]****。 
    
5. 在"编辑策略"页上，在**基于库的保留计划**下，输入要创建的策略的简要说明。 
    
6. **选择"添加保留阶段..."...**
    
     请注意，在"记录"下，您可以选择为记录定义不同的保留策略，为此选择"为记录定义不同的保留阶段"选项。 
    
7. 在"阶段属性"对话框中，选择保留期选项以指定文档或项目何时设置为过期。 執行下列其中一項動作：
    
  - 要基于日期属性设置到期日期，在**事件**\>**此阶段下基于项目上的日期属性，** 然后选择文档或项目操作（例如，创建或修改）和此操作后的时间增量（例如，您希望项目过期时的天数、月数或年数。 
    
  - 要使用自定义保留公式来确定过期，**请选择"按此服务器上安装的自定义保留公式设置"。** 
    
    > [!NOTE]
    >  仅当管理员设置了自定义公式时，此选项才可用。 
  
  - **在"操作"** 下，指定文档或项目过期时要执行的操作。 要使文档或项目执行特定操作（如删除），请从列表中选择一个操作。 
    
8. 仅当为已与其关联的工作流的列表、库或内容类型定义策略时，"**开始工作流"** 选项才可用。 然后，您可以选择工作流。 
    
9. **在"重复"** 下，**选择"重复此阶段的操作..."** 并输入您希望操作再次发生的频率。 
    
    > [!NOTE]
    >  仅当可以选择的操作可以重复时，此选项才可用。 例如，您不能**为"永久删除"** 操作设置重复。 
  
10. 選擇 [確定]****。
    
## <a name="apply-a-site-collection-policy-to-a-content-type"></a>将网站集策略应用于内容类型
<a name="__apply_a_site"> </a>

如果已为您的网站创建信息管理策略作为网站收集策略，则可以将其中一个策略应用于内容类型。 通过执行此操作，可以将相同的策略应用于网站集中不共享相同父内容类型的多个内容类型。
  
 如果要将策略应用于网站集中的多个内容类型，并且配置了托管元数据服务，则可以使用"内容类型发布"将信息管理策略发布到多个网站集。 有关详细信息，请参阅[跨网站集应用策略](#apply-a-policy-across-site-collections)一节。 
  
1. 导航到包含要应用策略的内容类型的列表或库。
    
2. 在功能区上，**选择"库"** 或"**列表"**\>**选项卡""库设置"** 或"**列表设置"。**
    
    在"共享点**联机"中，单击"设置"，****然后单击"列表设置"****或"库设置"。** 
    
3. **在"权限和管理**\>**信息管理策略"下设置**。
  
![文档库设置页面上的信息管理策略链接](media/9fa6d366-6aab-49e1-a05c-898ac6f536e6.png)
  
4. 验证策略源是否**设置为"内容类型"，****并在"内容类型策略"** 下选择要应用策略的内容类型。 
    
5. 在"**指定策略**\>**使用网站集策略"** 下，然后从列表中选择要应用的策略。 
    
    > [!NOTE]
    >  如果"**使用网站集策略"** 选项不可用，则尚未为网站集定义任何网站集策略。 
  
6. 選擇 [確定]****。
    
     如果您使用的列表或库支持多种内容类型的管理，则可以**在"内容类型"** 下选择要为其指定信息管理策略的内容类型。 这将直接带您到上面的步骤 5。 
    
## <a name="apply-a-policy-across-site-collections"></a>跨网站集应用策略
<a name="__toc260646789"> </a>

通过使用托管元数据服务应用程序设置内容类型发布，跨网站集共享内容类型。 内容类型发布可帮助您在网站中一致地管理内容和元数据，因为内容类型可以集中创建和更新，并且更新可以发布到多个订阅网站集或 Web 应用程序。
  
## <a name="create-a-template-from-an-existing-policy-to-use-across-site-collections"></a>从现有策略创建模板，以便跨网站集使用
<a name="__toc262125409"> </a>

您可以定义信息管理策略，然后从其中创建模板，以便根据需要跨多个网站集使用。 如果要备份信息策略，也可以将其用作使用内容类型发布跨网站集应用策略的替代方法，则可以使用此方法。 通过将策略从一个网站集导出，然后将其导入到保存的位置或另一个网站集，可以创建策略的模板或备份。
  
> [!IMPORTANT]
>  如果使用导出/导入功能来制作一组策略模板，请记住策略 .xml 文件中存在唯一标识符。 因此，如果不更改此唯一标识符，则无法多次将该策略导入网站。 
  
### <a name="export-a-policy"></a>导出策略
<a name="__toc260646790"> </a>

1. 在网站集主页上，选择取代"网站设置"**的"设置""**![小设置"](media/a47a06c3-83fb-46b2-9c52-d1bad63e3e60.png)**** 网站设置\>.
    
    在 SharePoint 组连接的站点中，单击"**设置"，****单击"网站内容"，****然后单击"网站设置"。** 
    
2. 在"网站设置"页上，**在"网站集管理**\>**内容类型策略模板"** 下。

### <a name="import-a-policy-to-a-different-site-collection"></a>将策略导入其他网站集
<a name="__toc260646791"> </a>

通过导入信息管理策略，您可以将其应用于任何给定网站集中的网站或列表级别的多个内容类型。 这样做的好处是双重的：您不必在每个内容类型上重新定义和应用策略，并且只需在一个位置更改策略，即可更轻松地管理策略修改。
  
1. 在要应用策略的网站集的主页上，选择取代"网站设置"**的"设置""**![小设置"](media/a47a06c3-83fb-46b2-9c52-d1bad63e3e60.png)**** 网站设置\>.
    
    在 SharePoint 组连接的站点中，单击"**设置"，****单击"网站内容"，****然后单击"网站设置"。** 
    
2. 在"网站设置"页上，**在"网站集管理**\>**内容类型策略模板"** 下。
    
3. 在"策略"\>页上**导入**\>**浏览**以查找策略的 XML 文件。 
    
4. 选择策略已保存在\>其中的 XML 文件**打开**。 
    
5. 在"导入网站集策略"页上\>**导入**以将策略添加到网站集。 
    
导入的策略现在可以应用于网站或列表级别的一个或多个内容类型。 
  
信息管理策略使您的组织能够控制保留内容的时间、审核人员如何处理内容以及向文档添加条形码或标签。 策略可帮助强制遵守法律和政府法规或内部业务流程。 作为管理员，您可以设置策略来控制如何跟踪文档以及保留文档的时间。

您可以创建信息管理策略可以在站点层次结构中的三个不同位置（从最宽到最窄）：
- 创建用于网站集中的多种内容类型的策略。
- 为网站内容类型创建策略。
- 为列表或库创建策略。

有关详细信息，请参阅[信息管理策略简介](intro-to-info-mgmt-policies.md)。

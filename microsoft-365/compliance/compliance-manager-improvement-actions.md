# <a name="assign-and-complete-improvement-actions-in-compliance-manager"></a>在合規性管理員中指派及完成改進動作

**本文內容：** 本文說明如何使用改進動作來 **管理您的合規性工作流程** 。 瞭解如何指派執行與測試、**管理更新** 及匯出 **報表** 的 **改進動作**。

## <a name="manage-compliance-workflows-with-improvement-actions"></a>使用改進動作管理合規性工作流程

改進相容性活動的改進動作。 每個改進動作都提供詳細的執行指導，以協助您與資料保護法規和標準相符。 動作可指派給組織中的使用者，以執行實施和測試工作。 您也可以在動作中儲存檔、記事及記錄狀態更新。

您的所有改進動作都會列于 [改進動作] 頁面上。 深入瞭解如何 [查看您的改善動作](compliance-manager-setup.md#improvement-actions-page)。

## <a name="improvement-actions-details-page"></a>改進動作詳細資料頁面

每個改進動作都有詳細資料頁面顯示其目前的狀態、相關的標準與法規需求，以及建議的實施指引。 [技術動作](compliance-score-calculation.md#technical-and-non-technical-actions) 包括 **立即啟動** 連結，它會帶您前往適當的實施方案。 您可以將實施和測試檔案直接附加到改進動作的 [詳細資料] 頁面。

若要查看改進動作的詳細資料頁面：

1. 移至您的 [改進動作] 頁面。
2. 選取您想要的「改進」動作所在的列，這會開啟其詳細資料頁面。

您可以在畫面的右上角選取向上或向下箭號，輕易地查看清單中的下一個或上一個改善動作。 如果您已在 [改進動作] 頁面上篩選清單，向上或向下移動會移到該篩選清單中的下一個專案。

## <a name="assign-improvement-actions"></a>指派改進動作

若要開始執行改進動作，您可以自行執行工作或指派給其他使用者。 指派的人員可能是：

- 企業原則擁有者
- IT 實施者
- 另一個員工負責執行任務

一旦您識別適當的受託人，請確定他們擁有足夠的 [合規性管理員角色](compliance-manager-setup.md#set-user-permissions-and-assign-roles) ，才能執行工作。 請遵循下列步驟來指派「改進」動作：

1. 從 [改進動作詳細資料] 頁面中，選取畫面左上方旁邊的 [ **編輯狀態** ]。

2. 在 [編輯狀態] 飛入窗格中，選取 [ **指派物件** ] 方塊，以顯示 **建議的人員** 清單。 您可以從清單中選取使用者，或輸入要指派給其的人員的電子郵件地址。

3. 選取 [ **儲存並關閉**]。 指派的使用者會收到一封電子郵件，說明已將改進動作指派給他們，並直接連結到 [改進] 動作。  (注意：在指派動作給美國政府團體 (GCC) 高的客戶時，將不會收到電子郵件。 ) 

然後，已指派的使用者可以執行建議的動作。

## <a name="perform-work-and-store-documentation"></a>執行工作和儲存庫檔

您可以將與執行和測試相關的檔案和附注直接上傳至 [ **附注與檔** ] 區段。 此環境是一種安全且集中的存放庫，可協助您示範控制的滿意度，以符合法規標準和規定。 任何具有唯讀存取權的使用者都可以讀取此區段中的內容。 只有具有編輯許可權的使用者可以上傳和下載檔案，並輸入或編輯附注。

[ **附注與檔** ] 區段包含上傳的檔、執行附注、測試附注及其他附注的欄位。

#### <a name="uploaded-documents"></a>上傳檔

- 選取 [ **管理檔** ] 以上傳任何相關檔案。
- 當 [管理檔] 飛出窗格開啟時，選取 [ **新增** 檔]，然後從您的系統選取檔案。 公認的檔案類型：
    - 檔 ( .doc、.xls、.ppt、.txt、.pdf) 
    - 影像 ( .jpg，.png) 
    - 影片 ( mkv) 
    - 壓縮檔 ( .zip、rar) 
- 當您的檔案在窗格中解析時，請選取 [ **關閉**]，這會自動儲存檔附件。 接著，您將會看到 [ **上傳** 的檔] 下方所列的檔案。
- 若要下載或刪除檔，請從檔案清單下選取 [ **管理檔** ]。 在飛入窗格上，選取檔列加以反白顯示，然後選取 [ **下載** ] 或 [ **刪除**]。

#### <a name="implementation-notes-test-notes-and-additional-notes"></a>執行附注、測試備註和其他附注

- 若要在這三個欄位中新增附注，請選取 [ **編輯執行附注** ] 下的任何欄位。
- 當快顯視窗開啟時，請在 [文字] 欄位中輸入記事，然後選取 [ **儲存並關閉**]。
- 若要編輯附注，請選取 [ **編輯執行附注**]，進行編輯，然後選取 [ **儲存並關閉**]。

[附注] 欄位中沒有字元限制。 建議您保留附注摘要，讓您能輕鬆地從 [改進動作詳細資料] 頁面中進行查看及編輯。

## <a name="change-improvement-action-status"></a>變更改進動作狀態

您可以記錄每個改進動作的「實施」狀態和「日期」和「測試狀態」和「日期」。 任何具有編輯許可權的使用者都可以編輯 [ **實施** ] 和 [ **測試狀態** ] 欄位，而不是僅由指派的人員編輯。

若要編輯改進動作的狀態，請選取 [詳細資料] 頁面左上部的 [ **編輯狀態** ]。 以下是可用的欄位及狀態選項：

- **實施狀態**
    - **尚未執行** -尚未執行的動作
    - 已 **執行** 的動作
    - **替代執行** -如果您使用其他協力廠商工具，或採取其他不包含在 Microsoft 建議中的動作，請選取此選項。
    - **計畫的** 動作已計畫執行
    - **超出範圍** –動作與您的組織不相關，也不會影響您的分數
- **實施日期**：當實施狀態為「已實施」或「替代實施」時，可供選取
- **測試狀態**：當實施狀態為「已實施」或「替代實施」時，可供選取：
    - **未評估–未** 測試的動作
    - **傳遞** 的執行已由 assessor 驗證
    - **失敗的低風險** -測試失敗、低風險
    - **失敗之中度風險** -測試失敗，中度風險
    - **失敗的高風險** –測試失敗、高風險
    - **超出範圍** –行動超出評估的範圍，不會影響您的分數
- **測試日期**：切換到 [行事曆] 彈出式功能表以選取日期

不同群組的常見動作同步處理。 當相同群組中的兩個不同評估共用您所管理的改善動作時，您對動作的執行詳細資料或狀態所做的任何更新，都將自動同步處理至群組中任何其他評估中的相同動作。 這項同步處理可讓您執行一個改進動作，並符合多種法規的需求。

## <a name="assign-improvement-action-to-assessor-for-completion"></a>將改進動作指派給 assessor 以完成

完成工作、執行測試及上傳證據後，下一步是將「改進」動作指派給 assessor 以進行驗證。 Assessor 會驗證工作並檢查檔，並選取適當的測試狀態。

**如果測試狀態設定為 [已傳遞]**：動作已完成，而且點數會顯示達到的最大點數。 然後將點數計入您的整體合規性分數。

**如果測試狀態設定為 "Failed"**：動作不符合需求，assessor 可將其指派回適當的使用者進行其他工作。

## <a name="accepting-updates-to-improvement-actions"></a>接受改進動作的更新

當有改進動作的更新可供使用時，您會在其名稱旁看到一則通知。 您可以接受更新，也可以稍後將其推遲。

#### <a name="what-causes-an-update"></a>什麼導致更新

當相關的變更與計分、自動化或範圍有關時，就會發生更新。 變更可能包含根據法規變更而改進動作的新指導方針，或可能因為產品變更。 只有組織所管理的改進動作會收到更新通知。

#### <a name="where-youll-see-assessment-update-notifications"></a>您將在何處看到評估更新通知

更新改進動作後，您會在 [改進動作] 頁面上的其名稱旁邊，以及其相關評估的 [詳細資料] 頁面上，看到一個 **擱置的更新** 標籤。

移至 [改進動作的詳細資料] 頁面，然後選取上方橫幅中的 [ **檢查更新** ] 按鈕，以查看變更的詳細資料，並接受或推遲更新。

#### <a name="review-update-to-accept-or-defer"></a>審閱更新以接受或延期

從 [改進動作詳細資料] 頁面中選取 [ **複查更新** ] 之後，畫面右側會顯示一個彈出窗格。 飛入窗格提供有關更新的重要詳細資料，例如評估影響的評估，以及分數和範圍的變更。

選取 [ **接受更新** ]，以接受對「改進」動作的所有變更。 **可接受的變更是永久** 的。

> [!NOTE]
> 當您接受某項動作的更新時，您也會接受此動作的任何其他版本或實例的更新。 更新將會在整個租使用者的技術動作中傳播，而且會傳播非技術動作的整個群組。

如果您選取 [ **取消**]，更新將不會套用到 [改進] 動作。 不過，您會繼續看到 **待處理的更新** 通知，直到您接受更新為止。

**為什麼建議您接受更新**

接受更新有助於確保您具備使用解決方案的最新指導方針，並採取適當的改進動作，以協助您滿足目前的認證需求。

**為何您可能想要推遲更新**

如果您正在完成包括「改進」動作的評估，您可能想要確定已完成的工作，再接受更新。 您可以在 [審閱更新] 浮出窗格上選取 [ **取消** ]，以推遲更新的時間。

## <a name="export-a-report"></a>匯出報告

在螢幕的左上角選取 [ **匯出** ]，下載包含所有改進動作的 Excel 工作表，以及 [改進動作] 頁面上顯示的篩選類別。

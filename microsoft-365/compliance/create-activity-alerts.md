# <a name="create-activity-alerts"></a>建立活動警示

您可以建立活動警示，當使用者在 Office 365 中執行特定活動時，會傳送電子郵件通知。 活動警示類似于在審計記錄檔中搜尋事件，但當您已建立警示之活動的事件時，就會傳送電子郵件。

 **為什麼要使用活動警示，而不是搜尋審計記錄？** 特定使用者可能會執行某些類型的活動或活動，但您實際想知道。 您可以使用活動提醒，在使用者執行這些活動時，將電子郵件訊息傳送 Microsoft 365 給您，而不必記得搜尋審計記錄。 例如，您可以建立活動警示，當使用者刪除 SharePoint 中的檔案時，也可以建立警示，當使用者從其信箱中永久刪除郵件時，通知您。 傳送給您的電子郵件通知，包含已執行活動的相關資訊，以及執行它的使用者。

> [!NOTE]
> 活動警示會被取代。 建議您在安全性與合規性中心中開始使用警示原則，而不是建立新的活動提醒。 警示原則提供額外的功能，例如建立在任何使用者執行指定活動時觸發警示的報警原則，以及在安全性與合規性中心的「 **查看警示** 」頁面上顯示警示。 如需詳細資訊，請參閱 [警示原則](alert-policies.md)。

## <a name="confirm-roles-and-configure-audit-logging"></a>確認角色及設定審核記錄

- 您必須在安全性 & 合規性中心內指派組織設定角色，以管理活動警示。 根據預設，此角色會指派給合規性管理員和組織管理角色群組。 如需新增成員至角色群組的詳細資訊，請參閱 [授與使用者存取安全性 & 規範中心](../security/office-365-security/grant-access-to-the-security-and-compliance-center.md)。

- 您 (或另一個系統管理員) 必須先開啟組織的審計記錄，才能開始使用活動警示。 若要這麼做，請按一下 [**活動提醒**] 頁面上的 [**開始錄製使用者和系統管理員活動**]。  (如果您沒有看到此連結，則表示您的組織已開啟審核。 ) 您也可以在 [安全性 & 規範中心] 中的「 **審核記錄搜尋** 」頁面上開啟審核 (移至 **搜尋** \> **審核記錄搜尋**) 。 您只需要為組織執行一次此作業。

- 您可以在審核記錄中為您可以搜尋的相同活動建立警示。 請參閱 [詳細資訊](#more-information) 一節，以取得常見案例的清單 (和監視您可以為其建立提醒) 的特定活動。

- 您可以使用 [安全性 & 規範中心] 中的 [ **活動提醒** ] 頁面，為組織通訊錄中所列的使用者所執行的活動建立警示。 您無法使用此頁面來建立未列在通訊錄中的外部使用者執行的活動提醒。

## <a name="create-an-activity-alert"></a>建立活動警示

1. 移至<https://compliance.microsoft.com/managealerts>。

2. 使用您的公司或學校帳戶登入。

3. 在 [ **活動提醒** ] 頁面上，按一下 [ ![ 新增圖示] [ ](../media/8ee52980-254b-440b-99a2-18d068de62d3.gif) **新增**]。

   隨即會顯示建立活動提醒的飛出頁面。


    ![建立活動警示](../media/53888bd5-9fa2-4398-8ccc-1a9dc72517ac.png)

4. 完成下欄欄位以建立活動警示：

    a. **Name** -輸入警示的名稱。 警示名稱在您的組織中必須是唯一的。

    b. **Description** (Optional) -描述警示，例如要追蹤的活動和使用者，以及傳送電子郵件通知的使用者。 描述提供快速而簡單的方法，說明向其他系統管理員報警的目的。

    c. **警示類型** -請確定已選取 [ **自訂** ] 選項。

    d. 當按-按一下 [**傳送此提醒時傳送此提醒**]，然後設定這兩個欄位時 **傳送此警示**：

    - **活動** -按一下下拉式清單，以顯示您可以為其建立警示的活動。 這是您在搜尋審核記錄檔時所顯示的相同活動清單。 您可以選取一或多個特定活動，也可以按一下活動組名稱，以選取群組中的所有活動。 如需這些活動的說明，請參閱 [搜尋審核記錄](search-the-audit-log-in-security-and-compliance.md#audited-activities)檔中的「已審核的活動」一節。 當使用者執行您已新增至提醒的任何活動時，會傳送電子郵件通知。

     - **使用者** -按一下此方塊，然後選取一或多個使用者。 如果此方塊中的使用者執行您新增至 [ **活動** ] 方塊的活動，將會傳送警示。 當您組織中的任何使用者執行警示所指定的活動時，請將 [ **使用者** ] 方塊保留空白以傳送警示。

    e. **傳送此提醒至** -按一下 [ **傳送此提醒**]，然後 **在 [收** 件者] 方塊中輸入名稱，以新增當使用者在 [ **使用者** ] 方塊中指定的使用者 () 執行 [ **活動** ] 方塊) 中所指定的活動 (時，將會收到電子郵件通知的使用者。 請注意，根據預設，您會新增到收件者清單。 您可以從這個清單中移除您的名稱。

5. 按一下 [ **儲存** ] 來建立警示。

    新的警示會顯示在 [ **活動提醒** ] 頁面上的清單中。

    ![警示清單會顯示在 [活動提醒] 頁面上。](../media/02b774f2-1719-41de-bbc9-5e5b7576f335.png)

    警示的狀態設定為 [ **開啟**]。 請注意，也會列出接收到警示時，收到電子郵件通知的收件者。

## <a name="turn-off-an-activity-alert"></a>關閉活動警示

您可以關閉活動警示，以避免傳送電子郵件通知。 當您關閉活動通知之後，它仍會顯示在您組織的活動警示清單中，您仍然可以查看其屬性。

1. 移至<https://compliance.microsoft.com/managealerts>。

2. 使用您的公司或學校帳戶登入。

3. 在您組織的活動警示清單中，按一下您想關閉的警示。

4. 在 [ **編輯提醒** ] 頁面上，按一下 **[開啟切換** ] 參數，將狀態變更為 [ **關閉**]，然後按一下 [ **儲存**]。

    [ **活動提醒** ] 頁面上警示的狀態會設為 [ **關閉**]。

若要將活動提醒重新開啟，只需重複這些步驟，然後按一下 [ **關閉** ] 切換參數，即可將狀態變更為 [ **開啟**]。

- 您也可以在安全性 & 規範中心 PowerShell 中使用 **New-ActivityAlert** 和 **Set-ActivityAlert** Cmdlet，以建立及編輯活動提醒。 如果您使用這些 Cmdlet 來建立或編輯活動警示，請記住下列事項：

# <a name="digital-production-partnership-dpp-united-kingdom"></a>數位生產合作關係 (DPP) 英國

## <a name="about-the-dpp"></a>關於 DPP

廣播者會開始對他們網站、 IT 基礎結構和系統越來越常見網路攻擊。 回應這些威脅[數位生產合作關係](https://www.thedpp.com/)(DPP) 建立合作關係的[**北美地區廣播者關聯**](https://nabanet.com/)(NABA) 來開發[的供應商的廣播者網路的安全性需求](https://nabanet.com/wp-content/uploads/2017/08/NABA_DPP_CyberSecurity_Requirements_3.pdf)。 資訊安全性人員從英國廣播者有背書這些稱為的最小 cybersecurity 需求。  
  
DPP 然後合作廣播者及供應商安全性專家以建立自我評估存貨 」，[已確認以安全性 Programme 廣播檢查清單](https://dpp-assets.s3.amazonaws.com/wp-content/uploads/2017/10/CTS_BroadcastChecklist.xlsx)，可讓供應商] 來示範廣播者以其部署 cybersecurity 最佳作法。 這項工作點擊建立正式[DPP 認可到安全性 Programme](https://www.thedpp.com/tech/security/committed-to-security/)年 10 月 2017，以啟動具有兩個不同的標記： 一個用於廣播，一個用於實際執行環境。

## <a name="microsoft-and-the-dpp"></a>Microsoft 和 DPP

Microsoft Azure 具有已獎勵 DPP 認可至廣播的安全性標記後完成數位實際執行的合作關係 (DPP) 自我評估問卷，*認可到安全性 Programme 廣播檢查清單*。 它需要記載一組的文件和測試、 驗證及安全性控制的最佳做法。 [Eurofins 數位測試](https://www.eurofins-digitaltesting.com/)、 品質保證專家和 DPP 成員檢閱在 Microsoft 回應登出並提交至最終驗證 DPP。  
  
這表示廣播者和其他使用 Azure 的媒體客戶可有信心，其健全的安全性和恢復服務，有助於滿足唯一廣播產業中，從傳輸至程式開發。

# <a name="apply-information-rights-management-irm-to-a-list-or-library"></a>將資訊版權管理 (IRM) 套用至清單或文件庫

您可以使用資訊版權管理 (IRM) ，以協助控制和保護從清單或文件庫中下載的檔案。 只有 Microsoft global cloud 支援此功能。 在全國雲端部署中 SharePoint 清單和文件庫不支援 IRM。
  
## <a name="administrator-preparations-before-applying-irm"></a>套用 IRM 之前的系統管理員準備工作

- Azure Rights Management service 從 Azure 資訊保護 (Azure RMS) ，以及內部部署對等 Active Directory Rights Management Services (AD RMS) ，支援網站的資訊版權管理。 不需要個別或其他安裝。

- 將 IRM 套用至清單或文件庫之前，您必須為網站啟用 IRM。 您將需要系統管理員許可權才能啟用 IRM。

- 若要將 IRM 套用至清單或文件庫，您必須具有該清單或文件庫的系統管理員許可權。

- 如果您使用的是線上 SharePoint，您的使用者可能會在下載較大的受 IRM 保護的檔案時遇到超時狀況。 若要避免超時，請使用 Office 程式來套用 IRM 保護，並在不使用 IRM 的 SharePoint 文件庫中儲存較大的檔案。

> [!NOTE]
> 如果您使用 SharePoint Server 2013，則伺服器管理員必須在所有的前端網頁伺服器上，針對組織中的人員要使用 IRM 來保護的每一種檔案類型，安裝保護程式。

7. 若要將其他限制套用至此清單或文件庫中的檔，請選取 [ **顯示選項**]，然後執行下列其中一項動作：

## <a name="what-is-information-rights-management"></a>何謂「資訊版權管理」？
<a name="__toc256598175"> </a>

資訊版權管理 (IRM) 可讓您限制使用者可對從清單或文件庫下載之檔案採取的動作。 IRM 會加密下載的檔案，並限制能夠解密這些檔案的使用者及程式。 IRM 也會限制能夠讀取檔案的使用者權限，如此一來，使用者就無法進行像是列印檔案複本或從檔案複製文字之類的動作。
  
您可以在清單或文件庫上使用 IRM，以限制機密內容的散播。 例如，如果您要建立文件庫，以與選取的行銷代表共用即將推出之產品的資訊，您可以使用 IRM 來防止這些個人與公司中的其他員工共用此內容。
  
在網站上，您可以將 IRM 套用到整個清單或文件庫，而不是個別檔案。 這可讓您更容易確定整個檔或檔案集的一致性層級。 因此，IRM 可協助您的組織強制實施公司原則，以管理機密或專有資訊的使用和傳播。
  
> [!NOTE]
> 此頁面上有關資訊版權管理的資訊會取代任何 Microsoft SharePoint Server 2013 中參照「資訊版權管理」的字詞，並 SharePoint 伺服器2016授權條款協定。 
  
### <a name="how-irm-can-help-protect-content"></a>IRM 如何協助保護內容
<a name="__toc256598176"> </a>

IRM 協助以下列方式保護限制內容：
  
- 協助防止授權的檢視器複製、修改、列印、傳真或複製並貼上未經授權使用的內容
    
- 協助防止授權的檢視器使用 Microsoft Windows 中的 [列印畫面] 功能來複製內容
    
- 協助防止未經授權的檢視器在從伺服器下載後以電子郵件傳送內容來查看內容
    
- 在指定的時間內限制存取內容，使用者必須先確認其認證，然後再次下載內容。
    
- 協助強制執行公司原則，以控制組織內的內容的使用和散播
    
### <a name="how-irm-cannot-help-protect-content"></a>IRM 無法如何協助保護內容
<a name="__toc256598177"> </a>

IRM 無法保護下列專案中的限制內容：
  
- 惡意程式（如特洛伊馬、按鍵記錄檔和某些類型的間諜軟體）的擦除、盜竊、捕獲或傳輸
    
- 因電腦病毒的動作而遺失或損毀
    
- 手動複製或重新鍵入螢幕上顯示的內容
    
- 螢幕上顯示內容的數位或膠片攝影
    
- 使用協力廠商的螢幕捕捉程式進行複製
    
- 透過使用協力廠商的畫面捕獲程式或複製和貼上動作) ，將內容中繼資料複製 (欄值
  
## <a name="how-irm-works-for-lists-and-libraries"></a>IRM 對於清單和文件庫的運作方式
<a name="__toc256598178"> </a>

IRM 保護適用于清單或文件庫層級的檔案。 針對文件庫啟用 IRM 時，版權管理會套用至該文件庫中的所有檔案。 當您為清單啟用 IRM 時，版權管理只會套用至清單專案附加的檔案，而非實際的清單專案。
  
當使用者在已啟用 IRM 的清單或文件庫中下載檔案時，這些檔案會進行加密，只有經過授權的人員才能加以查看。 每個版權管理檔案也包含發行授權，對查看檔案的人員有限制。 一般限制包括將檔案設為唯讀、停用文字的複製、防止使用者儲存本機副本，以及防止使用者列印檔案。 可讀取 IRM 支援之檔案類型的用戶端程式會使用版權管理檔案中的發行授權，以強制執行這些限制。 這是版權管理檔案在從伺服器下載之後，如何保留其保護。
  
從清單或文件庫下載檔案時，套用至該檔案的限制類型，取決於包含檔案之網站上的個別使用者許可權。 下表說明網站許可權如何對應至 IRM 許可權。
  
|**權限**|**IRM 許可權**|
|:-----|:-----|
|管理許可權、管理網站|由用戶端) 程式所定義 (**完全控制**：此許可權通常可讓使用者讀取、編輯、複製、儲存及修改受版權管理內容的許可權。|
|編輯專案、管理清單、新增和自訂頁面|**編輯**、 **複製** 及 **儲存**：只有在清單或文件庫的 [資訊版權管理設定] 頁面上選取 [ **允許使用者列印** 檔] 核取方塊時，使用者才可以列印檔案。|
|檢視項目|已 **閱讀**：使用者可以讀取檔，但無法複製或修改其內容。 只有在清單或文件庫的 [資訊版權管理設定] 頁面上選取 [ **允許使用者列印檔案** ] 核取方塊時，使用者才可以列印。|
|其他|其他許可權都不會直接對應至 IRM 許可權。|
   
當您為 SharePoint Server 2013 中的清單或文件庫啟用 IRM 時，您只能保護在所有前端網頁伺服器上安裝保護裝置的清單或文件庫中的檔案類型。 保護裝置是一種程式，可控制特定檔案格式之受版權管理之檔案的加密和解密。 SharePoint 包含下列檔案類型的保護器：
  
- Microsoft Office InfoPath 表單
    
- 下列 Microsoft Office 程式的97-2003 檔案格式： Word、Excel 及 PowerPoint
    
- 下列 Microsoft Office 程式的 Office Open XML 格式： Word、Excel 及 PowerPoint
    
- XML 紙張規格 (XPS) 格式
    
如果您的組織打算使用 IRM 來保護其他檔案類型，除了上述所列之外，您的伺服器管理員必須安裝其他檔案格式的保護程式。

# <a name="ediscovery-solutions-in-microsoft-365"></a>Microsoft 365 中的 eDiscovery 解決方案

電子搜索 (eDiscovery) 是識別與傳遞電子資訊的程序，而且此類資訊可在法律案件中做為呈堂證據。 您可以使用 Microsoft 365 中的 eDiscovery 工具來搜尋 Exchange Online、商務用 OneDrive、SharePoint 線上、Microsoft Teams、Microsoft 365 群組和 Yammer 小組中的內容。 您可以在相同的 eDiscovery 搜尋中搜尋信箱和網站，然後匯出搜尋結果。 您可以使用核心 eDiscovery 案例來識別、保留及匯出信箱和網站中的內容。 如果您的組織有 Office 365 E5 或 Microsoft 365 E5 訂閱 (或相關的 E5 附加元件訂閱) ，您可以在 Advanced eDiscovery 中使用功能豐富的 Microsoft 365 解決方案，進一步管理保管人及分析內容。

## <a name="ediscovery-solutions"></a>eDiscovery 解決方案

Microsoft 365 提供三種 eDiscovery 解決方案：「內容搜尋」、「核心 eDiscovery」和「Advanced eDiscovery」。

![Microsoft 365 eDiscovery 工具的主要功能](..\media\m365-ediscovery-solution-graphic.png)

- **內容搜尋**。 使用內容搜尋工具，在 Microsoft 365 資料來源中搜尋內容，然後將搜尋結果匯出至本機電腦。

- **核心 eDiscovery**。 在內容搜尋的基本搜尋和匯出功能上，核心 eDiscovery 組建可讓您建立 eDiscovery 案例，以及只能存取其成員之案例的 eDiscovery 管理員。 核心 eDiscovery 可讓您將搜尋和匯出與案例產生關聯，並可讓您在與案例相關的內容位置上放置 eDiscovery 暫止功能。

- **Advanced eDiscovery**。 Advanced eDiscovery 工具是以核心 eDiscovery 的現有案例管理、保留、搜尋和匯出功能為基礎。 Advanced eDiscovery 提供端對端工作流程來識別、保留、收集、審閱、分析及匯出回應組織內部和外部調查的內容。 它可讓法律小組管理保管人和法律封存通知工作流程，與案例中的保管人進行通訊。 它可讓您從 live service 收集資料並將資料複製到審查集，當您可以篩選、搜尋及標記內容以挑選內容，以挑選不相關的內容，讓您的工作流程能夠識別並關注最相關的內容。 Advanced eDiscovery 提供分析和機器學習式的預測編碼模型，以進一步縮小對最相關內容的調查範圍。

以下是每個 eDiscovery 功能的描述。

- **搜尋內容**。 搜尋儲存在 Exchange 信箱中的內容、商務帳戶的一個磁碟機、SharePoint 網站、Microsoft Teams、Microsoft 365 群組，以及 Yammer Teams。 這包括由其他 Microsoft 365 應用程式所產生的內容，這些應用程式會將資料儲存在信箱和網站中。

- **關鍵字查詢和搜尋條件**。 建立 KQL 關鍵字搜尋查詢，以搜尋符合查詢準則的內容。 您也可以加入條件，以縮小搜尋範圍。

- **搜尋統計資料**。 在您執行搜尋之後，您可以查看預估搜尋結果的統計資料，例如符合搜尋準則的專案數目及總大小。 其他統計資料包括包含搜尋結果的上方內容位置，以及符合搜尋查詢不同部分的專案數。

- **匯出搜尋結果**。 將搜尋結果匯出至組織中的本機電腦，並以兩個步驟為過程。 當您匯出搜尋結果時，會將專案從 Microsoft 365 中的原始內容位置複製到 Microsoft 提供的 Azure 儲存體位置。 然後，您可以將這些專案下載到本機電腦。

- 以 **角色為基礎的許可權**。 使用以角色為基礎的 access (RBAC) 許可權，以控制不同使用者可以執行的 eDiscovery 相關工作。 您可以使用者與內建的 eDiscovery 相關角色群組，或建立自訂角色群組指派特定的 eDiscovery 許可權。

- **案例管理**。 核心 ediscovery 和 Advanced eDiscovery 中的 eDiscovery 案例可讓您將特定的搜尋和匯出與特定調查產生關聯。 您也可以將成員指派給案例，以控制誰可以存取案例及查看案例的內容。

- **將內容位置** 設為 [合法保留]。 在案例中對內容位置進行合法保留，以保留與調查相關的內容。 這可讓您在調查過程中，保護以電子方式儲存的資訊，避免無意間 (或有意) 刪除。

- **保管人管理**。 在名為「 *保管人* 」的情況 (下，管理您已識別為感興趣的人員) 以及可能未與保管人相關聯的其他資料來源。 當您將保管人和非 custodial 資料來源新增至案例時，您可以在這些資料來源上進行合法保留、使用合法保留通知程式與保管人進行通訊，以及搜尋保管人和非 custodial 資料來源，以收集與案例相關的內容。

- **法律封存通知**。 管理案例保管人的通訊處理常式。 法律保留通知會指示保管人保留與案例相關的內容。 您可以追蹤保管人已接收、讀取及認可的通知。 Advanced eDiscovery 中的通訊工作流程可讓您建立及傳送初始通知、提醒及上報，如果保管人無法認可保留通知。

- **高級索引**。 當您將 custodial 和非保管人資料來源新增至案例時，會在稱為「 *高級索引*) 的程式中，將關聯的內容位置重新編制索引 (，這樣就能重新處理被視為部分索引的任何內容，使其在您收集調查資料時完全可供搜尋。

- **錯誤修正**。 使用稱為「 *錯誤修正*」的處理常式修正處理錯誤。 錯誤修正可讓您修正資料問題，使 Advanced eDiscovery 無法在高級索引期間正確處理內容。 例如，因為檔案遭到鎖定或加密，所以無法處理受密碼保護的檔案。 使用錯誤修正，您可以下載發生錯誤的檔案、移除密碼保護，然後上傳補救的檔案。

- **複查集**。 將相關資料新增至審閱集。 「審閱集」是 microsoft 雲端中，由 microsoft 提供的安全 Azure 儲存體位置。 當您將資料新增至審閱集時，收集的專案會從其原始內容位置複製到審閱集。 「審閱集」提供靜態的已知內容集合，您可以使用預測編碼模型進行搜尋、篩選、標記、分析及預測。 您也可以追蹤和報告新增至審閱集的內容。

- **支援雲端附件和 SharePoint 版本**。 當您將內容新增至審閱集時，您可以選擇包含雲端附件或連結的檔案。 這表示雲端附件或連結的檔案的目標檔案會新增至審閱集。 您也可以選擇將 SharePoint 檔的所有版本新增至審閱集。

- **光學字元辨識 (OCR)**。 當內容新增至審閱集時，OCR 功能會從影像析取文字，並包含包含新增至審閱集之內容的影像文字。 這可讓您在查詢審閱集中的內容時搜尋影像文字。

- **交談執行緒**。 從 Teams 和 Yammer 交談的聊天訊息新增至審閱集時，您可以選擇收集整個交談執行緒。 這表示包含符合集合準則之專案的整個聊天對話會新增至審閱集。 這可讓您查看前後交談內容中的聊天室專案。

- **集合統計資料和報告**。 建立草稿集合或認可 a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a a to a 您也可以預覽結果的子集。 此外，這包括從其上層專案解壓縮並新增為審閱集之個別專案的子專案數目。

- **複查設定篩選**。 將內容新增至審閱集合之後，您可以套用篩選器，只顯示符合篩選準則的專案集。 然後，您可以將篩選集合儲存為查詢，讓您快速重新套用已儲存的篩選。 複習 set 篩選和儲存的查詢，可協助您快速挑選內容至最適合您調查的專案。

- **標記**。 標記也可協助您挑選不相關的內容，以及識別最相關的內容。 當專家、律師或其他使用者查看審閱集中的內容時，可以使用標記來捕獲其與內容相關的觀點。 例如，若要挑選不必要的內容，使用者可以使用標記（如「沒有回應」）來標記檔。 在審閱及標記內容之後，可以建立「複查集」查詢，以排除標記為「無回應」的任何內容。 此程式會從 eDiscovery 工作流程的後續步驟中消除未回應的內容。

- **分析**。 Advanced eDiscovery 提供的工具可分析審閱集檔，以協助您以一致的方式組織檔，並減少要檢查的檔數量。 *接近重複偵測* 可群組的類似檔，以協助您更有效率地進行審閱程式。 *電子郵件執行緒* 識別在電子郵件線索中提供完整的交談內容的特定電子郵件。 *主題* 功能會嘗試分析回顧中所討論的主題，並將主題指派給檔，讓您可以查看具有相關主題的檔。 這些分析功能可協助您更有效率地進行您的審閱程式，讓檢閱者可以查看收集的檔數。

- **預測編碼模型**。 使用預測編碼模型，將大量案例內容縮小及挑選為一組可供查看的相關專案。 這是透過建立及訓練您自己的預測編碼模型，協助您在評審集中查看最相關專案的優先順序。 系統會使用訓練，將預測分數套用至評審集合中的每個專案。 這可讓您根據預測分數來篩選項目，這可讓您先查看最相關的 (或非相關的) 專案。

- **計算的檔中繼資料**。 許多 Advanced eDiscovery 的功能，例如高級索引、交談執行緒、分析和預測編碼新增中繼資料屬性，以查看已設定的檔。 此中繼資料包含有關執行特定功能之函數的相關資訊。 審閱檔時，您可以篩選中繼資料屬性，以顯示符合篩選準則的檔。 在匯出 set 檔之後，此中繼資料可匯入協力廠商的審閱應用程式。

- **長時間執行工作的透明度**。 Advanced eDiscovery 中的工作通常是由使用者動作所觸發的長時間執行程式，例如，將保管人新增至案例、將內容新增至審閱集、執行分析，以及訓練預測編碼模型。 您可以追蹤這些工作的狀態，並取得支援資訊，如果您需要將問題提升至 MS 支援。

- **匯出至客戶擁有的 Azure 儲存體位置**。 當您從審閱集匯出檔時，可以選擇將其匯出至組織所管理的 Azure 儲存體帳戶。 此外，Advanced eDiscovery 可讓您自訂匯出的資料。 這包括匯出檔案中繼資料、原生檔案、文字檔、標記，以及已儲存至 PDF 檔案的 redacted 檔。

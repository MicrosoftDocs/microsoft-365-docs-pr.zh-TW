# <a name="data-protection-impact-assessments"></a>資料保護影響評估

在一般資料保護規定 (GDPR) 下，資料控制者若要進行「可能會對自然人的權利和自由造成高度風險」的處理作業，必須準備資料保護影響評估 (DPIA)。Microsoft Azure 或 Dynamics 365 本身不具有使用它的資料控制者應該建立 DPIA 的項目。然而，是否需要 DPIA，要視控制者「如何」** 部署、設定、使用各產品的詳細資料和相關內容而定。

本文件旨在提供資料控制者資訊，協助他們判斷是否需要 DPIA，以及若需要，應包含哪些詳細資料。

- [Office 365](gdpr-dpia-office365.md)
- [Azure](gdpr-dpia-azure.md)
- [Dynamics 365](gdpr-dpia-dynamics.md)
- [Microsoft 支援和專業服務](gdpr-dpia-prof-services.md)

如需 Microsoft 資料保護的 GDPR 合規性詳細資訊，請參閱***服務信任入口網站***中的[合規性](https://servicetrust.microsoft.com/ComplianceManager)。

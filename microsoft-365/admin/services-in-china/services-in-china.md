# <a name="office-365-operated-by-21vianet"></a>由 21Vianet 運作的 Office 365

由世紀運作的 Office 365 旨在滿足在中國安全、可靠及可伸縮的雲端服務的需求。 這項服務由 Microsoft 授權給 21Vianet 的技術提供支援。
  
Microsoft 不直接營運此項服務。 世紀會運作、提供並管理服務的傳遞。 21Vianet 是中國最大的運營商中立網際網路資料中心服務提供者，提供託管、受管理的網路服務，以及雲端運算基礎結構服務。 使用 Microsoft 技術授權，世紀運作當地 Office 365 資料中心，讓您能夠使用 Office 365 服務，同時將您的資料保留在中國。 世紀也會提供您的訂閱和計費服務，以及支援。
  
> [!NOTE]
> 這些服務須遵守中國法律。 
  
 **在 WeChat 上關注我們**
  
掃描此 QR 碼，在 WeChat 上關注我們，並取得由 21Vianet 運作的 Office 365 的最新更新。
  
![掃描此 QR 碼以在 WeChat 關注我們](../../media/9bbbdf3b-b3ab-4355-82a0-37a84d70735b.png)
  
 **關於由世紀運作的 Office 365 中的服務**
  
以下各節強調每個服務的差異重點。 最後，我們的目標是與全域服務達成奇偶。 不過，由於 a 年在中國的資料中心的合作夥伴運作，其運作的獨特性質，有些功能尚未啟用。 經過一段時間後，客戶將會看到服務的功能逐漸完整。 如需適用于由世紀運作之每個 Office 365 計畫的服務的詳細資訊，請參閱[Office 365 服務描述](/office365/servicedescriptions/office-365-platform-service-description/office-365-operated-by-21vianet)。
  
    
如果您想要瞭解如何開始使用一般 Office 365 服務，請參閱[開始](../admin-overview/get-started-with-office-365.md)使用。

## <a name="exchange"></a>Exchange

 Exchange 2013 累積更新 5 (CU5) 、內部部署 Exchange 2013 組織和 Office 365 服務之間的全功能混合式部署現在已支援。 利用混合式設定向導中的新改進功能，Exchange 2013 CU5 支援您的內部部署和 Exchange Online 組織之間的下列混合式功能：  

- 內部部署和 Exchange Online 組織之間的安全郵件路由。 
- 以共用網域命名空間進行的郵件路由傳送。例如，內部部署和 Exchange Online 組織都採用 @contoso.com SMTP 網域。 
- 統一的全域通訊清單 (GAL)，也稱為「共用通訊錄」。  
- 在內部部署和 Exchange Online 組織之間共用空閒/忙碌和行事曆。  
- 集中式輸入與輸出郵件流程控制。您可設定所有輸入與輸出 Exchange Online 郵件以透過內部部署 Exchange 組織路由傳輸。 
- 內部部署和 Exchange Online 組織的單一 Office Outlook Web App URL。  
- 將現有內部部署信箱移至 Exchange Online 組織的能力。若需要，Exchange Online 信箱也可移回至內部部署組織。  
- 使用內部部署 Exchange 系統管理中心 (EAC) 進行集中式信箱管理。  
- Outlook 連絡人的 MailTips、HD 相片支援，以及內部部署和 Exchange Online 組織之間的多信箱搜尋。 
- 適用於內部部署 Exchange 信箱的雲端式郵件封存。

## <a name="data-subject-requests-for-gdpr"></a>GDPR 的資料主體要求

GDPR 授與個人 (，或資料主體) 某些與處理個人資料的處理相關的權利，包括更正不正確的資料、擦除資料或限制處理、接收資料，以及完成將資料傳輸至另一個控制器的要求。 由世紀運作之 Office 365 的承租人系統管理員角色可以以下列方式代表資料主體要求資料：
  
- 租使用者系統管理員可以使用 Azure Active Directory 系統管理中心，從 Azure Active Directory 和相關的服務中永久刪除資料主體。
    
- 承租人系統管理員可以使用資料記錄匯出，將系統產生的 Microsoft 服務記錄匯出。
    
如需詳細資訊和指示，請參閱 [GDPR (DSR) 的資料主體要求](https://www.trustcenter.cn/privacy/gdpr-office365.mdl)。

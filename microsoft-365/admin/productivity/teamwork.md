# <a name="teamwork--people-experiences"></a>團隊合作–人員經驗

生產力分數可協助組織使用有關人員如何使用 Microsoft 365 應用程式和支援技術經驗的衡量方式，來變換工作的運作方式。 分數會反映您的組織&#39;人員和技術經驗類別的效能，並將您的分數與像您的組織相比較。 「團隊合作」類別是「人員經驗」的一部分度量。 若要深入瞭解，請參閱 [生產力評分一覽](productivity-score.md) 。

## <a name="why-your-org39s-teamwork-score-matters"></a>為什麼您的組織&#39;的團隊合作分數重要

組織生產力的基本 pillar 是指具有共同目標的一群人員，以及共同作業成功的共同資源。 「調查」表示當人們共用工作區中的資訊及共同作業時，最多可節約4小時的時間。 他們可以找到相關的檔、尋找之前討論的內容，以及傳遞共用目標。 請參閱[證據](https://www.microsoft.com/microsoft-365/blog/wp-content/uploads/sites/2/2019/04/Total-Economic-Impact-Microsoft-Teams.pdf)

## <a name="how-we-calculate-the-score"></a>如何計算分數

我們會在經驗中提供此類別的主要計量。 然後，下列各項評分架構都是用來計算您的分數。

      - 透過小組傳送通道郵件
      - 讀取和建立內容 (我們共同指的是 SharePoint 小組網站中的內容互動) 。

        - 分子：透過 Exchange 將電子郵件傳送至群組信箱，或透過小組傳送通道郵件，或是在 SharePoint 小組網站中讀取及建立內容的人員人數。

## <a name="people-in-your-organization"></a>組織中的人員

本節可提供您想要重點關注的衡量標準，方法是提供與頁面上所有真知灼見相關之活動的使用者層級詳細資料。

下列各欄是在使用者層級的表格中呈現：

- **Username：**  是使用者的電子郵件地址。
- **顯示名稱：**  是使用者的完整名稱。
- **上次活動日期：**  是使用者最後一次執行與在過去28天內合作之任何深入瞭解相關的活動。
- **部門：** 在 Azure Active Directory (Azure AD) 中為使用者定義的部門。
- **公司：** 在 Azure AD 中為使用者定義的公司。
- **國家/地區：** 在 Azure AD 中為使用者定義的國家/地區。
- **狀態：** 在 Azure AD 中為使用者定義的狀態。
- **城市：** 在 Azure AD 中為使用者定義的城市。
- **工作區 active in：** 使用者已在使用中的工作區數目 (Microsoft 365 群組) 。
- **傳送天數電子郵件：** 使用者在過去28天內傳送電子郵件的天數。
- **傳送聊天或通道訊息的天數：** 使用者于過去28天內，使用者于 Microsoft 小組傳送聊天訊息或通道訊息的天數。
- **閱讀內容天數：** 使用者在過去28天內，在 OneDrive 或 SharePoint 上，存取 Microsoft Word/Excel/PowerPoint/OneNote 或 PDF) 類型的內容 (檔的天數。
- **建立內容的天數：** 在過去28天內，使用者已建立、修改或上傳的內容 (檔案類型為 Microsoft Word/Excel/PowerPoint/OneNote 或 PDF OneDrive) 的天數。
- **將電子郵件傳送至群組信箱的天數：** 在過去28天內，使用者已將電子郵件傳送至群組信箱的天數
- **每天通道訊息：** 過去28天內，使用者已在 Microsoft 小組傳送通道郵件的天數
- **在 SharePoint 小組網站上讀取內容的天數：** 使用者在過去28天內，在 SharePoint 小組網站上，存取 Microsoft Word/Excel/PowerPoint/OneNote 或 PDF) 類型的內容 (檔的天數
- **在 SharePoint 小組網站上建立內容的天數：** 過去28天內，使用者已在 SharePoint 小組網站上建立、修改或上傳內容 (檔的天數（Microsoft Word/Excel/PowerPoint/OneNote 或 PDF) ）

由於團隊合作是對共用工作區所做的人員的職能，因此也是使用這些共用工作區之人員使用方式的一部分。  若要查看工作區層級的詳細資料，請在下拉式功能表中選取 [ **顯示工作區資料** ]。

在工作區層級的表格中會顯示下列資料行：

- **工作區名稱：** 共用工作區名稱 (Microsoft 365 群組) 
 - [ **上次活動日期** ] 指的是最後一次，指的是最近28天內，任何與團隊合作之深入瞭解相關的活動。
 - **成員總數：** 工作區的成員數目。
 - **建立日期：** 建立工作區的日期。
 -  **有小組小組：** 布林值，指出是否工作區有相關聯的 Microsoft 團隊團隊。
 - **任何活動的天數：** 在過去28天內，工作區有任何活動（包括關聯的 SharePoint 網站上的群組電子郵件、通道訊息或檔案活動）的活動數目。
 - **電子郵件活動天數：** 工作區在過去28天內對電子郵件活動進行群組的天數。
 - **含頻道活動的天數：** 過去28天內，工作區有通道郵件活動的天數。
 - **包含讀取內容活動的天數：** 工作區在過去28天內從關聯的 SharePoint 網站存取 Office 檔案（例如 Word、Excel、PowerPoint、OneNote 或 PDF）的天數。
 - [ **建立內容] 活動的天數：** 工作區在過去28天內，從關聯的 SharePoint 網站建立、修改或上傳的 Office 檔案的天數。
 - **具有任何活動的人員：** 過去28天內，執行包含群組電子郵件、頻道訊息或關聯的 SharePoint 網站上的檔活動的使用者人數。
 - **具有電子郵件活動的人員：** 過去28天內，在工作區中執行任何群組電子郵件活動的使用者人數。
 - **具有頻道活動的人員：** 過去28天內，在工作區中傳送通道郵件的使用者人數。
 - **具有讀取內容活動的人員：** 過去28天內，從相關聯的 SharePoint 網站存取 Office 檔案的人員人數。
 - **具有 [建立內容] 活動的人員：** 從相關聯的 SharePoint 網站中建立、修改或上傳 Office 檔案的使用者人數。

## <a name="notes"></a>注意事項

- 所有對共用工作區的參考都包括 Microsoft 365 群組和關聯的小組小組、SharePoint 小組網站和群組信箱。
- 在生產力分數中，只會考慮介於2到500成員之間的共用工作區進行團隊合作。
- 團隊合作衡量組織內人員的參與。 它不會&#39;捕獲外部成員的活動。
- 合作中的通道郵件不包含透過 Microsoft 小組行動用戶端所傳送的郵件。
- 在我們所進行的通訊和團隊合作的類別之間，電子郵件計量與通道郵件之間可能會有細微的差異。

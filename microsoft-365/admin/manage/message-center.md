# <a name="message-center"></a>訊息中心


若要追蹤即將進行的變更，包括新功能和已變更的功能、規劃的維護或其他重要公告，請前往訊息 <a href="https://go.microsoft.com/fwlink/p/?linkid=2070717" target="_blank">中心</a>。 
  
若要開啟訊息中心：

::: moniker range="o365-worldwide"

- 在系統管理中心，請前往 **健康** > <a href="https://go.microsoft.com/fwlink/p/?linkid=2070717" target="_blank">訊息中心</a>。

::: moniker-end

::: moniker range="o365-germany"

- 在系統 <a href="https://go.microsoft.com/fwlink/p/?linkid=848041" target="_blank">管理中心，</a>請前往 **健康** > **訊息中心**。

::: moniker-end

::: moniker range="o365-21vianet"
 
- 在系統 <a href="https://go.microsoft.com/fwlink/p/?linkid=850627" target="_blank">管理中心，</a>請前往 **健康** > **訊息中心**。

::: moniker-end

您也可以在行動裝置上使用 [Microsoft 365 Admin](https://go.microsoft.com/fwlink/p/?linkid=627216) 應用程式來查看訊息中心，這是使用推入通知保持最新狀態最好的方式。

### <a name="messages"></a>郵件

訊息中心會以表格格式呈現所有使用中郵件的視圖。 根據預設，它會在清單頂端顯示最新的郵件。 您可以選取一個可查看所有使用中郵件、**高** 重要性訊息、**未讀取** 訊息和已 **解除訊息的 Tab。** 您可以使用篩選下拉式功能表查看特定郵件類別的所有郵件。

以下是您將在每個欄中看到之資訊的快速概觀。 選取任一欄標題以變更排序次序。
  
|**欄**|**描述**|
|:-----|:-----|
|核取記號  <br/> |選取欄標題列中的核取方塊會選取目前顯示的所有郵件。 選取一或多郵件旁的核取方塊，即可針對這些郵件採取行動。  <br/> |
|應用程式圖示 <br/> |圖示指出郵件所適用的應用程式。<br/> |
|郵件標題  <br/> |郵件標題是近期變更的簡短描述。 如果完整標題沒有顯示，請將游標停留在該標題上，整個標題會顯示在快顯視窗中。  <br/> |
|更多選項 <br/> |更多選項可讓您關閉訊息、將其標記為已讀取或未讀取，或與其他系統管理員共用。若要還原已解除的郵件，請選取關閉的郵件的Tab，選取郵件旁邊的核取記號，然後 **選取還原**。 <br/> |
|主要更新 <br/> |此欄中的驚嘆號表示有重大更新。 <br/> |
|採取行動者  <br/> |如果我們進行變更，要求您在特定期限之前採取行動，我們只會提供日期。 由於我們很少使用按欄 **表示** 法，如果您在這裡看到某些內容，您應該多加注意。  <br/> |
|類別  <br/> | 郵件會由下列三種類別之一識別： <br/><br/> **防止或修正問題**：通知您影響貴組織的已知問題，並要求您採取行動以避免服務中斷。 防止或修正問題與服務健康情況訊息不同，因為它們會提示您主動避免問題。 <br/> <br/> **規劃變更**：通知您 Microsoft 365 的變更，這些變更可能會要求您採取行動以避免服務中斷。 例如，我們會讓您知道系統需求變更或移除的功能。 我們嘗試在至少 30 天通知任何需要系統管理員採取行動才能讓服務正常運作的變更。 <br/> <br/> **隨時關注**：告訴您我們即將在貴組織中開啟的新功能或更新功能。 這些功能通常會先在 [Microsoft 365 藍圖中宣佈](https://go.microsoft.com/fwlink/?linkid=2070821)。 <br/><br/>也請告知您根據服務等級協定規劃的維護。 規劃的維護可能會導致無法存取 Microsoft 365、特定功能或電子郵件或商務用 OneDrive 等服務的時間。  <br/> |
|上次更新  <br/> |郵件的發佈日期或上次更新的日期。  <br/> |
|郵件識別碼  <br/> |Microsoft 會以郵件識別碼追蹤我們的訊息中心文章。 如果您想要提供意見或撥打支援電話給特定訊息的支援人員，可以參考此識別碼。  <br/> |

前往訊息 [中心管理](manage-messages.md) 郵件，以取得訊息功能詳細資訊。
  
### <a name="major-updates"></a>主要更新

您可以選取訊息中心頂端的高重要性按鈕來查看主要更新。

重要更新會至少提前 30 天通知需要採取行動，且可能包括：
  
- 對每日生產力的變更，例如，信箱、會議、會議、共用和存取

- 變更主題、網頁元件及其他可能會影響自訂功能的元件

- 增加或減少可見的容量，例如儲存空間、規則數量、專案或持續時間

- 產品商標的變更，可能：

  - 造成使用者混淆，

  - 導致變更技術支援人員程式與參考資料，或

  - 變更 URL

- 新的服務或應用程式

- 需要系統管理員採取行動的變更 (排除或修正系統) 

- 變更資料的儲存位置

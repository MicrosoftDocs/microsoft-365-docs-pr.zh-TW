# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>簡化的 GDPR： 您的小型企業的指南

 *使用 Microsoft 365 商務版來減輕並管理 GDPR 合規性* 
  
一般資料保護規定 (GDPR) 是所要求的組織應如何處理個人資料歐盟 (EU) 規定。 GDPR 是要在 2018 年 5 月 25 日進入強制到期日。 如果貴公司有至、 提供服務，或採用的歐盟公民，然後[GDPR](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en)會影響您。 
  
本文可協助您了解 GDPR 是什麼，它是關於的原因及如何 Microsoft 365 商務版可協助您的組織遵守 GDPR。
  
## <a name="a-quick-overview-of-the-gdpr"></a>GDPR 的快速概觀

GDPR 是往前的更新，並展開較早的資料保護指示詞 (DPD) 1995 年第一次准許歐盟規定。 GDPR 需要處理個人資料的隱私權、 是該個人用戶端、 客戶、 員工或業務合作夥伴。 GDPR 的目標是加強歐盟公民的個人資料保護是否他們位於歐盟或其他地方。 規定設定出期望及如何達成這些通知。 當 GDPR 變成強制在 2018 年晚到 」 時，組織必須擁有措施符合 GDPR 的需求。
  
GDPR 是往前所有相關資料，以及如何使用。 請為具有生命週期的資料。 循環啟動您收集的資料時，會繼續在您將它儲存，並使用它 （處理） 時，並結束時您完全刪除它從您的系統。 
  
GDPR 被有關下列類型的資料： 
  
- **個人資料：** 如果您可以連結到個人資料，並識別他們，然後該資料會被視為個人關於 GDPR。 個人資料的範例包括名稱、 地址、 生日以及 IP 位址的日期。 GDPR 考量即使編碼的資訊 （也稱為 「 假名化 」 資訊） 設為個人資料，不論如何難懂或技術的資料是，如果資料可以都連結至個人。
    
- **機密個人資料**這是將更多詳細資料新增至個人資料的資料。 範例包括宗教、 貿易聯集成員資格、 種族等等。 生物識別資料及 DNA，也包含機密個人資料。 在 GDPR 下的敏感資料會有更嚴格的保護規則，比個人資料。 
    
## <a name="gdpr-terms"></a>GDPR 條款

您會看到一些經常 GDPR 中所提及的詞彙。 請務必了解這些字詞。
  
 **同意**
  
GDPR 狀態: 「 服務出現後應設計處理個人資料 」。 GDPR 希望達到此目的使用同意當處理個人資料。 可能會詢問您的客戶若要接收電子郵件從貴公司的簡單動作。 這也表示您的網站上的沒有更多退出核取方塊，當您想要用於行銷部門中的資料。 您必須採取明確同意使用 「 清除除非法案 」。 然後您必須採取或撤銷同意時，同時保留的記錄。
  
 **資料主體權利**
  
GDPR 會建立資料主體權利，這表示，相對於其個人的資料、 客戶、 員工、 商務夥伴、 用戶端、 承包商、 學生、 供應商，依此類推具有的權限：
  
- **隨時了解其資料：** 您必須將個人通知您使用自己的資料。 
    
- **可以存取其資料：** 您必須讓個人存取任何其保留 （例如，藉由使用帳戶的存取權或以手動方式） 的資料。 
    
- **尋求資料修正：** 個人可以要求您修正不精準的資料。 
    
- **尋求要刪除的資料：** 也稱為 '從右至清除'，此權限可讓個別要求他們的公司已收集的個人任何的資料已刪除跨所有使用它，或共用的系統。 
    
- **要求限制處理：** 個人可以要求您抑制或限制他們的資料。 不過，只有在某些情況下適用。 
    
- **有資料可攜性：** 個人可以要求轉接至其他公司資料。 
    
- **物件：** 個人可以用於各種用法，包括直接行銷其資料的物件。 
    
- **要求不受到自動化決策，包括分析：** GDPR 有關於使用設定檔人員及自動化決策根據該分析資料的嚴格規則。 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>小型企業版常見問題 GDPR

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>我是唯一的 proprietor-我確實不必擔心 GDPR 嗎？

GDPR 是關於您處理非您所擁有的員工數的資料。 會影響到公司的所有大小，即使唯一 proprietors。 少於 250 員工與公司沒有某些免除，例如減少記錄保留，不過，只有當您是否確定資料處理並不會影響個別的權限，且偶爾處理。
  
做為範例，非個人資料的處理就是不受，或需要降低量值。 不過，如果即使它只是偶爾，您必須記錄此資料處理處理視為 「 特殊類別機密資料，」 的任何資料。 「 偶爾處理 」 的定義是模糊，但它已經是要套用至一次或很少使用的資料。
  
您也應該要確定您收集的個人資料已受到保護。 這表示，您需要加密，並確定其存取權由控制使用最少的密碼。 將您的客戶資料保持在桌面上以無保護試算表不符合 GDPR 的期望。 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>如何告訴我們公司網站是否 GDPR 合規性？

要詢問您自己的第一個問題是： 不要您收集個人資料無所不在上您的網站？ 例如，您可能必須要求的名稱和電子郵件地址的連絡人表單。 如果您想要傳送行銷電子郵件，請確定您新增 '選擇集'] 核取方塊，以說明完全也將使用的資料。 收件者會檢查該方塊時，才可以您使用他們的個人資料的行銷用途。
  
此外，請檢查可保護將資料儲存的資料庫。 您 web 主控公司或雲端存放裝置廠商將能夠在此通知。 如果您使用 Microsoft 365 商務版，儲存資料是 GDPR 合規性。 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>我的公司超出 Europe。 並 GDPR 確實會影響我們嗎？

GDPR 是往前保護歐盟公民規定。 如果您的公司，處理歐盟公民現在，或者您希望在未來，您將會受到影響。 這適用於這兩個公民住在歐盟狀態以及其他地方住。
  
請考慮下列範例：
  
- 雇用歐盟公民的車輛美國公司需要滿足 GDPR 需求，當他們收集和處理客戶資料。 公司需要採取同意當他們需要客戶的資料，並確保資料安全地儲存。 他們也需要確定客戶可以套用所有的資料主體權利。 
    
- 澳洲公司 sells online、 產品和其使用者設定的線上帳戶。 GDPR 資料主體權利並同意將會套用至歐盟公民開啟帳戶。 公司需要確定客戶可以套用所有的資料主體權利。 
    
- 國際慈善收集 donors 相關資料，並使用它來送出更新並徵求 donations。 GDPR 狀態: '...]供直接行銷用的個人資料的處理可能會被視為為實行的合法的利息。 」 不過，責任是組織，以證明其興趣覆寫那些資料主體。 公司 （或在此案例中，慈善組織） 應該必定可取得明智、 明確、 選擇加入同意。
    
如果客戶資料移動整個框線，也適用於 GDPR。 如果您使用雲端運算的資料存放區，您必須確定服務已完全 GDPR 合規性。 如果資料存放區位於具備資料保護不佳記錄的位置，則可以變得很複雜。 如果您使用 Microsoft 365 商務版，我們有正確的法律文件就地封面 GDPR 需求。
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>確定，我收集資料，但某些其他公司將其儲存。 沒有，取得我掛？

GDPR 規定的如果您收集的資料便會以某種程度影響您。 GDPR 有資料處理器和資料控制者的概念： 
  
- **資料控制者：** 個人或組織 （您可以有聯合控制者），決定資料收集的方式，該怎麼做，和原因。 他們可能會儲存使用其他公司雲端伺服器。 例如，收集客戶資料的網站會控制站。 
    
- **資料處理器：** 個人或組織代表控制程式儲存資料，並處理這些所要求的資料。 例如，Office 365 商務版資料儲存區作為處理器，而是完全 GDPR 合規性。 
    
    組織或系統可以當做一個控制器和處理器。 Microsoft 365 商務版可以做為這兩種，並遵守 GDPR。
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>我仍可以傳送出行銷電子郵件給我舊的客戶嗎？

您必須確定您的客戶，即使您已有年的該地同意其資料用於行銷。 您可能會擷取過同意，以及記錄以顯示它。 如果是的話，您正在繼續行銷部門的所有設定。 如果不是，您需要從客戶繼續行銷給他們取得權限。 這通常會傳送要求客戶移至您的網站，然後選取選項，以同意接收未來的電子郵件的電子郵件。 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>我有當我招募新員工擔心 GDPR 嗎？ 目前的員工呢？

GDPR 不只會影響客戶資料;它太延伸至員工資料。 二百位於通常使用 LinkedIn 等社交媒體平台。 請確定您不要將儲存其快速未經許可任何潛在招募資料。
  
與現有的員工和新的員工合約，合約結尾處的簽章不一定假設同意，尤其是當非除非子句用於合約。 在此情況下，您必須擷取同意子句相關聯以明確方式。 這表示您員工的合約，而定，但您可以運用在某些情況下的"合法利息 」，並新增員工的資料處理通知，以確定您的員工都知道您將如何處理其資料。
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-business"></a>符合使用 Microsoft 365 商務版的隱私權疑慮

變成與 GDPR 合規性是確定可保護個人資料。 GDPR 有稱為隱私權的設計與預設的概念。 這表示，資料保護應該 」 內設 「 系統 」 和 「 產品，以便能滿足隱私權疑慮第二個性質。 
  
與其較大的對應，如小型企業所需便利兼顧安全性。 Microsoft 365 商務版專為少於 300 員工的公司。 小型公司可以使用 Microsoft 雲端架構工具，以提升商業生產力。 使用 Microsoft 365 商務版、 小型企業可以管理電子郵件、 文件，以及偶數會議及事件。 它也有內建的安全性措施和裝置管理，這是很重要的 GDPR 合規性。
  
Microsoft 365 商務版可協助您使用 GDPR 程序如下：
  
- **探索：** GDPR 合規性的一個重要步驟會知道您有哪些資料。 
    
- **管理：** 控制資料的存取權和管理其用途是 GDPR 中不可或缺的一部分。 Microsoft 365 商務版保護根據您想要套用至裝置原則的商務資料。 裝置管理是很重要的保留天數員工遠端工作的所在。 Microsoft 365 商務版包含裝置管理功能，可確保所有裝置上保護資料。 例如，您可以指定貴公司中的所有 Windows 10 裝置已透過 Windows Defender 都保護。 
    
- **保護：** Microsoft 365 商務版的設計目的的安全性。 其裝置管理和資料保護控制項跨公司網路，包括遠端裝置，以協助保護資料安全。 Microsoft 365 商務版提供等 Office 應用程式和加密的文件中的隱私權設定控制項。 使用 Microsoft 365 商務版，您可以執行 GDPR 合規性監控來確定您具有的權限層級的保護設定。 
    
- **報告：** GDPR 將置於強調報告上許多。 即使有單一的員工，企業如果該商務程序非常大量的資料，則需要文件及報告他們的程序。 Microsoft 365 商務版會帶出報表需求較小的組織麻煩。 
    
    例如稽核記錄工具可讓您追蹤和報告資料移動。 報告包含來分類收集和儲存，您使用資料，以及傳輸的資料所做的資料。 
    
客戶、 員工及用戶端察覺更多資料隱私權的重要性和現在預期的公司或組織應遵守的隱私權。 Microsoft 365 商務版為您提供的工具，可達成及維護不到您的業務大規模 upheaval 的 GDPR 合規性。

# <a name="get-started-driving-adoption-of-microsoft-viva-topics"></a>開始推動採用 Microsoft Viva 主題

開始採用之前，您必須瞭解知識管理和 Viva 主題中的相關概念。 下圖顯示在主題探索和 curation 期間發生的情況：

![Viva 主題架構](../media/knowledge-management/topic-management-architecture.png)

- **探索**：使用者可以透過主題卡片，在每天使用的應用程式中探索知識 - 他們也可以在 Microsoft 搜尋中探索主題。
- **策劃**：主題專家 (SME) 會透過主題頁面精簡主題，且 AI 會從他們的輸入中學習。 主題中心包含使用者可以探索及專家可管理的主題頁面。
- **識別**：使用 Microsoft Graph 和人工智慧 (AI) 知識與人員 (主題、技能等等) 會識別並自動組織成相關主題。 SharePoint 內容會以安全性內容編製索引。
- **分機**：使用 Microsoft Graph 內容連接器 (即將推出) ，您可以從外部服務和資料存放庫中攝取知識。

如需詳細資訊，您可以查看簡介的 [概述](topic-experiences-overview.md) 。

請記住下列事項：

- 當更多內容可供使用時，主題探索就會改善。
- 資料的安全性、隱私權和位置都會保留，即使資訊已呈現在新的經驗中也是一樣。
- 使用者需要授權才能檢視 Viva Topics。
- 探索最初是使用英文內容。

為了協助準備，請思考以下問題：

- 主題探索應該使用哪些內容？
- 誰管理主題？
- 誰會看到主題卡片和重點項目？
- 預期有哪些主題？

請檢閱讓 Viva Topics 充分發揮的這份必要條件清單：

|產品或功能 |描述 |
|:-------|:--------|
|具有新式 SharePoint 頁面的 SharePoint Online |主題挖掘只會包含 SharePoint 網站上的內容，而且主題卡片只能在新式頁面上呈現。|
|Microsoft Graph |您可以使用 Microsoft Graph 設定來控制主題包含或排除於搜尋或 Delve 中。 |

## <a name="plan-for-adoption"></a>規劃採用

若要規劃採用 Viva 主題，您必須：

![規劃採用的步驟](../media/knowledge-management/km-adoption-plan-adoption.png)

1. 規劃您的方法與目標案例：
    - 請考慮定義 [案例](#target-scenarios)並排定其優先順序。
    - 請考慮您需要參與的專案 [關係人](#identify-stakeholders) 和專案小組成員。  
    - 找出您想要推動哪些業務影響，以及如何 [衡量成功](#create-a-success-plan)。

2. 讓貴組織參與：
    - 找出需要參與的商務群組和商務小組，並與您所規劃的案例取得關聯。
    - 開始思考如何吸引一些早期採用者，以取得重要、早期的意見反應，以便您可以逐一查看以取得最佳解決方案。
    - 開始組建社區，並考慮這些不同群組如何在組織中使用 Viva 主題。

3. 訓練您的組織：大多數人員會以直觀方式瞭解主題的概念，以及主題卡片如何編譯相關的資訊，以及瞭解及查看其價值。 不過，您可能會想要建立專為您自己的文化和組織量身定制的訓練，以顯示您想要使用 Viva 主題的方式。 一些訓練資源：
    - [Project Cortex 資源中心](https://aka.ms/projectcortex)。 包含概述和功能資訊、錄製的 office 小時影片及簡報，以及合作夥伴及其服務的相關資訊。
    - 即將推出，針對使用者的訓練影片及協助。

4. [建立冠軍網路](#build-a-champion-network)：
    - 您可能已經有練習社群或風雲人物網路。 這些是社交和宣傳以及讓同儕參與互相協助的絕佳方式。 他們也可以共用成功案例，這會很有價值。 他們可以提供建議並引起興奮感。

### <a name="target-scenarios"></a>目標案例

決定您要如何在組織中使用 Viva 主題，以順利使用。 以下是一些知識管理和主題可協助您組織的情況：

- 角色上架 & 訓練：瞭解新組織的術語、主要專案及文化是上架中的重要步驟。 輕鬆探索主題可協助新員工在新的工作、角色或專案中快速上手。
- 專業知識尋找和資訊共用：當主題受管理和共用時，您組織中的人員可以更輕鬆地找到資訊和專家，以協助他們的日常工作。
- 擴大決策和加速上市時間：只要能輕鬆找到資訊和專家，就能更輕鬆做出決策，並縮短專案時間。

#### <a name="example-scenario-for-role-onboarding"></a>角色上架的範例案例

HR 管理員需要將資訊提供給新員工，以協助其快速在公司及其小組上架。 他們想要將它們指向正確的資源、檔和小組成員，以快速且有效地上架。 他們正在尋找一種解決方案，可讓新員工快速找出所需的資訊，而不需要跨多個存放庫進行搜尋，或保留已使用的應用程式。

例如：

- 員工 (約旦) 會加入新的角色，或已新近雇用，而且只是從角色開始。 約旦想要儘快取得與生產力。 不過，約旦也需要協助您尋找開始地點。
- 您的同事 (Kim) 誰是在約旦建立的主題頁面，可協助新員工和其他人尋找該資訊。
- Kim 是 SME，具有查看未確認主題頁面的許可權。 未確認的主題頁面是絕佳起點，可查看 AI 已發現和建立的資料，Kim 能夠加以編輯來新增專家資源、定義，並釘選其他資源。
- 在 SharePoint 上，當您在上看到一份新的帖子，他們會看到一個主題反白顯示，並將其懸停在其上方，以快速取得字詞的定義，以及如何與其他問題聯繫。 之前，Jordan 可能必須四處搜尋此資訊，並連絡同事來找人提問。
- 透過主題公開此資訊非常有用，因為雖然以前就有此資訊，但可能孤立某處，很難找到。 將其帶入正在使用的應用程式，並協助他們找到這些專家也可以促進合約和社區的認識。 這也有助於他們感覺在新角色上更得心應手。

當您自動化此案例時，您可以確定：

- 新員工可以在正確專案上快速聯繫到適當人員。
- 新員工可以在工作流程內立即存取最新專案資訊。
- 大幅縮短搜尋時間。
- 大幅縮短上線時間。

#### <a name="example-scenario-for-customer-call-center"></a>客戶通話中心的範例案例

您可以讓客戶服務快速尋找檔案和專家，以協助解決不尋常的稅收問題，並快速記錄其他人以 Viva 主題輕鬆存取的答案。

例如，支援代表必須快速尋找知識庫文章、檔和原則，讓他們能夠支援客戶。 他們想要在適當的時間尋找適當的資訊，而不必手動透過數個數據基底、存放庫或應用程式，或派遣來電進行。 此外，他們還在尋找一種解決方案，讓他們可以保留在其主要呼叫提示內，以及其交談流程中的訪問原則、法規和指導方針，讓他們能快速回應問題並更新案例。

當您使用 Viva 主題自動化此案例時，您可以確定：

- 縮短支援通話時間。
- 減少呈報第二層和第三層支援。
- 減少特定案例的回電次數。
- 提高客戶滿意度。

#### <a name="prioritize-your-scenarios"></a>設定案例的優先順序

在您識別案例之後，您可以設定案例的優先順序：

排定優先順序的其中一個方法，是在格線中繪製您的案例，顯示影響與輕鬆實作。 尋找具有高影響且易於實施的案例，並將這些案例設為最高優先順序。 影響低且難以實作的案例是最低優先順序。 當您有影響高且易於實作案例的快速成功時，可協助使用者感到興奮，並查看使用主題的可能性。

![高影響，易於實施案例為高優先順序](../media/knowledge-management/topics-prioritize-scenarios.png)

挑選幾個一開始著重的重要案例、與早期採用者合作以取得一些意見反應，然後分階段推出。 如此一來，您就可以迴圈查看、改進並取得意見反應，這樣您就能隨著時間成長採用。

### <a name="identify-stakeholders"></a>識別專案關係人

識別您專案的專案關係人。 主要角色包括執行發起者、成功擁有者和冠軍。

|角色 |責任 |部門 |
|:-------|:-------|:--------|
| 執行贊助者 (s)    | 向公司傳達高階願景和價值   |  執行領導   |
| Project 潛在客戶 (s)  | 監督整個啟動的執行和推出程序 | 專案管理 |
| 知識系統管理員| 安裝和設定 Viva Topics | IT 部門 |
| 知識管理員 | 管理主題及監管分類法 | 任何部門 |
| 分類法管理員 | 監管分類法 | 任何部門 |
| 主題專家和主題參與者 | 產生或檢閱主題和描述 | 任何部門 |
| 風雲人物 | 協助宣傳及管理異議處理 | 任何部門 (員工) |
| 租用戶系統管理員 | 設定租用戶層級設定 | IT 部門 |
| Power Platform 系統管理員| 設定一般資料服務環境 | IT 部門 |
| 搜尋系統管理員或管理員 | 設定搜尋設定 | IT 部門 |

在較大的組織中，您可能也會有多個人員使用這些角色，您必須在兩者之間進行協調。 在較小型的公司中，單一人員可能會執行其中幾個角色。 不同的角色可能更會深入地參與專案的不同階段。 例如，租使用者管理員在設定功能方面更有説明，而主題專家和擁護者不會參與，直到您開始定義主題為止。
 
雖然我們建議您在整個部署中完成上述每個角色，但您可能會發現您不需要所有這些角色即可開始使用已識別的解決方案。

### <a name="create-a-success-plan"></a>建立成功計畫

使用這些指示來測量組織中 Viva 主題的成敗。 看：

1. 主題用法：
      - 主題閱聽
      - 主題數量 - 您策劃的主題清單中已確認和未確認主題。
      - 已發佈主題頁面的數目。
1. 來自主題卡片的使用者意見反應。
1. 進行員工滿意度調查。 Viva 主題應該改善員工尋找資訊的能力，以尋找收集其輸入的方法，以及對該經驗的意見反應。
1. 對搜尋分析的積極影響。 由於主題出現在搜尋體驗中，因此隨著時間的增加，您可能會看到低的放棄搜尋率，因為人們可以更輕鬆地找到搜尋中的主題。 

### <a name="build-a-champion-network"></a>建立冠軍網路

在您的組織中建立擁護者網路。 擁護者很重要，因為它們可以：

- 在其團隊內建立影響圓形
- 磁片磁碟機主題管理 & 維護

您可以從不同的角色招聘冠軍--知識主管與主題專家。

許多冠軍網路都使用 Yammer 作為其平臺。 在 Yammer 中，使用者可以張貼問題並取得問題，並分享成功案例。 您很難只自行進行換用，所以您可以依靠公司內的人員網路為同事提供建議，並顯示其團隊使用 Viva 主題的方式，讓其他小組可以思考自己的案例。

有些組織使用 hackathons (正式或非正式、虛擬或) 人員，以收集可在特定專案上運作的人員群組。 例如，您可以收集主題專家，並讓他們一起使用，以 curate 一組主題頁面。

請考慮您可以如何辨識您的冠軍。 獎勵其活動、提供一些認可，並產生公開的社區感覺和預訂，讓他們感覺像是對某些事項帶來的，也就是從他們的投資取得。

現在，您已準備好要開始實施，您想要確定您已鼓勵進行日常服務。

- 維護您的冠軍的使用中 Yammer 群組。
- 共用成功案例。
- 定期主控參與事件，以分享案例或引進新功能。
- 設定人員和執行 competitions 的挑戰。

## <a name="next-steps"></a>後續步驟

當您準備好推出 Viva Topics 時，您需要讓使用者參與。

- 開始介紹功能集，並讓他們思考其案例。
- 聚集專案關係人並建立案例。
- 推動社群，並思考您將如何吸引他們。
- 然後完成準備步驟。 有些可能是技術準備，也可能是一些商務準備。
- 最後，進行社交和宣傳。

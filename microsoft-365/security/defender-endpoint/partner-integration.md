# <a name="microsoft-defender-for-endpoint-partner-opportunities-and-scenarios"></a>Microsoft Defender for Endpoint partner 機遇和案例

[!INCLUDE [Microsoft 365 Defender rebranding](../../includes/microsoft-defender.md)]

**適用於：**
- [適用於端點的 Microsoft Defender](https://go.microsoft.com/fwlink/p/?linkid=2146631)
- [Microsoft 365 Defender](https://go.microsoft.com/fwlink/?linkid=2118804)


> 想要體驗 Defender for Endpoint？ [註冊免費試用版。](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp?ocid=docs-wdatp-exposedapis-abovefoldlink) 


合作夥伴可以輕鬆地將其現有的安全性產品擴充到開放架構的上方，以及豐富且完整的 APIs 集合，以使用 Defender for Endpoint 來建立分機和整合。 

APIs 範圍的功能範圍包括偵測、管理、回應、弱點，以及使用全球的智慧功能案例。 根據使用案例及需求，協力廠商可以從 Defender 進行資料 stream 或查詢資料。 


## <a name="scenario-1-external-alert-correlation-and-automated-investigation-and-remediation"></a>案例1：外部警示關聯性及自動化調查和修正
Defender for Endpoint 提供了獨特的自動化調查和修正功能，以在規模中促進事件回應。 

將自動化調查和回應功能與其他解決方案（如網路安全性產品或其他端點安全性產品）整合，可協助您解決警示。 整合也會將網路和裝置信號關聯的複雜性降到最低，並有效地簡化裝置上的調查和威脅修正動作。

Defender for Endpoint 會以下列形式新增此案例的支援：

- 外部提醒可以推入 Defender for Endpoint，並以來自于來自 Defender for Endpoint 的其他裝置型提醒並排呈現。 這個視圖提供完整的警示內容，包含實際的處理常式和攻擊的完整案例。

- 警示一經產生，便會在企業中的 Endpoint protected 端點的所有 Defender 間共用信號。 Defender for Endpoint 會採取立即自動或接線員輔助的回應來處理警示。

## <a name="scenario-2-security-orchestration-and-automation-response-soar-integration"></a>案例2：安全性 orchestration 和 automation 回應 (SOAR) 整合
Orchestration 方案可協助您建立行動手冊，並整合用於端點 APIs 公開進行協調回應的工作，例如查詢裝置資料、觸發裝置隔離、封鎖/允許、解析警示及其他的動作。

## <a name="scenario-3-indicators-matching"></a>案例3：指標匹配 
 (IoCs) 相符的指示器是每個 endpoint protection 解決方案中的基本功能。 這項功能可在 Defender for Endpoint 中使用，並可為實體的預防、偵測和排除設定標記清單。 一個可以定義要採取的動作，以及套用動作的時間。

上述案例是平臺擴充性的範例。 您不只局限于這些範例，我們一定會鼓勵您利用開放式架構來探索及探索其他案例。

請遵循 [成為 Microsoft defender For endpoint partner](get-started-partner-integration.md) 中的步驟，以在 Defender for endpoint 中整合您的解決方案。

### <a name="throttling-limits"></a>節流限制

每個 API 都有它自己的節流限制。<br>
依小時及每分鐘的節流限制。<br>
例如，事件 API 的節流限制為：每分鐘50個通話和1500每小時。<br>
在節流的情況下，HTTP 回應狀態碼會是429。

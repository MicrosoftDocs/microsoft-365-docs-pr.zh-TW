# <a name="microsoft-security-portals-and-admin-centers"></a>Microsoft 安全性入口網站和系統管理中心
雖然[Microsoft 365 安全中心](overview-security-center.md)來監控和管理安全性不同身分識別、 資料、 裝置、 應用程式的新 [首頁，您將需要存取各種入口網站的某些特殊工作。

> [!TIP] 
> 若要從 Microsoft 365 安全性中心存取各種相關的入口網站，選取 [在功能窗格中的**更多的資源**。

## <a name="security-portals"></a>安全性入口網站

安全性運算子和系統管理員可以前往下列入口網站來管理特定的安全性設定，以調查潛在威脅活動、 回應作用中的威脅，並與 IT 系統管理員，以修復問題共同作業。
<p></p>

| 入口網站名稱 | 描述 | 連結 |
|---|---|---| 
| Microsoft 365 安全性中心 | 監視及回應威脅活動及不同的身分識別、 電子郵件、 資料、 端點及應用程式與[Microsoft Threat Protection](microsoft-threat-protection.md)加強安全性狀態 | [security.microsoft.com](https://security.microsoft.com/) |
| Microsoft defender 資訊安全中心 | 監視及回應威脅您使用功能隨附於[Microsoft Defender ATP](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/microsoft-defender-advanced-threat-protection)的端點上的活動 | [securitycenter.windows.com](https://securitycenter.microsoft.com/) |
| Office 365 安全性與合規性中心 | 管理[Exchange Online Protection](https://docs.microsoft.com/microsoft-365/security/office-365-security/exchange-online-protection-overview?view=o365-worldwide)和[Office 365 ATP](https://docs.microsoft.com/microsoft-365/security/office-365-security/office-365-atp?view=o365-worldwide)來保護您電子郵件和協同作業的服務，並確保各種資料處理法規合規性 | [https://protection.office.comhttps://protection.office.com](https://protection.office.com) |
| Azure 資訊安全中心入口網站 | 使用[Azure 資訊安全中心](https://docs.microsoft.com/azure/security-center/security-center-intro)，以加強安全性狀態，您的資料中心和您在雲端中的混合式工作負載 | [portal.azure.com/#blade/Microsoft_Azure_Security](https://portal.azure.com/#blade/Microsoft_Azure_Security/SecurityMenuBlade/0) |
| Azure ATP 入口網站 | 識別、 偵測及調查進階的威脅、 遭入侵的身分識別，並使用[Azure ATP](https://docs.microsoft.com/azure-advanced-threat-protection/what-is-atp)中的 Active Directory 訊號惡意測試人員動作 | [portal.atp.azure.com](https://portal.atp.azure.com/) |
| Cloud App Security 入口網站 | 使用[Microsoft Cloud App Security](https://docs.microsoft.com/cloud-app-security/what-is-cloud-app-security)來取得豐富的可見性、 控制資料旅行和複雜的分析，以找出並抵禦 cyberthreats 在雲端服務 | [portal.cloudappsecurity.com](https://portal.cloudappsecurity.com/) |
| Microsoft Defender 安全性智慧入口網站 | 取得安全性的 Microsoft Defender ATP 智慧更新，送出範例中，瀏覽威脅百科全書 | [microsoft.com/wdsi](https://microsoft.com/wdsi) |

## <a name="portals-for-other-workloads"></a>適用於其他工作負載的入口網站

雖然這些入口網站不是專用於管理安全性，它們支援各種工作負載和工作，可能會影響您的安全性。 請造訪這些入口網站管理身分識別、 權限、 裝置設定] 和處理原則的資料。
<p></p>

| 入口網站名稱 | 描述 | 連結 | 
|---|---|---| 
| Azure 入口網站 | 檢視及管理[Azure Active Directory](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-whatis)和所有您[Azure 的資源](https://docs.microsoft.com/azure/azure-resource-manager/management/overview)  | [portal.azure.com](https://portal.azure.com/) |
| Microsoft 365 合規性中心 | 管理資料處理原則，並確認[與法規符合性](https://docs.microsoft.com/microsoft-365/compliance/offering-home?view=o365-worldwide) | [compliance.microsoft.com](https://compliance.microsoft.com/) |
| Microsoft 365 系統管理中心 | 設定 Microsoft 365 服務;管理角色，授權，以及追蹤您的 Microsoft 365 服務的更新 | [admin.microsoft.com](https://admin.microsoft.com/) |
| Microsoft 端點 Manager 系統管理中心 | 使用[Microsoft 端點管理員](https://docs.microsoft.com/configmgr/)來管理並保護裝置使用合併的 Intune 和 Configuration Manager 功能 | [devicemanagement.microsoft.com](https://devicemanagement.microsoft.com/) |
| Microsoft Intune 入口網站 | 使用[Microsoft Intune](https://docs.microsoft.com/intune/fundamentals/what-is-intune)部署裝置原則並監視規範的裝置 | [portal.azure.com/#blade/Microsoft_Intune_DeviceSettings](https://portal.azure.com/#blade/Microsoft_Intune_DeviceSettings/ExtensionLandingBlade/overview)

# <a name="microsoft-security-portals-and-admin-centers"></a>Microsoft 安全性入口網站與系統管理中心

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender.md)]

[雖然 Microsoft 365](overview-security-center.md)資訊安全中心是監控和管理身分、資料、裝置和應用程式安全性的新首頁，但您必須存取特定特殊工作的各種入口網站。

> [!TIP] 
> 若要從 Microsoft 365 資訊安全中心存取各種相關入口網站，請在流覽窗格中選取更多資源。

## <a name="security-portals"></a>安全性入口網站

安全性運算子和系統管理員可以前往下列入口網站，以管理特定安全性設定、調查可能的威脅活動、回應作用中威脅，並與 IT 系統管理員共同作業以修復問題。
<p></p>

| 入口網站名稱 | 說明 | 連結 |
|---|---|---| 
| Microsoft 365 安全性中心 | 使用[Microsoft 365 Defender](microsoft-threat-protection.md)監控和回應威脅活動，並強化您身分、電子郵件、資料、端點和應用程式的安全性影響 | [security.microsoft.com](https://security.microsoft.com/) |
| Microsoft Defender 資訊安全中心 | 使用 Microsoft Defender for Endpoint 提供的功能監控和回應端點 [上的威脅活動](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/microsoft-defender-advanced-threat-protection) | [securitycenter.windows.com](https://securitycenter.microsoft.com/) |
| 安全性與合規性中心 | 管理 [Exchange Online Protection](https://docs.microsoft.com/microsoft-365/security/office-365-security/exchange-online-protection-overview?view=o365-worldwide) 和 Office [365 的 Microsoft Defender](https://docs.microsoft.com/microsoft-365/security/office-365-security/office-365-atp?view=o365-worldwide) 以保護您的電子郵件和共同合作服務，並確保遵守各種資料處理法規 | [protection.office.com](https://protection.office.com) |
| Azure Defender 入口網站 | 使用 [Azure Defender](https://docs.microsoft.com/azure/security-center/security-center-intro) 來強化資料中心的安全性，以及雲端中的混合式工作負載 | [portal.azure.com/#blade/Microsoft_Azure_Security](https://portal.azure.com/#blade/Microsoft_Azure_Security/SecurityMenuBlade/0) |
| Microsoft Defender 身分識別入口網站 | 使用 Active Directory 訊號與 Microsoft Defender 的身分識別，使用 Active Directory 訊號識別、遭到盜用的身分識別及惡意 Insider 動作，以識別、偵測及調查進一步威脅、身分識別 [被入侵和惡意 Insider 動作](https://docs.microsoft.com/azure-advanced-threat-protection/what-is-atp) | [portal.atp.azure.com](https://portal.atp.azure.com/) |
| 雲端 App 安全性入口網站 | 使用 [Microsoft Cloud App 安全性](https://docs.microsoft.com/cloud-app-security/what-is-cloud-app-security) 來取得豐富可見度、控制資料旅行，以及複雜分析以識別並對抗雲端服務上網路威脅 | [portal.cloudappsecurity.com](https://portal.cloudappsecurity.com/) |
| Microsoft Defender 資訊安全入口網站 | 取得適用于端點之 Microsoft Defender 的安全性情報更新、提交範例，並探索威脅百科 | [microsoft.com/wdsi](https://microsoft.com/wdsi) |

## <a name="portals-for-other-workloads"></a>其他工作負載的入口網站

雖然這些入口網站並非專門管理安全性，但可支援可能會影響您安全性的各種工作量和工作。 流覽這些入口網站以管理身分驗證、許可權、裝置設定和資料處理政策。
<p></p>

| 入口網站名稱 | 說明 | 連結 | 
|---|---|---| 
| Azure 入口網站 | 查看和管理 [Azure Active Directory 以及](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-whatis) 您的 [所有 Azure 資源](https://docs.microsoft.com/azure/azure-resource-manager/management/overview)  | [portal.azure.com](https://portal.azure.com/) |
| Microsoft 365 合規性中心 | 管理資料處理政策並確保 [符合法規](https://docs.microsoft.com/microsoft-365/compliance/offering-home?view=o365-worldwide) | [compliance.microsoft.com](https://compliance.microsoft.com/) |
| Microsoft 365 系統管理中心 | 設定 Microsoft 365 服務;管理角色、授權，並追蹤 Microsoft 365 服務的更新 | [admin.microsoft.com](https://admin.microsoft.com/) |
| Microsoft 端點管理員系統管理中心 | 使用 [Microsoft Endpoint Manager，](https://docs.microsoft.com/mem/configmgr/) 使用合併的 Intune 和 Configuration Manager 功能來管理及保護裝置 | [devicemanagement.microsoft.com](https://devicemanagement.microsoft.com/) |
| Microsoft Intune 入口網站 | 使用 [Microsoft Intune](https://docs.microsoft.com/intune/fundamentals/what-is-intune) 來部署裝置策略及監控裝置合規性 | [endpoint.microsoft.com](https://endpoint.microsoft.com/#blade/Microsoft_Intune_DeviceSettings/DevicesMenu/overview)

# <a name="incidents-overview-in-microsoft-365-defender"></a>Microsoft 365 Defender 中的事件概觀

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender.md)]


適用於：
- Microsoft 365 Defender

> 想要體驗 Microsoft 365 Defender 嗎？ 您可以在實驗室 [環境中進行評估，](https://aka.ms/mtp-trial-lab) 或在生產 [環境中執行試驗專案](https://aka.ms/m365d-pilotplaybook)。
>


事件是根據相關的警示。 當在您的網路上發現惡意事件或活動時，系統會產生警示。 個別警示可提供有關進行中攻擊的寶貴線索。 不過，攻擊通常會使用各種向量和技巧以執行外泄。 將個別的線索放在一起可能相當困難且耗時。

這段短片提供 Microsoft 365 Defender 中事件概觀。
<br>

>[!VIDEO https://www.microsoft.com/videoplayer/embed/RE4Bzwz?]

事件是一組相關警示集合，這些警示會指出攻擊事件。 在網路的不同裝置、使用者和信箱實體中發現惡意和可疑事件時，Microsoft 365 Defender 會自動匯總。 將相關警示分組到事件後，安全性維護者可全面查看攻擊。 

例如，安全性保護人員可以看到攻擊的開始位置、使用的策略，以及攻擊已進入網路多遠。 他們也可以查看攻擊的範圍，例如受攻擊的裝置數量、使用者和信箱數量、影響的範圍，以及受影響實體的其他詳細資料。

如果啟用，Microsoft 365 Defender 可透過自動化和人工智慧自動調查並解決個別警示。 安全性保護者也可以執行額外的修復步驟，直接從事件檢視中解決攻擊。 

過去 30 天的事件會顯示在事件佇列中。 從這裡，安全性維護者可以看到哪些事件應該根據風險等級及其他因素來排列優先順序。 

安全性保護者也可以重新命名事件、將事件指派給個別分析師、將事件分類，以及新增標記到事件，以提供更好的自訂事件管理體驗。

# <a name="remediate-malicious-emails-that-were-delivered-in-office-365"></a>修正 Office 365 中傳遞的惡意電子郵件

修正是指對威脅採取 proscribed 動作。 傳送到您組織中的惡意郵件可能會由系統清除，不論是透過整個 (零小時自動清除) ，或是安全小組透過修正動作（如移至 [收件匣]、[移至垃圾郵件]、[移至刪除的郵件] 資料夾、虛刪除或實刪除）。 Office Advanced 威脅防護 (Office ATP) P2/E5 可讓安全性運作小組透過手動搜尋和自動調查來調解電子郵件和共同作業的威脅。

> [!NOTE]
> [在這裡](https://docs.microsoft.com/microsoft-365/security/office-365-security/investigate-malicious-email-that-was-delivered?view=o365-worldwide)跳到調查文章。

## <a name="what-you-need-to-know-before-you-begin"></a>開始之前所需注意的事項

> [!NOTE]
> 若要讓安全性小組修復電子郵件，他們必須有指派的搜尋和清除角色。 角色指派是透過安全性與合規性中心中的許可權來完成。 <p>

## <a name="remediations-through-manual-investigation-or-air"></a>透過手動調查或空氣 Remediations

當安全性小組使用威脅瀏覽器中的搜尋和篩選功能，以手動方式識別威脅時，就會發生手動搜尋。 **手動修正**電子郵件可以透過任何電子郵件 View (惡意程式碼、網路釣魚或所有電子郵件) ，在找到需要補救的一組電子郵件之後，加以觸發。

![Office 365 威脅瀏覽器的手動搜尋依日期。](../../../media/tp-RemediationArticle1.png "威脅總管")

透過威脅瀏覽器，可以多種方式完成電子郵件的選擇：

1. 手領料電子郵件：這表示安全性運作小組可以在各自的視圖中使用篩選器，並從需要補救的威脅瀏覽器中選取一些電子郵件。 選取電子郵件的上限為 100 (100) 。 安全性作業小組無法手動挑選超過一百封電子郵件。

2. 查詢選取：安全操作小組可以使用上方的「全選」按鈕來選取整個查詢。 同樣的查詢也會顯示在動作中心郵件提交詳細資料中。

3. 具有排除的查詢選取專案：有時，安全性運作小組決定透過選取整個查詢，並手動排除一些來自查詢的電子郵件，就能修正電子郵件。 若要這樣做，系統管理員可以使用 [全選] 核取方塊，然後向下滾動以手動排除一些電子郵件。 在此情況下，查詢可以保留的電子郵件數目上限為 1000 (1000) ，而排除數目上限為 100 (100) 。

從威脅瀏覽器中選取電子郵件之後，修復建立作業可以從直接動作開始，或是讓電子郵件排隊以進行動作：

 - 直接核准：當具有適當許可權的安全性人員選擇「移至收件匣」、「移至垃圾郵件」、「移到刪除的專案」、「虛刪除」、「實刪除」的動作時，修復程式開始執行選取的動作。 暫時飛入顯示進行中的修復。

 - 兩個步驟核准：「新增至修正」動作可由沒有適當許可權的系統管理員或需要等候更長時間執行動作的系統管理員採取。 在此情況下，不會直接執行修正動作。 相反地，電子郵件會新增至必須核准執行的修復容器。 在核准修復之前，不會修正電子郵件。 修正一經核准，就會對電子郵件採取動作。

透過手動挑選要調查的電子郵件，在威脅瀏覽器內，以符合系統 (ZAP 中的準則、使用者報告 ) 等方式，以及由安全性作業小組所進行的 (，可觸發) 調查中的**自動調查和回應**。 這些調查可能包含一些必須由安全操作小組核准的建議動作。 這些修正動作會顯示在自動調查中的 [動作] 索引標籤上。  

:::image type="content" source="../../../media/tp-RemediationArticle3.png" alt-text="具有惡意程式碼的郵件是 Zapped 頁面，顯示 Zap 執行的時間。":::

所有 remediations (直接核准或兩步驟核准) 手動透過威脅瀏覽器建立，或是來自自動調查的核准動作，都顯示在 [行動中心] 中，可透過 [*檢查*] >  **動作中心**底下的左側導覽來存取。

:::image type="content" source="../../../media/tp-RemediationArticle4.png" alt-text="依日期和嚴重性列出威脅清單的動作中心。":::

「行動中心」顯示過去30天的所有修復動作。 透過瀏覽器所採取的動作，會顯示在建立修復時，安全性作業小組所提供的相同名稱。 透過「自動化調查」採取的動作，是以可觸發調查之相關警示（如「Zap 電子郵件叢集 ...」）開始的標題所呈現。

您可以開啟每項修復專案，以查看其詳細資料。 當修復專案開啟時，會顯示基本修復詳細資料、修正名稱、建立日期、描述、威脅嚴重性及狀態。 它也會顯示兩個索引標籤。 

1. [**郵件提交]** 索引標籤：這些是透過威脅瀏覽器提交的電子郵件數目，或是要補救的自動調查。 這些電子郵件可以是：

   :::image type="content" source="../../../media/tp-RemediationArticle5.png" alt-text="行動中心，具有可行動且不具可操作性的威脅。":::

   **可行**：可以處理並移動下列雲端信箱位置中的電子郵件，亦即，任何 remediable 類別內的電子郵件都可以從一個位置移動到另一個位置：
  - 收件匣 
  - 垃圾  
  - 已刪除資料夾
  - 虛刪除資料夾

   > [!NOTE]
   > 目前，只有具有信箱存取權的使用者才能從 [虛刪除] 資料夾復原專案。

   **不可行**：下列位置中的電子郵件無法進行處理或移動為電子郵件動作的一部分，也就是說，無法將非 remediable 類別中的電子郵件移動到非 remediable 類別中，也無法移至 remediable。 非 remediable 的位置如下：

   - 隔離
   - 實刪除資料夾
   - 部署/外部
   - 失敗/丟棄
  
   提交的郵件會分類為 remediable 或非 remediable。 在大多數情況下，remediable 和非 remediable 郵件應該會新增至提交的郵件總數。 不過，在很少的情況下，送出的郵件可能不會新增至 remediable 和非 remediable 專案的總和，也可以高於或低於提交的郵件計數總數。 這可能是因為系統延遲、超時或過期的郵件而發生。 郵件會根據您組織的 Explorer 保留期間而過期。

   除非您在組織的瀏覽器保留期間內修正舊郵件，否則請重新嘗試補救專案。 針對系統延遲，修正更新通常會在幾個小時內重新整理。

   如果組織在瀏覽器中的電子郵件保留期間是30天，而您修正的電子郵件會傳回29-30 天，郵件提交數目可能不一定會隨著電子郵件已開始從保留期間內開始移動而增加。

   如果在一段時間內，修復處於「正在進行中」狀態，可能是因為系統延遲。 修正會花幾小時的時間。 您可以在郵件提交計數中看到變化，因為某些電子郵件不是查詢的一部分，因為系統延遲，所以無法啟動修復。 在此情況下，最好重新嘗試修正。  

   為了獲得最佳結果，應以較小的批次50k 或較少的電子郵件來進行修正。  

   在郵件提交中的所有電子郵件中，remediable 電子郵件是在修正期間只會採取的動作。 Office 365 電子郵件系統無法補救非 remediable 的電子郵件，因為它們不是儲存在雲端信箱中。

   針對隔離區中的電子郵件，系統管理員可以移至隔離區，以在必要時對這些電子郵件採取動作，但如果不是手動清除，則電子郵件將會過期隔離。 使用者無法存取因惡意內容而隔離的電子郵件，因此，安全性人員無需採取任何特定動作來擺脫隔離威脅。 如果電子郵件是部署或外部電子郵件，則可以聯繫使用者來處理可疑的電子郵件，或使用個別的電子郵件伺服器/安全性工具來進行移除。 您可以透過在威脅瀏覽器中套用傳遞位置 = 部署/外部篩選來識別這些電子郵件。 若失敗或丟的電子郵件，或是使用者無法存取的電子郵件，就不應該有電子郵件可供您緩解，因為他們不會到達信箱。

   這是提交在重要訊息中心中的顯示方式。 修正可包含多個提交。 如果透過一次自動調查取得多個動作，則每個電子郵件或電子郵件叢集動作都會以不同提交的相同修復方式顯示。

   :::image type="content" source="../../../media/tp-RemediationArticle6.png" alt-text="Zap 電子郵件聚向外彈出面板。":::

   按一下郵件提交專案時，會顯示該修復程式的詳細資料，例如，當透過自動調查或威脅瀏覽器觸發修正時會透過「自動調查」或威脅瀏覽器觸發， (以進行修正的查詢) 、開始時間和結束時間。 它也會顯示已送出以進行修正的郵寄清單。 當郵件從瀏覽器保留期間內移出來時，郵件將會從清單中消失。 此清單也會顯示清單中 remediable 的個別郵件。

2. [**動作記錄]** 索引標籤：此索引標籤會顯示修正的郵件結果，包含的詳細資料（如已核准的日期）、核准者 (系統管理員核准動作) 、動作、狀態及計數。  

   「狀態」是修復的整體狀態。 狀態可以是：

   - **已啟動**：觸發修正時。
   - 已**排入佇列**：修復電子郵件時，修復會進入佇列。
   - **進行中**：正在進行緩解。
   - **已完成**：當所有 remediable 電子郵件的緩解順利完成或發生失敗時。 
   - **失敗**：沒有 remediations 順利。

   只有 remediable 電子郵件可以進行處理時，每個電子郵件的清理都會被視為成功或失敗。 我們從總的 remediable 電子郵件，公開成功和失敗的緩解。

   - **成功**：當 remediable 電子郵件所需的動作完成時，即會符合所需的目的地位置和系統管理員的目的。例如：系統管理員想要移除信箱中的電子郵件，讓他們採取虛刪除電子郵件的動作。 在採取動作後，如果在虛刪除的資料夾中找到 remediable 電子郵件，狀態將會顯示為 [成功]。  

   - **失敗**：當 remediable 電子郵件所需的動作失敗時。 例如：系統管理員想要移除信箱中的電子郵件，因此他採取虛刪除電子郵件的動作。 如果仍在信箱中找到 remediable 電子郵件，狀態將會顯示為 [失敗]。

   按一下動作記錄檔中的任何專案，會顯示修正的詳細資料。 若要取得成功的專案，如果詳細資料顯示、成功或在信箱中找不到，則表示已從信箱中移除專案。 有時候由於修復時發生系統性錯誤而發生問題，在這種情況下，請務必重新嘗試修正。  

修正是一種強大的工具，可減輕威脅，並處理可疑的電子郵件。 這有助於維護組織安全且安全。

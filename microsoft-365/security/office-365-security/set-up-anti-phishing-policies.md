# <a name="set-up-office-365-atp-anti-phishing-and-anti-phishing-policies"></a>設定 Office 365 ATP 防網路釣魚功能及防網路釣魚原則

[ATP 反網路釣魚保護](atp-anti-phishing.md)，部分的[Office 365 高級威脅防護](office-365-atp.md)，可協助保護您的組織免受惡意模擬型網路釣魚攻擊和其他網路釣魚攻擊。 如果您是 Office 365 Enterprise global 或安全性管理員，您可以設定 ATP 反網路釣魚原則。

網路釣魚攻擊來自于以商品為目標的 spear 網路釣魚或 whaling 的各種形式。 隨著複雜性不斷增加，甚至很難識別一些複雜的攻擊。 幸運的是，Office 365 的高級威脅防護可能會有所説明。 您可以設定 ATP 反網路釣魚原則，以協助確保您的組織受到這類攻擊的保護。

> [!NOTE]
> ATP 反網路釣魚僅可用於高級威脅防護（ATP）。 下列訂用帳戶中包含 ATP：[Microsoft 365 企業版](https://www.microsoft.com/microsoft-365/enterprise/home)、[Microsoft 365 商務版](https://www.microsoft.com/microsoft-365/business)、Office 365 企業版 E5 和 Office 365 教育版 A5 等。如果您組織擁有的 Office 365 訂用帳戶不包含 Office 365 ATP，您可能會以附加元件的形式購買 ATP。 如需詳細資訊，請參閱 [Office 365 進階威脅防護方案和定價](https://products.office.com/exchange/advance-threat-protection)和 [Office 365 進階威脅防護服務說明](https://docs.microsoft.com/office365/servicedescriptions/office-365-advanced-threat-protection-service-description) (部分機器翻譯)。 請確認您的組織在 Windows 上使用最新版的 Office 365 ProPlus，以充分利用 ATP 反網路釣魚保護。

防網路釣魚原則也可用於 Office 365 Exchange Online Protection，其中包含一組有限的反欺騙防護，其適用于防範驗證型和以 deception 為基礎的攻擊。

要執行的動作：

1. 檢閱必要條件。

2. 深入瞭解您的防網路釣魚和 ATP 反網路釣魚原則選項。

3. 設定反網路釣魚原則或 ATP 反網路釣魚原則。

> [!IMPORTANT]
> 若要瞭解如何套用多種技術，請參閱[當您的電子郵件上執行多個保護方法和偵測掃描時，會套用什麼原則](how-policies-and-protections-are-combined.md)。

## <a name="review-the-prerequisites"></a>檢閱必要條件

- 若要定義 (或編輯) ATP 原則，您必須獲派適當的角色。 下表中有一些範例描述：

  |角色|指派位置/條件|
  |---------|---------|
  |Office 365 全域管理員|註冊購買 Office 365 的人會預設為為全域系統管理員。 (請參閱[關於 Office 365 系統管理員角色](https://docs.microsoft.com/office365/admin/add-users/about-admin-roles)以深入了解。)|
  |安全性系統管理員|Azure Active Directory 系統管理中心 ([https://aad.portal.azure.com](https://aad.portal.azure.com))|
  |Exchange Online 組織管理|Exchange 系統管理中心 ([https://outlook.office365.com/ecp](https://outlook.office365.com/ecp)) <br>或 <br>  PowerShell Cmdlet (請參閱 [Exchange Online PowerShell](https://docs.microsoft.com/powershell/exchange/exchange-online/exchange-online-powershell))|

  若要深入了解角色和權限，請參閱 [Office 365 安全性的權限與合規性中心](permissions-in-the-security-and-compliance-center.md)。

- 您很可能會為您的組織設定多個反網路釣魚原則。 Office 365 會依照安全&amp;規範中心中的**反網路釣魚頁面**和**ATP 反網路釣魚**網頁上列出的順序來執行這些原則。 當您複習[原則選項](#learn-about-atp-anti-phishing-policy-options)之後，請花一些時間來決定您需要的原則數目及優先順序。

- 計畫花大約5-15 分鐘的時間來設定第一個反網路釣魚原則。

- 最多需要 30 分鐘的時間，最新或更新的原則才會散佈至所有 Office 365 資料中心。

## <a name="set-up-an-anti-phishing-or-atp-anti-phishing-policy"></a>設定反網路釣魚或 ATP 反網路釣魚原則

Office 365 中的每個組織都有預設的反網路釣魚原則，可套用至所有使用者。 您可以建立多個自訂的反網路釣魚原則，您可以將其範圍限定在組織內的特定使用者、群組或網域。 您建立的自訂原則會優先于預設原則。 您可以在 Office 365 安全性&amp;與合規性中心內新增、編輯和刪除反網路釣魚原則。

1. 移至 [https://protection.office.com](https://protection.office.com) 然後以您的公司或學校帳戶當入。

2. 在 [Office 365 安全性&amp;與合規性中心] 的左功能窗格中，按一下 [**威脅管理**] 底下的 [**原則**]。

3. 在 [**原則**] 頁面上，選擇 [**反網路釣魚**] 或 [ **ATP 反網路釣魚**]。

4. 在 [**反網路釣魚**或**ATP 反網路釣魚**] 頁面上，執行下列其中一項操作：

   - 若要新增原則，請選取 **+ Create**。

   - 若要編輯現有的原則，請從 [**反網路釣魚**] 頁面上顯示的清單中選取原則名稱。 （或者，您也可以挑選清單上方的 [**預設] 原則**。）在出現的頁面上，選擇 [**編輯原則**]。

5. 指定原則的名稱、描述及設定。 如需詳細資訊，請參閱[瞭解 ATP 反網路釣魚原則選項](#learn-about-atp-anti-phishing-policy-options)。

6. 檢查您的設定之後，請選擇 [**建立這個原則**（或**儲存**）]。

## <a name="learn-about-atp-anti-phishing-policy-options"></a>深入瞭解 ATP 反網路釣魚原則選項

當您設定或編輯 ATP 反網路釣魚原則時，您可以選擇數種選項，以提供最複雜和全面的保護，如下表所述：

|**此設定**|**執行此動作**|**使用時機：**|
|:-----|:-----|:-----|
|**新增要保護的使用者**|定義原則會保護哪些電子郵件地址。 您最多可以新增60個要從模擬中保護的內部和外部地址。|當您想要確定來自組織外部的郵件不會模仿您正在保護的使用者清單中的其中一個使用者。 您可能想要保護的使用者範例包括高層級的主管、商務擁有者、外部董事會成員等等。  <br/> 此受保護的使用者清單與原則所套用的人員清單不同，也就是強制實施原則的人員。 您可以在 [原則選項] 的 [套用**至**] 區段中，定義 [套用至] 清單。  <br/> 例如，如果您新增`Mary Smith <marys@contoso.com>`為要保護的使用者，請將該原則套用至群組「所有使用者」。 這可確保已傳送給「所有使用者」群組中使用者的郵件，會由原則進行處理。|
|**新增要保護的網域**|可讓您選擇要從模擬中保護的網域。 您可以指定原則包括所有自訂網域、逗號分隔的網域清單，或兩者的組合。 如果您選擇 [**自動包含我擁有的網域**]，後來又將網域新增至 Office 365 組織，則新網域的此反網路釣魚原則將會就地使用。|每當您想要確保來自組織外部的郵件不會模仿已驗證的網域清單中所定義的其中一個網域，或是協力廠商網域。|
|**選擇動作**|選擇當 Office 365 偵測對您新增至原則的使用者和網域的模仿嘗試時，要採取的動作。 您可以在相同的反網路釣魚原則中，為使用者和網域選擇不同的動作。 這些動作適用于任何已由 Office 365 識別的內送電子郵件，如模仿此反網路釣魚原則保護的使用者帳戶或網域。  <br/> **隔離郵件**電子郵件會傳送至 Office 365 隔離區。 當您選擇此選項時，不會將電子郵件傳送給原始收件者。  <br/> **將郵件重新導向至另一個電子郵件地址**電子郵件會傳送至您指定的電子郵件地址。 您可以指定多個電子郵件地址。 當您選擇此選項時，不會將電子郵件傳送給原始收件者。  <br/> **將郵件移至收件者的 [垃圾郵件] 資料夾**電子郵件會傳送至收件者的 [垃圾郵件] 資料夾。 當您選擇此選項時，電子郵件仍會傳送至原始收件者，但不會放在收件者的收件匣中。  <br/> **傳遞郵件並將其他位址新增至 Bcc 行**電子郵件將會傳遞給原始收件者。 此外，您所識別的使用者會在郵件傳遞之前新增至郵件的 [密件副本] 行。 當您選擇此選項時，電子郵件仍會傳送至原始收件者的收件匣。  <br/> **不要套用任何動作**電子郵件會傳遞至原始收件者的收件匣。 不會對電子郵件訊息採取其他任何動作。  <br/> **開啟網路釣魚防護提示**在電子郵件中啟用反網路釣魚安全性秘訣。|當您想對郵件執行動作時，Office 365 已經決定是原則中所定義的使用者或網域的模仿。|
|**啟用信箱智慧**|啟用或停用此原則的信箱智慧。 您只能為雲端式帳戶啟用信箱智慧，也就是其信箱已完全主控于 Office 365 的帳戶。| 這項功能會使用機器學習來判斷使用者的電子郵件模式及其連絡人。 透過此資訊，AI 可更好地區別真實和網路釣魚電子郵件。|
|**啟用信箱智慧型類比保護**|為此原則啟用或停用類比保護的信箱智慧。 這裡的重要方面是對指定信箱的模仿控制權。|當您想要以每位使用者的個別寄件者對應，針對使用者增強類比結果。 這種智慧功能可讓 Office 365 自訂使用者模擬偵測，並更好地處理誤報。 偵測到使用者模擬時，您可以根據信箱智慧來定義要對郵件採取的動作。|
|**新增受信任的寄件者與網域**|定義不會被此原則視為模仿的電子郵件地址和網域。 若郵件來自新增為受信任寄件者和網域的寄件者電子郵件地址和網域，則不會歸為冒充式攻擊。 因此，此原則中的動作和設定不會套用到這些寄件者和網域中的郵件。  <br/><br/>這兩個清單的上限大約是1000個專案。|當使用者與觸發模擬但視為安全的網域或使用者互動時。 例如，如果協力廠商具有與清單中所定義的使用者相同/類似的顯示名稱或功能變數名稱。|
|**套用對象**|定義收件者，其內送的電子郵件將受限於原則的規則。 您可以建立與原則相關聯之收件者的條件和例外狀況。  <br/> 例如，您可以將規則套用至網域中的所有收件者，為您的組織建立全域原則。  <br/> 您也可以建立例外規則，例如不掃描特定收件者群組之電子郵件的規則。|每個原則都必須與一組使用者相關聯，例如特定群組或網域中的使用者。|
|**高級網路釣魚臨界值**|定義如何處理網路釣魚郵件的設定層級。  <br/> **Standard**：懷疑電子郵件網路釣魚的電子郵件是以標準方式處理。  <br/> **嚴格**：系統會處理可疑的電子郵件，使其具有極高的信賴度，與懷疑具有極高可信度的方式相同。  <br/> **更嚴格**的：系統會處理可疑的電子郵件，其可能是以中等或高的置信度，也就是懷疑具有極高信賴程度的電子郵件。  <br/> **最積極**的：系統會處理可疑的電子郵件，其具有低、中或高的置信度，其方式與懷疑具有極高信賴程度的電子郵件相同。|當您想要更積極地處理 Office 365 內可能的網路釣魚郵件時。 例如，安全性非常高的郵件會在網路釣魚上採取最嚴格的動作，而低概率的郵件則會執行較低的動作。 此設定也會影響篩選系統的其他部分，以結合信號。 這並不一定意味著會執行不同的動作。  實際上，您會設定郵件成為網路釣魚的可能性，以判斷（相同）指定的動作。 隨著設定層級的增加，移動優質郵件的機率會隨之增加。|

## <a name="learn-about-anti-phishing-policy-options"></a>深入瞭解反網路釣魚原則選項

當您設定或編輯反網路釣魚時，您可以選擇數個選項，如下表所述：

|**此設定**|**執行此動作**|**使用時機：**|
|:-----|:-----|:-----|
|**套用對象**|定義收件者，其內送的電子郵件將受限於原則的規則。 您可以建立與原則相關聯之收件者的條件和例外狀況。  <br/> 例如，您可以將規則套用至網域中的所有收件者，為您的組織建立全域原則。  <br/> 您也可以建立例外規則，例如不掃描特定收件者群組之電子郵件的規則。|每個原則都必須與一組使用者相關聯，例如特定群組或網域中的使用者。|
|**選擇動作**|選擇當 Office 365 偵測到組織內部或外部組織的電子組織欺騙嘗試時，要採取的動作。 這些動作適用于任何已由 Office 365 識別的內送電子郵件，因為受到保護此反網路釣魚原則的使用者的欺騙嘗試。  <br/> **隔離郵件**電子郵件會傳送至 Office 365 隔離區。 當您選擇此選項時，不會將電子郵件傳送給原始收件者。  <br/> **將郵件移至收件者的 [垃圾郵件] 資料夾**電子郵件會傳送至收件者的 [垃圾郵件] 資料夾。 當您選擇此選項時，電子郵件仍會傳送至原始收件者，但不會放在收件者的收件匣中。  <br/> **不要套用任何動作**電子郵件會傳遞至原始收件者的收件匣。 不會對電子郵件訊息採取其他任何動作。|當您想對郵件執行動作時，Office 365 認為是原則中所定義的內部或外部網域的哄騙嘗試。|

在您的組織設定反網路釣魚原則或 ATP 反網路釣魚原則之後，您可以透過[查看高級威脅防護的報告](view-reports-for-atp.md)，查看服務的運作方式。

## <a name="example-anti-phishing-policy-to-protect-a-user-and-a-domain"></a>範例：保護使用者和網域的反網路釣魚原則

此範例會設定名為「網域和 CEO」的原則，此原則會提供使用者和網域的類比保護，然後將該原則套用至網域`contoso.com`中使用者所接收的所有電子郵件。 安全性系統管理員已決定原則必須符合下列商務需求：

- 原則需要可保護 CEO 的電子郵件帳戶和整個網域。

- 只要是判斷為嘗試冒充 CEO 使用者帳戶的郵件，都必須重新導向至安全性系統管理員的電子郵件地址。

- 判斷為嘗試冒充網域的郵件較不緊急，應將其隔離以便日後檢閱。

Contoso 的安全性系統管理員可能會使用如下的值，以建立符合這些需求的反網路釣魚原則。

|||
|:-----|:-----|
|**設定或選項**|**範例**|
|名稱|網域和 CEO|
|描述|確保 CEO 和我們的網域沒有遭到冒充。|
|新增要保護的使用者|至少填入 CEO 的電子郵件地址。|
|新增要保護的網域|包含 CEO 辦公室的組織網域。|
|選擇動作|如果模仿的使用者傳送電子郵件：選擇 [重新**導向郵件至其他電子郵件地址**]，然後輸入安全性管理員的電子郵件地址，例如`securityadmin@contoso.com`。  <br/> 如果電子郵件來自冒充的網域：請選擇 **[隔離郵件]**。|
|信箱情報|當您建立新的反網路釣魚原則時，系統會預設選取信箱情報。 請將此設定保留為 **[開啟]**，以獲得最佳結果。|
|新增受信任的寄件者與網域|在此範例中，請不要定義任何覆寫。|
|套用對象|請選取 **[收件者的網域是]**。 在 **[任一項]** 底下選取 **[選擇]**。 選取 **[+ 新增]**。 選取網功能變數名稱稱旁邊的核取方塊，例如`contoso.com`，在清單中，然後選取 [**新增**]。 選取 **[完成]**。|

## <a name="delete-an-anti-phishing-or-atp-anti-phishing-policy"></a>刪除反網路釣魚或 ATP 反網路釣魚原則

您可以使用安全性&amp;與合規性中心來刪除您建立的自訂原則。 您無法刪除組織的預設原則。 建議您使用安全性&amp;相容性中心，以查看或編輯任何 ATP 原則。

1. 移至 [https://protection.office.com](https://protection.office.com) 然後以您的公司或學校帳戶當入。

2. 在左側導覽中，選擇 [威脅管理]**** 下方的 [原則]****。

3. 在 [**原則**] 頁面上，選擇 [**反網路釣魚**] 或 [ **ATP 反網路釣魚**]。

4. 在 [**反網路釣魚**或**ATP 反網路釣魚**] 頁面上，從清單中選取原則名稱。

5. 在出現的頁面上，選擇 [**刪除原則**]。 最多需要 30 分鐘的時間，變更才會散佈至所有 Office 365 資料中心。

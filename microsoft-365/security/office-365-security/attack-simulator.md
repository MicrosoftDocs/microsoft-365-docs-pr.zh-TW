# <a name="attack-simulator-in-microsoft-defender-for-office-365"></a>Microsoft Defender for Office 365 中的攻擊模擬器

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

如果您的組織有 Microsoft Defender for Office 365 Plan 2 （包括 [威脅調查和回應功能](office-365-ti.md)），您可以使用安全性 & 規範中心內的攻擊模擬器，在您的組織中執行實際的攻擊案例。 這些模擬的攻擊可協助您找出並找出有漏洞的使用者，而真實的攻擊會影響您的下一行。 若要深入瞭解，請閱讀本文。

> [!NOTE]
> 攻擊類比和訓練相關的資料會與其他 Microsoft 365 服務的客戶資料一起儲存。 如需詳細資訊，請參閱 [Microsoft 365 資料位置](/microsoft-365/enterprise/o365-data-locations)。

> [!TIP]
> 攻擊模擬訓練可用於 Microsoft 365 的安全性中心的公開預覽。 請參閱 [使用 Microsoft Defender For Office 365 模擬網路釣魚攻擊](attack-simulation-training.md) 以深入瞭解。

## <a name="what-do-you-need-to-know-before-you-begin"></a>開始之前有哪些須知？

- 若要開啟安全性與合規性中心，請移至 <https://protection.office.com/>。 攻擊模擬器可在 **威脅管理** \> **攻擊模擬器** 中取得。 請直接移至 [攻擊模擬器]，開啟] <https://protection.office.com/attacksimulator> 。

- 如需不同 Microsoft 365 訂閱中攻擊模擬器可用性的相關資訊，請參閱 [Microsoft Defender For Office 365 service description](https://docs.microsoft.com/office365/servicedescriptions/office-365-advanced-threat-protection-service-description)。

- 您必須是「 **組織管理** 」或「 **安全性管理員** 」角色群組的成員。 如需有關安全性與合規性中心中角色群組的詳細資訊，請參閱[安全性與合規性中心裡的權限](permissions-in-the-security-and-compliance-center.md)。

- 您的帳戶必須設定為進行多重要素驗證 (MFA) 以在攻擊模擬器中建立及管理活動。 如需相關指示，請參閱 [設定多重要素驗證](https://docs.microsoft.com/microsoft-365/admin/security-and-compliance/set-up-multi-factor-authentication)。

- 網路釣魚活動會收集事件，並處理30天的事件。 在您啟動市場活動之後，歷史活動資料將可長達90天。

- 攻擊模擬程式沒有對應的 PowerShell Cmdlet。

## <a name="spear-phishing-campaigns"></a>Spear 網路釣魚活動

*網路釣魚* 是一種常見的電子郵件攻擊術語，可嘗試竊取看似來自合法或信任寄件者的郵件中的機密資訊。 *Spear 網路釣魚* 是一種目標網路釣魚攻擊，它會使用專門為目標收件者量身定制的焦點和自訂內容 (通常是由攻擊者) 的收件者上的偵測之後。

在攻擊模擬器中，有兩種不同類型的 spear 網路釣魚活動可供使用：

- **Spear 網路釣魚 (認證搜集)**：攻擊會嘗試說服收件者按一下郵件中的 URL。 如果他們按一下連結，則會要求他們輸入他們的認證。 如果是的話，它們會移至下列其中一個位置：

  - 預設頁面，說明這只是測試，並提供識別網路釣魚郵件的秘訣。

    ![使用者按一下 [網路釣魚] 連結並輸入其認證時看到的內容](../../media/attack-simulator-phishing-result.png)

  - 您指定的自訂頁面 (URL) 。

- **Spear 網路釣魚 (附件)**：攻擊會嘗試說服收件者在郵件中開啟 .docx 或 .pdf 附件。 附件包含的來自預設網路釣魚連結的內容相同，但是第一個句子會以 " \<Display Name\> ，您會看到此郵件為最近開啟的電子郵件訊息 ..."。

> [!NOTE]
> 目前，攻擊模擬器中的 spear 網路釣魚活動不會過期。

### <a name="create-a-spear-phishing-campaign"></a>建立 spear 網路釣魚活動

任何 spear 網路釣魚活動的重要部分是傳送給目標收件者之電子郵件的外觀與風格。 若要建立及設定電子郵件訊息，您可以使用下列選項：

- **使用內建的電子郵件範本**：有兩個內建的範本可供使用： **獎品 Giveaway** 和 **工資單更新**。 您可以在建立及啟動市場活動時，進一步自訂範本中的部分、全部或無電子郵件屬性。

- **建立可重複使用的電子郵件範本**：在您建立及儲存電子郵件範本之後，您可以在未來 spear 網路釣魚活動中再次使用它。 您可以在建立及啟動市場活動時，進一步自訂範本中的部分、全部或無電子郵件屬性。

- **在嚮導中建立電子郵件訊息**：當您建立及啟動 spear 網路釣魚活動時，您可以直接在嚮導中建立電子郵件訊息。

#### <a name="step-1-optional-create-a-custom-email-template"></a>步驟 1 (選用) ：建立自訂的電子郵件範本

若要使用其中一個內建範本或直接在嚮導中建立電子郵件訊息，您可以略過此步驟。

1. 在 [安全性 & 規範中心] 中，移至 [ **威脅管理**] \> **攻擊模擬器**。

2. 在 [ **模擬攻擊** ] 頁面的 [ **Spear 網路釣魚 (認證收集])** 或 **Spear 網路釣魚 (附件)** 區段中，按一下 [ **攻擊詳細資料**]。

   您建立範本的方式無關緊要。 範本中的可用選項與這兩種類型的網路釣魚攻擊相同。

3. 在開啟的 [ **攻擊詳細資料** ] 頁面中，于 [ **網路釣魚範本** ] 區段的 [ **建立範本** ] 區域中，按一下 [ **新增範本**]。

4. **設定網路釣魚範本** 嚮導會在新飛入的視窗中啟動。 在 [ **開始** ] 步驟中，輸入範本的唯一顯示名稱，然後按 **[下一步]**。

5. 在 [ **設定電子郵件詳細資料** ] 步驟中，設定下列設定：

   - **從 (名稱)**：郵件寄件者所用的顯示名稱。

   - **從 (電子郵件)**：寄件者的電子郵件地址。

   - **網路釣魚登入伺服器 URL**：按一下下拉式清單，然後從清單中選取其中一個可用的 URLs。 這是使用者將被誘惑按一下的 URL。 選項包括：

     - <http://portal.docdeliveryapp.com>
     - <http://portal.docdeliveryapp.net>
     - <http://portal.docstoreinternal.com>
     - <http://portal.docstoreinternal.net>
     - <http://portal.hardwarecheck.net>
     - <http://portal.hrsupportint.com>
     - <http://portal.payrolltooling.com>
     - <http://portal.payrolltooling.net>
     - <http://portal.prizegiveaway.net>
     - <http://portal.prizesforall.com>
     - <http://portal.salarytoolint.com>
     - <http://portal.salarytoolint.net>

     > [!NOTE]
     >
     > URL 信譽服務可識別一或多個 URLs 為不安全的。 在網路釣魚活動中使用 URL 之前，請先檢查支援的網頁瀏覽器中的 URL 可用性。

   - **自訂登陸頁面 URL**：輸入使用者在按一下 [網路釣魚] 連結並輸入其認證時所採用的選用登陸頁面。 此連結會取代預設登陸頁面。 例如，如果您有內部的知識意識訓練，您可以在這裡指定該 URL。

   - **類別**：目前不使用此設定 () 忽略您輸入的任何專案。

   - 主旨 **：電子** 郵件的 **主旨欄位。**

   完成後，按 [下一步]。

6. 在 [ **撰寫電子郵件** ] 步驟中，建立電子郵件訊息的郵件內文。 您可以使用 [ **電子郵件** ] 索引標籤 (豐富的 html 編輯器) 或 [ **來源** ] 索引標籤 (原始的 html 程式碼) 。

   HTML 格式設定可以簡單或複雜，您必須在需要時。 您可以插入影像和文字，以在收件者的電子郵件客戶程式中增強郵件的 believability。

   - `${username}` 插入收件者的名稱。

   - `${loginserverurl}` 插入上一個步驟中的 **網路釣魚登入伺服器 URL** 值。

   完成後，按 [下一步]。

7. 在 [ **確認** ] 步驟中，按一下 **[完成]**。

     >

## <a name="password-attack-campaigns"></a>密碼攻擊活動

*密碼攻擊* 會嘗試猜測組織中使用者帳戶的密碼，通常是在攻擊者識別一或多個有效的使用者帳戶之後。

在攻擊模擬器中，有兩種不同類型的密碼攻擊活動可供您測試使用者密碼的複雜性：

- **強力密碼 (字典攻擊)**： *強力強制* 或 *字典* 攻擊會在使用者帳戶上使用大量的密碼字典檔案，以供其中一位使用者在一個帳戶) 使用許多密碼 (。 不正確的密碼鎖定有助於封鎖強力密碼攻擊。

  針對字典攻擊，您可以指定一或多個密碼以嘗試 (手動輸入或上傳的檔案) 中，也可以指定一或多個使用者。

- **密碼噴塗攻擊**： *密碼噴塗* 攻擊會在使用者帳戶清單上使用相同的密碼，針對許多帳戶)  (一個密碼。 密碼噴塗攻擊比強力強制密碼攻擊較難偵測 (當攻擊者嘗試多個帳戶的一個密碼，而不會冒出使用者不正確的密碼鎖定功能) 時，成功的可能性。

  在密碼噴塗攻擊中，您只能指定一個密碼進行嘗試，也可以指定一或多個使用者。

> [!NOTE]
> 攻擊模擬器中的密碼攻擊會將使用者名稱和密碼基本身分驗證要求傳遞給端點，讓使用者也可以使用其他驗證方法， (AD FS、密碼雜湊同步處理、傳遞 PingFederate 等等 ) 。 針對已啟用 MFA 的使用者，即使密碼攻擊嘗試其實際密碼，該嘗試永遠會註冊為失敗 (換句話說，MFA 使用者永遠不會出現在市場活動的「 **成功嘗試** 計數」) 中。 這是預期的結果。 MFA 是協助防護密碼攻擊的主要方法。

### <a name="create-and-launch-a-password-attack-campaign"></a>建立及啟動密碼攻擊活動

1. 在 [安全性 & 規範中心] 中，移至 [ **威脅管理**] \> **攻擊模擬器**。

2. 在 [ **模擬攻擊** ] 頁面上，根據您要建立的活動類型進行下列其中一項選擇：

   - 在 [ **強力密碼 (字典攻擊])** 區段中，按一下 [ **啟動攻擊** ] 或按一下 [ **攻擊詳細資料** \> **啟動攻擊**]。

   - 在 [ **密碼噴塗攻擊** ] 區段中，按一下 [ **啟動攻擊** ] 或按一下 [ **攻擊詳細資料** \> **啟動攻擊**]。

3. **設定密碼攻擊** 嚮導會在新飛入的視窗中啟動。 在 [ **開始** ] 步驟中，輸入活動的唯一顯示名稱，然後按 **[下一步]**。

4. 在 [ **目標使用者** ] 步驟中，執行下列其中一個步驟：

   - 按一下 [ **通訊錄** ]，以選取市場活動的收件者 (使用者或群組) 。 每個目標收件者都必須有 Exchange Online 信箱。 如果您按一下 [ **篩選** ] 並套用但未輸入搜尋準則，則會傳回所有收件者， **並將其** 新增至市場活動。

   - 按一下 **[匯入]** ，然後匯入 **檔** 匯入以逗號分隔的值 (CSV) 或電子郵件地址的行分隔檔。 每一行都必須包含收件者的電子郵件地址。

   完成後，按 [下一步]。

5. 在 [ **選擇攻擊設定** ] 步驟中，選擇根據行銷活動類型執行的動作：

   - **強力密碼 (字典攻擊)**：執行下列其中一個步驟：

     - **手動輸入密碼**：在 **按 enter 鍵以新增密碼** 方塊中，輸入密碼，然後按 enter 鍵。 視需要重複此步驟多次。

     - **從字典檔案上傳密碼**：按一下 **[上傳** ]，可匯入在每一行上包含一個密碼的現有文字檔，以及一個空白的最後一行。 文字檔的大小必須為 10 MB 或以下，而且不能包含30000個以上的密碼。

   - **密碼噴塗攻擊**：在 **[攻擊] 方塊中的 [密碼 (s) 中使用** ] 方塊中，輸入一個密碼。

   完成後，按 [下一步]。

6. 在 [ **確認** ] 步驟中，按一下 **[完成]** 以啟動市場活動。 您指定的密碼會嘗試您指定的使用者。

## <a name="view-campaign-results"></a>查看市場活動結果

啟動活動之後，您可以在 [主要 **模擬攻擊** ] 頁面上檢查進度和結果。

使用中的活動會顯示狀態列、已完成百分比值和「 (總計使用者的 (已完成使用者) ) 計數。 按一下 [重新整理] 按鈕，會更新任何使用 **中的活動** 活動的進度。 您也可以按一下 [ **終止** ]，以停止使用中的活動。

當市場活動完成時，狀態變更為 [ **攻擊已完成**]。 您可以執行下列其中一項動作，以查看活動的結果：

- 在 [主 **模擬攻擊** ] 頁面上，按一下 [活動名稱] 底下的 [ **查看報告** ]。

- 在「主要 **模擬攻擊** 」頁面上，按一下該攻擊類型區段中的 [ **攻擊詳細資料** ]。 在開啟的 [ **攻擊詳細資料** ] 頁面上，選取 [ **攻擊歷程記錄** ] 區段中的 [活動]。

上述任一項動作都會帶您前往名為 [ **攻擊詳細資料**] 的頁面。 以下各節將說明此頁面上針對每種類型的活動所提供的資訊。

### <a name="spear-phishing-credentials-harvest-campaign-results"></a>Spear 網路釣魚 (認證搜集) 活動結果

每個活動的 [ **攻擊詳細資料** ] 頁面上提供下列資訊：

- 活動的期間 (開始日期/時間和結束日期/時間) 。

- **目標使用者總數**

- **嘗試成功**：按一下連結 **並** 輸入其認證的使用者數目 () 的 *任何* 使用者名稱和密碼值。

- **整體成功率**：以 **成功嘗試** 為  /  **目標之使用者總數** 所計算的百分比。

- **最快速按一下**：在您啟動市場活動之後，第一位使用者按一下連結所花費的時間。

- **平均按一下**：每個人依序按一下連結所花費的時間與按一下連結的使用者數目的總和。

- **按一下 [成功率**：] (使用者人數) /使用者已 **設定** 的使用者人數計算的百分比。

- **最快的認證**：在您啟動市場活動之後，第一位使用者進入其認證所需的時間。

- **平均認證**：每個人輸入認證所花費的時間，除以輸入其認證的使用者人數。

- **認證成功率**： (使用者輸入其身分憑證的使用者人數) / **使用者所針對的使用者** 人數所計算的百分比。

- 顯示 **連結已按一下** 的柱狀圖圖，以及每日所 **提供的憑證** 數量。

- 圓形圖，顯示所 **按一下的連結**、 **提供的認證**，以及 **無** 活動的百分比。

- 「已 **遭破壞的使用者** 」區段會列出按一下連結之使用者的詳細資料：

  - 使用者的電子郵件地址

  - 按一下連結時的日期/時間。

  - 用戶端 IP 位址。

  - 使用者版本的 Windows 和網頁瀏覽器的詳細資料。

  您可以按一下 [ **匯出** ]，將結果匯出至 CSV 檔案。

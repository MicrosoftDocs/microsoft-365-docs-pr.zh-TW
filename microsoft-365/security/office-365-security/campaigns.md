# <a name="campaign-views-in-office-365-atp"></a>Office 365 ATP 中的行銷活動檢視

> [!NOTE]
> 本主題中所述的功能目前處於預覽，且可能會變更。

行銷活動檢視是 Office 365 安全性與合規性中心中進階威脅防護 (ATP) 的一項功能，可識別和分類服務中的網路釣魚攻擊。 行銷活動檢視可協助您：

- 更有效率地調查和回應網路釣魚攻擊。

- 更進一步了解攻擊範圍。

- 向決策者展現價值。

行銷活動檢視可讓您較任何人類所能夠，更快速且更完整地查看攻擊的全貌。

## <a name="what-is-a-campaign"></a>什麼是行銷活動？

行銷活動是針對一或多個組織的協調式電子郵件攻擊。 現今，竊取認證和公司資料的電子郵件攻擊是一項龐大且能賺錢的生意。 隨著為了阻擋攻擊的技術增加，攻擊者更為老練而足以修改其方法，以確保持續成功。

Microsoft 利用大量防網路釣魚、反垃圾郵件和反惡意程式碼資料，以及在全球各地的整個 Office 365 服務的經驗，以找出行銷活動。 系統會根據數個因素分析和分類攻擊資訊。 例如：

- **攻擊來源**：來源 IP 位址和寄件者電子郵件網域。

- **攻擊訊息屬性**：攻擊訊息的內容、樣式和調性。

- **攻擊收件者**：收件者網域、收件者工作職掌 (系統管理員、執行人員等)、公司類型 (大型、小型、公有、私人等等) 以及產業。

- **攻擊承載**：惡意連結、附件或其他承載。

## <a name="campaign-views-the-office-365-security--compliance-center"></a>Office 365 安全性與合規性中心行銷活動檢視

您可以在下列位置的[安全性與合規性中心](https://docs.microsoft.com/microsoft-365/compliance/go-to-the-securitycompliance-center)中取得行銷活動檢視：

- **威脅管理** \> **總管** \> **檢視** \> **網路釣魚** \> **熱門活動 (預覽)**

- **威脅管理** \> **總管** \> **檢視** \> **所有電子郵件** \> **熱門活動 (預覽)**

![安全性與合規性中心的行銷活動概觀](../media/campaigns-overview.png)

> [!TIP]
> 目前唯一可用的篩選是日期範圍。 如果您沒有看到任何行銷活動資料，請嘗試變更日期範圍。

概觀頁面會顯示有關行銷活動的下列資訊：

- **名稱**

- **樣本主旨**：行銷活動中其中一個訊息的主旨列。 請注意，並非行銷活動中的_所有_訊息都會有這個相同的主旨列。

- **類型**：目前此值將一律是**網路釣魚**。

- **子類型**：本行銷活動要網路釣魚的品牌 (如適用)。 當偵測是由 ATP 技術驅動時，系統會在子類型值中新增字首 **ATP-**。

- **收件者**：此行銷活動鎖定的使用者數目。

- **已傳送**：在其收件匣中收到來自此行銷活動訊息的使用者數目。

- **ID**：行銷活動的唯一識別碼。

當您按一下行銷活動的名稱時，行銷活動詳細資料會顯示在飛出視窗中。

## <a name="campaign-details"></a>行銷活動詳細資料

在行銷活動詳細資料檢視中，提供有關行銷活動的許多資訊：

- 行銷活動資訊：

  - **識別碼**：與概觀畫面的行銷活動相同的唯一識別碼。

  - **開始**和**結束**：您選取的日期範圍篩選。

  - **影響**：在您選取的日期範圍中傳送的訊息數量、「加入收件匣」(也就是傳送到收件匣) 的數量，以及有多少個使用者在網路釣魚訊息中的 URL 承載上點選。

  - 行銷活動時間表：行銷活動的開始和結束時間，以及一段時間內的訊息量。

#### <a name="url-clicks"></a>URL 點選

使用者總是有機會對傳送到收件者的 [收件匣] 或 [垃圾郵件] 資料夾中的訊息採取行動 (亦即，使用者將點選訊息中的惡意 URL)。 如果使用者沒有點選，這是極少程度的成功，不過您當然必須判斷為什麼最初將有害的訊息傳送至其信箱。

如果使用者已點選惡意的 URL，動作會顯示在圖表的 [URL 點選]**** 區域。

![包含使用者 URL 點選的行銷活動詳細資料](../media/campaign-details-with-recipient-actions.png)

- **安全連結封鎖**：此值表示收件者點選訊息中的承載 URL，但它已由組織中的 [ATP 安全連結](atp-safe-links.md) 原則封鎖。

- **安全連結封鎖覆寫**：此值也表示收件者已點選訊息中的承載 URL，而 [ATP 安全連結] 嘗試阻止這些連結，但它們已被允許覆寫封鎖。 您需要調查[安全連結原則](set-up-atp-safe-links-policies.md)，以查看為何允許使用者覆寫安全連結結果，並點選惡意 URL。

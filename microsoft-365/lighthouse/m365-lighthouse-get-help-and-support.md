# <a name="get-help-and-support-for-microsoft-365-lighthouse"></a>取得 Microsoft 365 Lighthouse 的說明和支援 

> [!NOTE]
> 本文所述的功能都是在預覽中進行變更，而且只有符合 [需求](m365-lighthouse-requirements.md)的合作夥伴才能使用。 如果您的組織沒有 Microsoft 365 Lighthouse，請參閱[註冊 Microsoft 365 Lighthouse](m365-lighthouse-sign-up.md)。

如果您需要協助，有數個選項可供使用。 從檢查客戶租使用者服務目前的健康情況開始：

1. 在 Microsoft 365 Lighthouse 的左功能窗格中，選取 [**服務健康情況**]。
2. 查看目前和過去問題的詳細資訊。

若要檢查目前的 Microsoft 365 Lighthouse 承租人服務狀況：

1. 移至 Microsoft 365 系統管理中心 <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">https://admin.microsoft.com</a> 。
2. 在左功能窗格中，選取 [**狀況**  >  **服務健康情況**]。
3. 在服務清單中，尋找 **Microsoft 365 套件**，然後展開以顯示套件中的所有服務。
4. 尋找 **Microsoft 365 Lighthouse** 並檢查健康情況。

如果您遇到的是服務健康情況儀表板中未列出的問題，請依照本文中的指示，查看自助選項或建立服務要求。

> [!NOTE]
> 在預覽 Microsoft 365 Lighthouse 時，支援僅限英文。

## <a name="before-you-begin"></a>在您開始之前

- 若要建立及管理服務要求，您必須至少有一個 Azure Active Directory (Azure AD) 角色指派給您，並設定下列屬性： **supportTickets/allEntities/allTasks**。 如需 Azure AD 角色的清單，請參閱 [AZURE ad 內建角色](/azure/active-directory/roles/permissions-reference)。 如需如何指派角色的詳細資訊，請參閱 [將 AZURE AD 角色指派給使用者](/azure/active-directory/roles/manage-roles-portal)。

- 如果您需要建立服務要求，請盡可能收集有關問題的詳細資訊，例如診斷記錄、租使用者識別碼和使用者 IDs (如果特定使用者受到影響) 。

## <a name="access-help-and-support"></a>存取協助和支援

1.  在 Microsoft 365 Lighthouse 中，選取 **？** 圖示，以開啟 [ **說明] 窗格** ，然後執行下列其中一項操作：
    
    -  如果您位於發生問題的入口網站頁面上，請選取 [ **顯示診斷**]。

        這會建立包含資訊的 JSON 檔案，以協助支援代理人進行問題的疑難排解。 儲存檔案，以便將其附加至服務要求。

        > [!NOTE]
        > JSON 檔案會包含個人身分識別資訊。

    -  如果問題未隔離于入口網站的目前頁面上，請移至下一個步驟。

2.  **在 [說明**] 窗格中，選取 [說明] 及 [**支援**] 按鈕。 這會開啟 [ **我們可如何協助？** ] 窗格。

    > [!NOTE]
    > 如果 [ **我們可以協助嗎？** ] 窗格未開啟，您必須聯繫合作夥伴租使用者中具有全域管理員許可權的人員，並要求他們提供協助。

3.  在 [ **如何協助？** ] 窗格中，輸入您問題的描述，然後按 **enter**。 建議您在描述中包含完整的產品名稱 *Microsoft 365 lighthouse* ，以確保搜尋結果包含相關的説明文章。

4.  查看建議文章的清單，以查看是否有任何可協助解決您的問題。

    如果您輸入問題的描述，但未傳回說明文章的清單，請重新表述描述，然後嘗試其他搜尋。

5.  如果建議的文章不提供協助，請選取 [ **連絡人支援**]。

6.  填寫表單中的資訊，附加所有的螢幕擷取畫面和您在步驟1中儲存的 JSON 檔案 &nbsp; （如果適用的話），然後選取 [ **與我們聯繫**]。 預期的等候時間會顯示在窗格中。
